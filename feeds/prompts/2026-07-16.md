PROMPT_TITLE: Scalable System Blueprint

THE_PROMPT:
You are a systems architect tasked with designing a scalable system for {PURPOSE} that handles {EXPECTED_USER_LOAD} users efficiently. Include the following elements in your design:
1. **Architecture Overview**: Describe the overall structure (e.g., microservices, monolithic) and justify your choice based on scalability requirements.
2. **Technology Stack**: List the technologies you would use for each component (e.g., databases, APIs, front-end frameworks), including rationale for their scalability features.
3. **Load Balancing and Redundancy**: Outline strategies for load distribution and failover mechanisms to ensure system reliability.
4. **Performance Metrics**: Define key performance indicators (KPIs) to monitor system health and scalability.
5. **Future Scaling Considerations**: Explain how the system can adapt to increased load over time, including potential trade-offs.

WHEN_TO_USE:
This prompt is ideal when you need to create robust system designs that anticipate growth in user demand. Use it to explore architecture choices and technology options that ensure longevity and performance as your user base scales.

EXAMPLE_INPUT:
PURPOSE: An e-commerce platform  
EXPECTED_USER_LOAD: 100,000 concurrent users

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a comprehensive system design blueprint, detailing architecture decisions, appropriate technologies, load balancing strategies, performance metrics, and future scalability considerations, tailored to the specified use case.

PRO_TIPS:
- Clearly define your purpose to guide the architectural choices and technology recommendations.
- Specify the expected user load as accurately as possible to tailor the system design effectively.
- Consider including potential bottlenecks in your analysis to proactively address scalability challenges.
