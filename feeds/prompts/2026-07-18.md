PROMPT_TITLE: Clear Technical Documentation Wizard

THE_PROMPT:
Create comprehensive technical documentation for {PRODUCT_OR_FEATURE_NAME}. The documentation should include the following sections:
1. **Overview**: Provide a brief introduction and purpose of {PRODUCT_OR_FEATURE_NAME}.
2. **Installation Instructions**: Outline step-by-step instructions for installation in different environments (e.g., Windows, macOS, Linux).
3. **Usage Guidelines**: Describe how to use {PRODUCT_OR_FEATURE_NAME}, including code snippets or command examples where applicable.
4. **Troubleshooting**: Offer common issues and solutions related to {PRODUCT_OR_FEATURE_NAME}.
5. **FAQs**: Compile frequently asked questions and their answers.
6. **Additional Resources**: List links to related documentation, tutorials, or support forums.

Ensure that the language used is clear and concise and that each section is easy to navigate.

WHEN_TO_USE:
Use this prompt when you need to produce clear, organized, and user-friendly technical documentation for a product or feature. It’s ideal for developers, technical writers, or anyone responsible for creating instructional content.

EXAMPLE_INPUT:
{PRODUCT_OR_FEATURE_NAME} = "DataAnalyzer Pro"

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces structured technical documentation that provides users with in-depth knowledge about "DataAnalyzer Pro." It will offer an intuitive overview, detailed installation instructions, practical usage guidelines, troubleshooting tips, frequently asked questions, and additional resources, creating an accessible guide for both new and experienced users.

PRO_TIPS:
- Use bullet points for clarity in installation and usage instructions.
- Be consistent with terminology throughout the document to avoid confusion.
- Include screenshots or diagrams where possible to enhance understanding.

WORKS_WITH: Claude · GPT-4 · Gemini
