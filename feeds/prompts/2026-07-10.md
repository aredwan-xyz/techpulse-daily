PROMPT_TITLE: Code Vulnerability Audit Checklist

THE_PROMPT:
You are a security auditor tasked with analyzing the following code snippets for potential vulnerabilities. Please assess the code for common security issues such as SQL injection, cross-site scripting (XSS), buffer overflows, improper authentication, and insecure data storage. Focus on best practices in security and provide actionable recommendations for each identified issue. Use the structure below:

1. Code Snippet: {CODE_SNIPPET}
2. Security Assessment:
   - Vulnerability Found: {VULNERABILITY_TYPE}
   - Description: {VULNERABILITY_DESCRIPTION}
   - Severity Level: {SEVERITY_LEVEL}
   - Recommendation: {RECOMMENDATION}
3. Summary of Security Posture: {SUMMARY}

Please ensure your response contains a detailed yet concise evaluation of the code snippet provided.

WHEN_TO_USE:
Use this prompt anytime you have code that needs to be audited for security vulnerabilities. It is suitable for developers and security teams looking to improve the safety of their software before deployment or during regular maintenance.

EXAMPLE_INPUT:
1. Code Snippet: 
```python
cursor.execute("SELECT * FROM users WHERE username = '" + username + "';")
```
2. Security Assessment:
   - Vulnerability Found: SQL Injection
   - Description: The code concatenates user input directly into the query string, making it vulnerable to SQL injection attacks.
   - Severity Level: High
   - Recommendation: Use parameterized queries or prepared statements to mitigate this risk.
3. Summary of Security Posture: The code contains a critical vulnerability that poses a serious risk to application integrity.

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a structured analysis of the provided code snippets, identifying specific vulnerabilities and offering clear, actionable recommendations. The output includes detailed descriptions, severity assessments, and a comprehensive summary of the code's security posture.

PRO_TIPS:
- Always provide clear and concise code snippets to improve accuracy in vulnerability detection.
- Specify the context in which the code is used, as that can influence the type of vulnerabilities present.
- Encourage the use of up-to-date best practices in security to enhance the quality of recommendations.
