PROMPT_TITLE: Code Clarity Refactor

THE_PROMPT:
You are a code refactoring expert. Your task is to refactor the following piece of code to enhance its clarity and maintainability. Consider best practices, standard naming conventions, and the principle of single responsibility when making changes. 

Here is the code that needs refactoring: 
{CODE_SNIPPET} 

Please explain your refactoring decisions step-by-step, focusing on how each change improves readability and maintenance. Additionally, provide comments throughout the refactored code to clarify the purpose of each section.

WHEN_TO_USE:
Use this prompt when you have a block of code that feels convoluted or difficult to understand. This is particularly useful in scenarios where code has been hastily written or when onboarding new team members who need to grasp the project quickly.

EXAMPLE_INPUT:
```
def calc(a, b, op):
    if op == 'add':
        return a + b
    elif op == 'subtract':
        return a - b
    elif op == 'multiply':
        return a * b
    else:
        return None
```

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces refactored code that is more readable and maintainable. It offers clear explanations for each change made, including improvements in naming, structure, and logic flow, making it easier for others to follow and update the code in the future.

PRO_TIPS:
- Ensure you provide a complete code snippet, as context is crucial for effective refactoring.
- Specify any conventions or styles that your team adheres to for consistency.
- Encourage the use of meaningful variable names and smaller functions to enhance clarity.
