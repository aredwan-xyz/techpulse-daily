PROMPT_TITLE: Clear Technical Documentation Template

THE_PROMPT:
You are tasked with creating clear and concise technical documentation for {PRODUCT_NAME}. The documentation should include the following sections:
1. Introduction: Briefly describe what {PRODUCT_NAME} is, its purpose, and its intended audience.
2. Installation Guide: Provide detailed steps on how to install {PRODUCT_NAME}, including prerequisites and troubleshooting tips.
3. Usage Instructions: Outline how to use {PRODUCT_NAME} with clear examples and commands. Include any necessary configuration settings.
4. API Reference: Detail the core functionalities and endpoints of {PRODUCT_NAME} (if applicable), including input/output formats.
5. Frequently Asked Questions (FAQs): List common issues or questions users may have regarding {PRODUCT_NAME} and provide solutions or clarifications.
6. Conclusion: Summarize key points and provide links to additional resources or support channels.

Please ensure the documentation is formatted clearly and is easily navigable, using bullet points, headings, and visual aids where helpful.

WHEN_TO_USE:
Use this prompt when you need to develop comprehensive technical documentation for a new software product or feature. It’s perfect for ensuring consistency and clarity across different sections of the document.

EXAMPLE_INPUT:
PRODUCT_NAME: SuperApp

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces structured and coherent technical documentation that covers all essential aspects of a product. It ensures that the information is accessible to both new and experienced users, with clear instructions, examples, and useful FAQs.

PRO_TIPS:
- Tailor each section to your specific audience to enhance user understanding.
- Use visuals like tables, diagrams, or screenshots to clarify complex concepts.
- Regularly update the documentation with user feedback to improve accuracy and usability.
