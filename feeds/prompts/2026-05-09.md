PROMPT_TITLE: Clear Technical Documentation Template

THE_PROMPT:  
Create a clear and concise technical documentation section for {TOPIC}. This documentation should include the following components:  
1. **Overview**: Write a brief introduction that explains the purpose of {TOPIC} and its importance.  
2. **Installation/Setup Instructions**: Provide step-by-step instructions for installing or setting up {TOPIC}, ensuring that any prerequisites are clearly mentioned.  
3. **Usage Guideline**: Include detailed instructions on how to effectively use {TOPIC}, with code snippets or examples if applicable.  
4. **Common Issues and Troubleshooting**: List potential common issues users might encounter with {TOPIC} and provide troubleshooting steps for each.  
5. **Conclusion**: Wrap up with a summary of key points and encourage users to follow best practices.  

WHEN_TO_USE:  
Use this prompt when you need to produce high-quality, structured technical documentation for software, tools, or any other technical subjects. It is particularly useful for onboarding new users or providing guidelines for developers working with a particular technology.

EXAMPLE_INPUT:  
{TOPIC}: "MySQL Database Integration"

EXAMPLE_OUTPUT_SUMMARY:  
The output generated by this prompt will consist of a well-organized documentation section on "MySQL Database Integration" that covers all necessary elements, making it easy for users to understand and implement the integration effectively. It will ensure that the documentation is comprehensive yet straightforward, addressing both novice and experienced users.

PRO_TIPS:  
- Use clear, simple language to ensure accessibility for all skill levels.  
- Incorporate visual aids, such as diagrams or screenshots, wherever applicable to enhance understanding.  
- Review the generated documentation for accuracy and completeness before sharing it with users.
