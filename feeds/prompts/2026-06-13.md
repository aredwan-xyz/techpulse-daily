PROMPT_TITLE: API Blueprint Creator

THE_PROMPT:
Design an API that adheres to RESTful principles and provides a clear, consistent interface for developers. Your API should include the following components:
1. **API Name:** {API_NAME}
2. **Base URL:** {BASE_URL}
3. **Resource Endpoints:** List the main resources with appropriate HTTP methods (GET, POST, PUT, DELETE) for each. Example: {RESOURCE_ENDPOINT} with {METHODS}
4. **Request Parameters:** Detail the necessary parameters for each endpoint, categorized as headers, query parameters, or body.
5. **Response Structure:** Define the expected response format, including success and error response examples.
6. **Authentication Method:** Specify the type of authentication required (e.g., API Key, OAuth).
7. **Versioning:** Describe your versioning strategy (e.g., URL versioning, header versioning).
8. **Documentation:** Provide a brief plan for user-friendly documentation and how you will ensure consistent API usability.

WHEN_TO_USE:
Use this prompt when you need to create a new API from scratch or refine an existing one. It’s ideal for ensuring that your API is consistent and intuitive, thus improving developer experience and integration.

EXAMPLE_INPUT:
API_NAME: User Management API
BASE_URL: https://api.example.com/v1
RESOURCE_ENDPOINT: /users with GET, POST, PUT, DELETE
REQUEST_PARAMETERS: headers: Authorization, body: { "username": "string", "password": "string" }
RESPONSE_STRUCTURE: Success example: { "id": 1, "username": "john_doe" }, Error example: { "error": "User not found" }
AUTHENTICATION_METHOD: Bearer Token
VERSIONING: URL versioning 
DOCUMENTATION: Use Swagger for the API documentation.

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a comprehensive blueprint of an API that covers all essential aspects, including endpoints, request/response formats, and authentication. It results in structured and user-friendly API designs that are easy to implement and understand.

PRO_TIPS:
- Define clear and logical resource names to enhance API clarity.
- Ensure all responses contain meaningful error messages to assist developers.
- Keep the documentation up to date and consider feedback from users to continually improve the API.
