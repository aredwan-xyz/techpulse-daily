PROMPT_TITLE: Complex Bug Diagnostic Framework

THE_PROMPT:
I’m experiencing a complex bug in my {PROGRAM_NAME} application, specifically in the {MODULE_NAME} module. The bug manifests as {DESCRIPTION_OF_BUG} during {CONDITIONS_TRIGGERING_BUG}. I have already attempted the following troubleshooting steps: {LIST_OF_ATTEMPTS}. The technology stack includes {TECHNOLOGY_STACK}. Based on this information, please provide a step-by-step diagnosis and recommendations for resolving this issue, including possible root causes and how to test your proposed solutions.

WHEN_TO_USE:
This prompt is ideal when you're facing persistent bugs that are difficult to isolate and have already tried common fixes without success. It provides a structured approach to diagnosing and resolving issues in complex software environments.

EXAMPLE_INPUT:
I’m experiencing a complex bug in my web application, specifically in the user authentication module. The bug manifests as a '401 Unauthorized' error during login attempts with valid credentials. I have already attempted the following troubleshooting steps: resetting the API keys, checking CORS settings, and reviewing server logs. The technology stack includes Node.js, Express, and MongoDB.

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a detailed diagnosis that includes potential root causes, suggested debugging steps, and testing methods. It aligns your findings with the specified conditions and offers insights tailored to the technology stack used.

PRO_TIPS:
- Provide as much context as possible for accurate recommendations.
- List all relevant configurations and versions of the software involved.
- Include any error logs or screenshots that may help clarify the issue.
