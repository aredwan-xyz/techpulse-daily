PROMPT_TITLE: Comprehensive Code Review Request

THE_PROMPT:
Please provide a detailed code review for the following code snippet. Focus on readability, maintainability, performance, and adherence to best practices. Here’s the information you need to include:

1. **Code Context:** Briefly describe the purpose of this code and how it fits into the larger project.
2. **Code Snippet:** Include the actual code you would like reviewed (use triple backticks for formatting).
3. **Areas of Concern:** Specify any particular aspects of the code you want feedback on, such as logic correctness, optimization possibilities, or potential bugs.
4. **Reference Standards:** List any relevant coding standards or conventions (e.g., language-specific guidelines or company policies).
5. **Outcome Desired:** Clarify what you aim to achieve with this review, whether it's to improve efficiency, fix errors, or prepare for a production release.

Please structure your feedback clearly, and provide suggestions for improvement along with justifications for each point.

WHEN_TO_USE:
Use this prompt when you have written a piece of code and are seeking an in-depth review from a senior engineer. It is particularly useful for ensuring high-quality code before merging into a main branch or deploying to production.

EXAMPLE_INPUT:
Code Context: This function processes user input for the signup form.  
Code Snippet:
```python
def signup_user(data):
    if not validate_data(data):
        return "Invalid data"
    save_to_database(data)
    return "User signed up successfully"
```
Areas of Concern: Logic correctness and performance.  
Reference Standards: Follow PEP 8 guidelines.  
Outcome Desired: Improve the function for scalability.

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a thorough review highlighting strengths and weaknesses of the code, offering specific suggestions for improvement. The feedback includes not only code quality tips but also insights into performance optimizations and best practice adherence.

PRO_TIPS:
- Be clear and concise in your areas of concern to receive focused feedback.
- Provide enough context about the project for a more informed review.
- Express any time constraints for a quicker response if needed.
