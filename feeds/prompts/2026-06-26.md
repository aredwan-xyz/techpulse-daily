PROMPT_TITLE: **Comprehensive Security Code Audit**

THE_PROMPT:
You are a security expert tasked with performing a thorough code audit on the provided code snippet. Your objective is to identify potential security vulnerabilities and suggest best practices for remediation. Carefully analyze the code, considering common vulnerabilities such as SQL Injection, Cross-Site Scripting (XSS), Insecure Direct Object References (IDOR), and others. 

Please review the following code snippet:
{CODE_SNIPPET}

Provide a detailed report including:
1. A summary of identified vulnerabilities.
2. Potential impact of each vulnerability.
3. Recommended fixes or best practices to improve security.
4. Any other observations that can enhance the code's security posture.

WHEN_TO_USE:
Use this prompt when you need to assess a piece of code for security flaws, whether it's part of an application or a specific module. It's ideal for developers seeking to harden their code or security professionals conducting an internal code review.

EXAMPLE_INPUT:
```
def login(username, password):
    cursor = db.cursor()
    cursor.execute("SELECT * FROM users WHERE username = '{}' AND password = '{}';".format(username, password))
    return cursor.fetchone()
```

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a detailed analysis of the vulnerabilities present in the code, such as SQL Injection due to string formatting in the SQL query. It will recommend the use of parameterized queries and other coding best practices, highlighting both the risks and suggested security enhancements.

PRO_TIPS:
- Always include code segments that are relevant and contain potential vulnerabilities.
- Be explicit about the programming language and frameworks used for better context.
- Specify any known security standards or compliance requirements to be considered during the audit.
