PROMPT_TITLE: Comprehensive Test Suite Generator

THE_PROMPT:
"Create a comprehensive test suite for the {APPLICATION_NAME} application, which is designed for {APPLICATION_PURPOSE}. The test suite should include the following categories: 
1. **Unit Tests**: Detail at least {NUMBER_OF_UNIT_TESTS} unit tests focusing on individual components, including their expected input, output, and edge cases. 
2. **Integration Tests**: Outline {NUMBER_OF_INTEGRATION_TESTS} integration tests that evaluate how various components work together, specifying any necessary mock data and expected outcomes. 
3. **End-to-End Tests**: Define {NUMBER_OF_END_TO_END_TESTS} end-to-end tests that simulate real user scenarios, complete with predefined user actions and the expected results. 
4. **Performance Tests**: Recommend {NUMBER_OF_PERFORMANCE_TESTS} performance tests that measure the application's responsiveness and stability under load.

Additionally, provide any necessary setup instructions for running these tests along with configuration details."

WHEN_TO_USE:
Use this prompt when you need to quickly generate a detailed and structured test suite for a software application. It’s particularly useful during the development phase or when preparing for a deployment to ensure comprehensive testing coverage.

EXAMPLE_INPUT:
APPLICATION_NAME: "E-commerce Platform"  
APPLICATION_PURPOSE: "allow users to buy and sell products online"  
NUMBER_OF_UNIT_TESTS: 5  
NUMBER_OF_INTEGRATION_TESTS: 3  
NUMBER_OF_END_TO_END_TESTS: 4  
NUMBER_OF_PERFORMANCE_TESTS: 2  

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a well-structured test suite that includes clearly defined unit, integration, end-to-end, and performance tests. Each category is populated with specific test cases, anticipated outcomes, and detailed setup instructions, ensuring thorough coverage for the application.

PRO_TIPS:
- Be specific with the application name and purpose to tailor the test cases.
- Adjust the number of tests according to the complexity of your application for greater coverage.
- Include additional notes on specific user roles or environments if relevant to enhance test accuracy.
