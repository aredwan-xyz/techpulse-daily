PROMPT_TITLE: Design a Consistent API Blueprint

THE_PROMPT:
You are tasked with designing a clean and consistent API for a project that focuses on {PROJECT_NAME}. The API should exhibit the following qualities: clarity, simplicity, and adherence to best practices. Please consider the following aspects while crafting your API design:

1. **Base URL**: What should the base URL look like for {PROJECT_NAME}?
2. **Resource Naming**: Suggest a consistent naming scheme for the main resources (e.g., /users, /products) within the API.
3. **HTTP Methods**: Define which HTTP methods (GET, POST, PUT, DELETE) should be implemented for these resources and their purposes.
4. **Response Structure**: What should the typical response structure look like? Include status codes, error handling, and data formats (JSON/XML) for success and error scenarios.
5. **Versioning**: How will the API be versioned to ensure backwards compatibility in the future?
6. **Authentication**: Recommend a secure authentication strategy (e.g., API keys, OAuth) to protect the API endpoints.

Please provide a detailed blueprint based on these specifications, incorporating any additional relevant information.

WHEN_TO_USE:
Use this prompt when you need to design a new API or refine an existing one, ensuring it meets industry standards and user needs. It's particularly useful for developers looking to improve their API's usability and maintainability.

EXAMPLE_INPUT:
PROJECT_NAME: Online Bookstore

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a comprehensive and detailed API design blueprint that outlines aspects like resource naming, endpoint definitions, HTTP methods, response structures, versioning, and authentication methods. It helps guide developers to create APIs that are intuitive and align with best practices.

PRO_TIPS:
- Clearly define the purpose of your API project to generate more relevant designs.
- Use examples from existing popular APIs for inspiration on naming conventions and structures.
- Keep user experience in mind to create a more intuitive API.
