PROMPT_TITLE: Scalable System Blueprint Design

THE_PROMPT:
You are an expert system architect tasked with designing a scalable system for a {SPECIFIC_USE_CASE} that can handle {EXPECTED_LOAD} users while ensuring {KEY_REQUIREMENTS}. Please provide a detailed architecture plan that includes: 
1. The choice of cloud infrastructure (e.g., AWS, GCP, Azure) and justification for it.
2. A breakdown of microservices to be implemented, including their functions and how they interact.
3. Strategies for load balancing and fault tolerance.
4. Data storage solutions that can efficiently handle the expected data volume.
5. Security measures to protect the system from potential threats.
Make sure to highlight key trade-offs and alternatives considered during your design process.

WHEN_TO_USE:
Use this prompt when needing to design a scalable system architecture for applications with specific performance requirements and user load expectations. It's particularly useful during the planning phase of software development projects.

EXAMPLE_INPUT:
- SPECIFIC_USE_CASE: e-commerce platform
- EXPECTED_LOAD: 100,000 concurrent
- KEY_REQUIREMENTS: high availability, low latency, and robust security

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a comprehensive blueprint for the scalable system, detailing the chosen architecture, microservices, load balancing strategies, data storage solutions, and security measures. The response will also justify the choices made and outline potential trade-offs.

PRO_TIPS:
- Clearly define the specific use case to ensure tailored solutions.
- Provide realistic expected load figures for more applicable design recommendations.
- Ask for alternatives to broaden the perspective on potential implementations.

WORKS_WITH: Claude · GPT-4 · Gemini
