PROMPT_TITLE: Trade-off Analysis for Architectural Decisions

THE_PROMPT:
You are tasked with evaluating the trade-offs of a specific architectural decision. Please fill in the variables below for a comprehensive analysis.

1. Decision Context: {TYPE_OF_SYSTEM} (e.g., web application, mobile app, cloud infrastructure) 
2. Alternatives: {OPTION_1}, {OPTION_2}, {OPTION_3} 
3. Criteria: {CRITERIA_1}, {CRITERIA_2}, {CRITERIA_3} (e.g., cost, scalability, performance, maintainability)
4. Stakeholders: {STAKEHOLDER_GROUPS} (e.g., developers, end-users, business leaders)

Analyze the above architectural decision by evaluating each option against the criteria. Provide a summary of strengths and weaknesses, considering the perspective of involved stakeholders. Include recommendations for which option might be best suited given the context.

WHEN_TO_USE:
This prompt is ideal for software architects, project managers, or engineers who need to weigh architectural options for technological implementations. It helps in making informed decisions by structuring the evaluation clearly.

EXAMPLE_INPUT:
Decision Context: Cloud Infrastructure  
Alternatives: AWS, Azure, Google Cloud  
Criteria: Cost, Scalability, Performance  
Stakeholders: Developers, Business Leaders

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a detailed comparison of AWS, Azure, and Google Cloud based on cost, scalability, and performance. It will highlight which service excels in each criterion, discuss potential trade-offs, and conclude with a recommendation suited for the stakeholders’ needs.

PRO_TIPS:
- Clearly define all stakeholders involved to ensure their perspectives are adequately addressed.
- Use distinct, quantifiable criteria to improve the depth of the analysis.
- Consider adding a risk assessment for each alternative to enhance decision-making insight.
