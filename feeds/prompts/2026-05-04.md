PROMPT_TITLE: Comprehensive Test Suite Generator

THE_PROMPT:
Generate a comprehensive test suite for the {APPLICATION_NAME} application, which serves as a {APPLICATION_PURPOSE}. Your test suite should cover the following categories: functional tests, non-functional tests, edge cases, and performance tests. Each test case should include:
1. Test Case ID
2. Test Case Description
3. Pre-conditions
4. Test Steps
5. Expected Result
Additionally, provide any relevant test data or configurations needed for each test case. Focus on real-world scenarios and ensure thorough coverage of the application's functionality.

WHEN_TO_USE:
Use this prompt when you need to create a detailed and structured test suite for a software application. It’s ideal for QA teams looking to streamline their testing processes and ensure comprehensive coverage of functional and non-functional requirements.

EXAMPLE_INPUT:
APPLICATION_NAME: "Chat App"
APPLICATION_PURPOSE: "a real-time messaging platform for users to communicate securely"

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a structured test suite that includes multiple test cases covering functionalities like sending messages, user registration, and error handling while also addressing performance metrics under load conditions. Each test case is detailed and provides clear steps, expected outcomes, and necessary configurations.

PRO_TIPS:
- Clearly define the application's purpose and functionalities for more tailored test cases.
- Consider including additional categories for specialized testing (e.g., security, usability).
- Keep your test data realistic and representative of actual user inputs for more effective testing. 

WORKS_WITH: Claude · GPT-4 · Gemini
