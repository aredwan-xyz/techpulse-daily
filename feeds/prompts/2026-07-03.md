PROMPT_TITLE: Bug Detective

THE_PROMPT:
As a debugging specialist, I need you to help me diagnose and fix a complex bug in my software. The details are as follows: 
- Software Name: {SOFTWARE_NAME}
- Programming Language: {LANGUAGE}
- Description of the bug: {BUG_DESCRIPTION}
- Expected behavior: {EXPECTED_BEHAVIOR}
- Actual behavior: {ACTUAL_BEHAVIOR}
- Steps to reproduce the bug: {REPRODUCTION_STEPS}
- Any error messages received: {ERROR_MESSAGES}
- Any recent changes made to the code or environment: {RECENT_CHANGES}

Please analyze this information and provide possible root causes of the bug, suggested debugging steps, and specific code fixes or changes I should implement to resolve the issue.

WHEN_TO_USE:
Use this prompt when you encounter a challenging bug that is difficult to diagnose and fix. It is particularly useful for complex errors in software that have a significant impact on functionality.

EXAMPLE_INPUT:
- SOFTWARE_NAME: MyWebApp
- LANGUAGE: JavaScript
- BUG_DESCRIPTION: User authentication fails intermittently.
- EXPECTED_BEHAVIOR: Users should be able to log in successfully at all times.
- ACTUAL_BEHAVIOR: Users receive an "Invalid credentials" message sporadically.
- REPRODUCTION_STEPS: 1. Open login page. 2. Enter valid credentials. 3. Click login.
- ERROR_MESSAGES: "Invalid credentials."
- RECENT_CHANGES: Updated authentication library to version 2.0.

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a list of potential root causes for the authentication issue, step-by-step debugging strategies, and precise code modifications to enhance reliability. It will guide the user through logical debugging processes, ensuring clarity and direction.

PRO_TIPS:
- Include as much detail as possible in your inputs to enhance diagnostic accuracy.
- If possible, share snippets of code for more targeted advice.
- Maintain a clear timeline of recent changes, as this context often reveals key factors in debugging.

WORKS_WITH: Claude · GPT-4 · Gemini
