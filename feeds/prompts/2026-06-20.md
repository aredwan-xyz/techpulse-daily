PROMPT_TITLE: Write Technical Documentation for Software Features

THE_PROMPT:
You are tasked with creating clear and comprehensive technical documentation for a software feature. Start with a concise introduction that outlines the feature's purpose and benefits. Use the following structure to organize your documentation:
1. **Feature Overview**: {Feature Name}
   - Describe what the feature does and why it is important.
2. **Prerequisites**: List any requirements or dependencies needed to use the feature, including software versions or configurations.
3. **Getting Started**: Provide step-by-step instructions on how to access or set up the feature.
4. **Usage Instructions**: Include examples that demonstrate how to use the feature effectively. Provide screenshots or code snippets if applicable.
5. **Common Issues and Troubleshooting**: Document potential problems users may encounter while using the feature and their solutions.
6. **FAQs**: Address frequent questions regarding the feature, focusing on common concerns.
7. **Conclusion**: Summarize the key takeaways and encourage users to reach out for further support if needed.

WHEN_TO_USE:
Use this prompt when you need to create detailed and structured documentation that helps users understand software features thoroughly. It’s perfect for onboarding new users or for complex functionalities that require thorough explanations.

EXAMPLE_INPUT:
Feature Name: User Authentication
Prerequisites: Must have an account and be logged into the system.
Getting Started: Visit the login page and enter your credentials.
Usage Instructions: After logging in, users will see their dashboard.

EXAMPLE_OUTPUT_SUMMARY:
This prompt generates a well-organized technical document that effectively conveys important information about a software feature. The output typically includes sections covering an overview, setups, usage examples, troubleshooting tips, and FAQs, making it easy for users to follow.

PRO_TIPS:
- Provide as much detail as possible in the feature overview to set clear expectations.
- Include real-world examples in the usage instructions to enhance user understanding.
- Regularly update the troubleshooting section based on user feedback to keep documentation relevant.
