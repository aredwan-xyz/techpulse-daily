PROMPT_TITLE: Design a Cohesive API Blueprint

THE_PROMPT:
You are tasked with designing a clean and consistent API for a {APPLICATION_NAME} that serves {TARGET_AUDIENCE}. First, outline the core functionalities the API should provide, including endpoints for {FUNCTIONALITY_1}, {FUNCTIONALITY_2}, and {FUNCTIONALITY_3}. Next, specify the request and response formats using JSON for clarity. Ensure that you use consistent naming conventions, verb tenses, and HTTP methods across all endpoints. Provide an example for each endpoint, including sample request and response payloads. Finally, detail any potential errors and their corresponding messages or codes to help developers understand how to handle them effectively. 

WHEN_TO_USE:
Use this prompt when you need to create a new API or refactor an existing one to ensure it meets modern design principles. This is ideal for developers looking to enhance user experience and streamline interactions with their applications.

EXAMPLE_INPUT:
APPLICATION_NAME: "Social Media Analytics Tool"
TARGET_AUDIENCE: "Marketing Teams"
FUNCTIONALITY_1: "User Engagement Data"
FUNCTIONALITY_2: "Content Performance Metrics"
FUNCTIONALITY_3: "Audience Insights"

EXAMPLE_OUTPUT_SUMMARY:
This prompt will produce a detailed API design document that outlines key functionalities, endpoint structures, request and response formats, and error handling mechanisms. Users can expect a well-organized blueprint that facilitates clean coding practices and more efficient development cycles.

PRO_TIPS:
- Consider the end-users when defining functionalities; they should solve real problems.
- Use clear, descriptive names for endpoints to enhance readability and usability.
- Leverage versioning in your API design to ensure future compatibility and ease of updates.
