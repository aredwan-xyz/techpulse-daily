PROMPT_TITLE: Consistent API Blueprint Creator

THE_PROMPT:
You are an API design specialist tasked with crafting a clean and consistent API specification for a {APPLICATION_NAME} application. The API must adhere to RESTful standards and provide a straightforward interface for developers. Include the following components in your API design:
1. Define the primary resources and their relationships, such as {RESOURCE_1}, {RESOURCE_2}, etc.
2. Detail the endpoints for each resource, specifying HTTP methods, request parameters, and response formats.
3. Ensure all endpoint responses follow a consistent structure, including success and error messages.
4. Incorporate versioning strategies for the API, considering future changes and backward compatibility.
5. Specify authentication methods (e.g., OAuth2, API keys) that will secure the API.

Provide a summary document that outlines usage examples, potential use cases, and best practices for developers who interact with your API. 

WHEN_TO_USE:
Use this prompt when you need to design a new API from scratch or when updating an existing API to improve clarity and usability. It's particularly useful for projects requiring a systematic approach to resource management and endpoint architecture.

EXAMPLE_INPUT:
APPLICATION_NAME: BookStore
RESOURCE_1: User
RESOURCE_2: Book

EXAMPLE_OUTPUT_SUMMARY:
This prompt consistently produces a structured API specification document, detailing endpoints, resources, and best practices tailored for the specified application. The output will include clear descriptions for each resource, example requests and responses, and guidelines for maintaining API integrity over time.

PRO_TIPS:
- Clearly define the relationships between your resources to enhance understanding and minimize confusion.
- Keep your endpoint names intuitive and consistent to ensure ease of use for developers.
- Regularly review and iterate on your API design as user feedback and compliance requirements evolve.
