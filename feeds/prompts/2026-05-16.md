PROMPT_TITLE: API Blueprint Constructor

THE_PROMPT:
Design a clean and consistent API for {APPLICATION_NAME} that serves the purpose of {API_PURPOSE}. Consider the following components in your design: 
1. **Endpoints**: Define at least {NUMBER_OF_ENDPOINTS} endpoints. For each, specify the HTTP methods required (GET, POST, PUT, DELETE), along with a brief description of its functionality.
2. **Request/Response Structures**: For each endpoint, outline the request parameters and the expected response structure, including status codes and examples.
3. **Authentication/Authorization**: Detail the authentication method to be used (e.g., OAuth 2.0, API keys), and any required user roles/permissions.
4. **Error Handling**: Create a list of potential error responses and their meanings.
5. **Documentation Plan**: Explain how users will access documentation for the API, including formats (OpenAPI/Swagger) and how to keep it updated.

WHEN_TO_USE:
Use this prompt when tasked with designing an API from scratch or refining an existing one. It ensures that the design is comprehensive and adheres to best practices, making it user-friendly for developers.

EXAMPLE_INPUT:
APPLICATION_NAME: PhotoShare 
API_PURPOSE: Allow users to upload, share, and manage photos 
NUMBER_OF_ENDPOINTS: 5 

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a detailed API design that includes a structured endpoint layout, request and response details, authentication protocols, and error handling strategies. It also highlights the importance of documentation to aid developers.

PRO_TIPS:
- Be specific with the application name and purpose to tailor the design to your needs.
- Provide as many endpoints as possible to create a more robust API.
- Consider the end user's experience when defining request and response structures.
