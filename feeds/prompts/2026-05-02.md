PROMPT_TITLE: API Design Blueprint

THE_PROMPT:
Design a clean and consistent API for a {APPLICATION_TYPE} that serves {PRIMARY_FUNCTION} using {DATA_FORMAT}. Your API should include the following endpoints:
1. **GET /{RESOURCE}**: Retrieves a list of {RESOURCE}s.
2. **GET /{RESOURCE}/{ID}**: Retrieves a specific {RESOURCE} by its unique identifier.
3. **POST /{RESOURCE}**: Creates a new {RESOURCE} entry.
4. **PUT /{RESOURCE}/{ID}**: Updates an existing {RESOURCE} entry by its ID.
5. **DELETE /{RESOURCE}/{ID}**: Deletes a {RESOURCE} entry by its ID.

For each endpoint, define:
- **Parameters**: (if any)
- **Request Body**: (for POST and PUT)
- **Response Format**: Ensure consistent structure, including success and error responses.

Additionally, outline the authentication method for your API (e.g., API keys, OAuth) and provide versioning strategy (e.g., /v1/).

WHEN_TO_USE:
This prompt is ideal for API developers and architects looking to create a robust, user-friendly API that adheres to best practices. It helps ensure a clear structure and common conventions, making development and maintenance easier.

EXAMPLE_INPUT:
APPLICATION_TYPE: e-commerce platform
PRIMARY_FUNCTION: manage products
DATA_FORMAT: JSON
RESOURCE: products

EXAMPLE_OUTPUT_SUMMARY:
This prompt yields a comprehensive API design, detailing endpoints for a product management system in an e-commerce context. It provides structured definitions of requests and responses, along with authentication strategies and versioning considerations, ensuring a well-organized and user-friendly API design.

PRO_TIPS:
- Clearly define your primary function and the data format to ensure consistency.
- Consider user authentication and error handling early in the design to avoid later complications.
- Review existing API standards (e.g., RESTful, GraphQL) that align with your application's needs for best practices.
