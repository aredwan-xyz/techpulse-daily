PROMPT_TITLE: Crystal Clear Product Requirements

THE_PROMPT:
You are a product manager tasked with writing clear product requirements for a new product. Please define the following sections:

1. **Product Name**: {PRODUCT_NAME}
2. **Overview**: Write a brief description of the product, its purpose, and target audience.
3. **Objectives**: List 3-5 key objectives of this product (e.g., increase user engagement by X%).
4. **Features**: Outline at least 5 core features and functionalities, specifying what each feature will do.
5. **User Experience**: Describe the expected user journey and how users will interact with the product.
6. **Success Metrics**: Identify key performance indicators (KPIs) that will measure the product’s success (e.g., user retention rates, sales volume).
7. **Timeline**: Provide an estimated timeline for development phases (e.g., design, testing, launch).

Please ensure that your requirements are concise, clear, and actionable.

WHEN_TO_USE:
Use this prompt whenever you need to create comprehensive and clear product specifications that will guide the development team and stakeholders. It is especially useful during the ideation phase of a project or when formalizing an existing idea.

EXAMPLE_INPUT:
PRODUCT_NAME: Smart Fitness Tracker
Overview: A wearable device designed to monitor fitness levels, track progress, and provide personalized workout recommendations for health-conscious individuals aged 18-40.
Objectives: 1. Improve user health by encouraging regular exercise. 2. Increase engagement by 50% over 6 months. 3. Collect user data to enhance personalized features.
Features: 1. Heart rate monitoring 2. Step tracking 3. Integration with smartphone health apps 4. Custom workout plans 5. Sleep quality analysis
User Experience: Users will wear the tracker, choose workout goals via an app, and receive real-time feedback. They’ll also track their progress over time.
Success Metrics: Monthly active users, average workout sessions per week, and user satisfaction ratings.
Timeline: 3 months for design, 6 months for development, and 2 months for testing.

EXAMPLE_OUTPUT_SUMMARY:
This prompt will generate a well-structured product requirements document that clearly outlines objectives, features, user experiences, and performance metrics. The output will be actionable and tailored to guide teams through the product development process.

PRO_TIPS:
- Be as specific as possible when defining features and objectives to ensure clarity.
- Prioritize the most important aspects in your overview for better focus.
- Regularly revisit and revise your requirements as the project evolves to keep them relevant.
