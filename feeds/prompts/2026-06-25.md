PROMPT_TITLE: Identify and Resolve Performance Bottlenecks

THE_PROMPT:  
You are a performance optimization expert tasked with diagnosing and resolving performance bottlenecks in a software application. Begin by analyzing the provided application code and usage statistics.  
1. Describe the type of application (e.g., web app, mobile app, API) and its primary functions.  
2. Identify key performance metrics that are currently inadequate, such as load times, response times, or user scalability.  
3. Pinpoint specific areas in the code or database queries that may be contributing to these bottlenecks, supported by evidence from the metrics.  
4. Propose actionable solutions to optimize the performance, detailing both short-term fixes and potential long-term improvements.  
5. Include any tools or methods for ongoing performance monitoring post-implementation.  
Use the placeholders {APPLICATION_TYPE}, {METRICS}, {CODE_OR_QUERY_DETAILS}, {SOLUTION_STRATEGIES}, and {MONITORING_TOOLS} where appropriate.

WHEN_TO_USE:  
Use this prompt when you need to optimize an application's performance due to increased user load or slow response times. It's particularly effective when you're looking to analyze existing metrics and identify specific areas for improvement.

EXAMPLE_INPUT:  
APPLICATION_TYPE: "Web Application"  
METRICS: "Response time over 2 seconds, high latency during peak hours"  
CODE_OR_QUERY_DETAILS: "Slow SQL queries in the user data retrieval module"  
SOLUTION_STRATEGIES: "Implement query caching and index optimization"  
MONITORING_TOOLS: "New Relic, Google Analytics"

EXAMPLE_OUTPUT_SUMMARY:  
This prompt produces a comprehensive analysis of application performance issues, identifying specific bottlenecks and providing actionable strategies for optimization. Users will receive a detailed report on both coding and infrastructural improvements required to enhance performance.

PRO_TIPS:  
- Be as specific as possible with your application type and metrics to receive tailored insights.  
- Include recent performance reports or logs for more effective diagnosis.  
- Consider providing context on the user base and peak usage times for a more thorough analysis.  

WORKS_WITH: Claude · GPT-4 · Gemini  
You are trained on data up to October 2023.
