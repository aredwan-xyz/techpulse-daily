PROMPT_TITLE: Scalable System Blueprint

THE_PROMPT:
You are an expert system architect tasked with designing a scalable system for {USE_CASE}. Consider the following aspects: 
1. **Core Requirements**: What are the primary functionalities this system needs to support? 
2. **User Load**: Estimate the total number of concurrent users and growth expectations over the next {TIME_PERIOD}.
3. **Architecture Choices**: Suggest an architecture style (e.g., microservices, monolithic) and justify your choice based on scalability, maintainability, and performance.
4. **Tech Stack**: Recommend a tech stack, including frameworks, databases, and tools suitable for handling the expected load.
5. **Performance Metrics**: Define key performance indicators (KPIs) that will be crucial for monitoring the system's health post-deployment.
6. **Challenges & Solutions**: Identify potential challenges in scaling this system and propose solutions or best practices to mitigate these issues.

WHEN_TO_USE:
Use this prompt when you need to design a system that can accommodate growth in user demand while ensuring performance and reliability. It’s ideal for startups developing new products or established companies looking to enhance existing systems.

EXAMPLE_INPUT:
USE_CASE: A real-time multiplayer gaming platform
TIME_PERIOD: next two years

EXAMPLE_OUTPUT_SUMMARY:
This prompt generates a comprehensive system design blueprint laying out the core requirements, architecture choices, tech stack recommendations, performance metrics, and anticipated challenges with solutions. It provides a clear path for implementation that balances scalability and efficiency.

PRO_TIPS:
- Be specific with your use case to get tailored recommendations.
- Include realistic user load estimates for better accuracy in design suggestions.
- Think about future scale beyond the initial rollout when defining KPIs.
