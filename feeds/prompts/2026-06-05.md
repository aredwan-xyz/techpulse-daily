PROMPT_TITLE: Debugging Detective

THE_PROMPT:
You are a debugging expert tasked with diagnosing and fixing a complex bug in the code. Please provide a detailed analysis using the information below.  
1. **Code Snippet**: {CODE_SNIPPET}  
2. **Error Message**: {ERROR_MESSAGE}  
3. **Expected Behavior**: {EXPECTED_BEHAVIOR}  
4. **Steps to Reproduce**: {STEPS_TO_REPRODUCE}  
5. **Environment Information**: {ENVIRONMENT_INFO}  

Analyze the provided details and follow these steps:
- Identify potential causes for the bug.
- Suggest specific debugging techniques to isolate the problem.
- Provide at least two possible solutions with code examples, if applicable.
- Recommend best practices to prevent similar issues in the future.

WHEN_TO_USE:
Use this prompt when facing challenging bugs that require deep analysis and debugging strategies. It is particularly useful for software developers looking to get structured guidance on resolving issues in their codebase.

EXAMPLE_INPUT:
1. **Code Snippet**: `def add(a, b): return a + b`  
2. **Error Message**: `TypeError: unsupported operand type(s) for +: 'int' and 'str'`  
3. **Expected Behavior**: `The function should return the sum of two numbers.`  
4. **Steps to Reproduce**: `add(5, '10')`  
5. **Environment Information**: `Python 3.8, running on Windows 10`

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a comprehensive analysis of the bug, identifying possible causes, outlining effective debugging techniques, and offering practical solutions with code examples. Additionally, it provides preventive measures to avoid similar issues.

PRO_TIPS:
- Ensure your code snippet is complete and includes necessary imports or context for better diagnosis.
- Include the specific line number or location of the error in your steps to reproduce for more accurate suggestions.
- Provide any recent changes made to the code or environment to help pinpoint the bug's origin.
