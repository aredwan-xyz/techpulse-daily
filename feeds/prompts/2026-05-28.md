PROMPT_TITLE: Diagnose and Optimize Performance

THE_PROMPT:
You are a performance optimization expert. Analyze the provided system or application to identify potential performance bottlenecks. Use the following structure to provide your analysis: 

1. **Overview of the System/Application**: Briefly describe what the system does.
2. **Identified Bottlenecks**: List the primary performance bottlenecks you have discovered, including metrics where applicable (e.g., response time, CPU usage).
3. **Impact Analysis**: Explain how these bottlenecks affect overall system performance and user experience.
4. **Recommended Solutions**: Provide specific and actionable recommendations for fixing the identified bottlenecks. Include potential optimizations or coding practices.
5. **Expected Improvements**: Estimate the performance improvements (e.g., percentages or time reductions) that can be achieved by implementing your recommendations.

Please fill in the details below:
- System/Application Name: {SYSTEM_NAME}
- Description of the System: {DESCRIPTION}
- Performance Metrics: {METRICS}

WHEN_TO_USE:
Use this prompt when you want to diagnose performance issues in software applications or systems, whether during development or after deployment. It's ideal for developers, system architects, or performance analysts tasked with optimizing existing solutions.

EXAMPLE_INPUT:
- System/Application Name: Online Shopping Portal
- Description of the System: A web-based platform for buying and selling products with user reviews and payment processing.
- Performance Metrics: Average page load time 5 seconds, CPU usage 85% during peak hours

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a structured analysis of performance bottlenecks, detailing their impact, and provides practical, prioritized recommendations for optimizations. Users can expect estimates of potential performance improvements, which can guide their implementation efforts.

PRO_TIPS:
- Provide detailed performance metrics to get more tailored recommendations.
- Include specific user scenarios to help identify bottlenecks that affect user experience.
- Consider the environment context (dev, staging, production) for more relevant insights.

WORKS_WITH: Claude · GPT-4 · Gemini
