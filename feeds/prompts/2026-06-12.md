PROMPT_TITLE: Security Code Audit Checklist

THE_PROMPT:
I am conducting a security audit on the following codebase: {CODEBASE_NAME}. Please analyze the provided code and identify potential security vulnerabilities, including but not limited to: SQL injection, cross-site scripting (XSS), insecure authentication, and exposure of sensitive data. Detail your findings in sections, including a brief description of each vulnerability {VULNERABILITY_TYPE}, the impact assessment, and recommendations for remediation. If applicable, provide code snippets to illustrate the issues and potential fixes. The code in question is: {CODE_SNIPPET}.

WHEN_TO_USE:
Use this prompt when you have a specific codebase you want to assess for security vulnerabilities. It is ideal for developers and security professionals looking to enhance the security posture of their applications before deployment or during regular security reviews.

EXAMPLE_INPUT:
CODEBASE_NAME: "Inventory Management System"
VULNERABILITY_TYPE: "SQL injection, XSS, Insecure authentication"
CODE_SNIPPET: "SELECT * FROM users WHERE username = '$username';"

EXAMPLE_OUTPUT_SUMMARY:
This prompt produces a structured analysis of the code snippet, listing identified vulnerabilities along with their potential impacts. It provides actionable recommendations for mitigating risks, illustrated with code examples to clarify the fixes.

PRO_TIPS:
- Ensure the code snippet is self-contained to avoid confusion and to facilitate more accurate assessments.
- Specify any frameworks or libraries used in the codebase to tailor the vulnerability analysis effectively.
- Include context about the application's purpose to assist in evaluating the severity of the identified vulnerabilities.
