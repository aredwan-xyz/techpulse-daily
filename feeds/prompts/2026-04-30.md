PROMPT_TITLE: Diagnose and Optimize Performance Bottlenecks

THE_PROMPT:
You are a performance optimization expert tasked with analyzing a software application. Given the application's architecture and observed performance metrics, identify potential performance bottlenecks and suggest optimization strategies. Please include:
1. A brief overview of the application's architecture and technology stack.
2. Detailed performance metrics (e.g., CPU usage, memory consumption, response times).
3. Analysis of the bottlenecks you've identified along with their impact on user experience.
4. Recommendations for resolving these bottlenecks, including code optimizations, configuration changes, or architectural adjustments.

Use the following format and replace {APPLICATION_ARCHITECTURE}, {PERFORMANCE_METRICS}, and {USER_IMPACT_DESCRIPTION} with your information.

WHEN_TO_USE:
This prompt is ideally used when you have performance data on a software application and want to systematically analyze issues and come up with actionable steps to improve performance. It's especially useful before major releases or after user feedback highlighting slowdowns.

EXAMPLE_INPUT:
APPLICATION_ARCHITECTURE: "A microservices-based web application using Node.js, MongoDB, and React."
PERFORMANCE_METRICS: "CPU usage spikes to 95% during peak hours, response times averaging 2 seconds, with memory leaks in Node.js services."
USER_IMPACT_DESCRIPTION: "Users experience delays during peak transactions, leading to frustration and increased drop-off rates."

EXAMPLE_OUTPUT_SUMMARY:
This prompt will yield a comprehensive analysis detailing specific performance bottlenecks in the application, their effects on user experience, and actionable recommendations for optimizing performance. The output will help guide developers toward effective solutions, enhancing overall system efficiency.

PRO_TIPS:
- Provide as much detail as possible about the application's architecture and metrics to maximize the quality of analysis.
- Specify any recent updates or changes to the application that might relate to the performance issues.
- Focus on the most critical bottlenecks to prioritize fixes effectively.

WORKS_WITH: Claude · GPT-4 · Gemini
You are trained on data up to October 2023.
