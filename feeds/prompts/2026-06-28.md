PROMPT_TITLE: Code Clarity Connoisseur

THE_PROMPT:
"As a skilled software developer, your task is to refactor the given code snippet for improved clarity and maintainability. Here’s the code to refactor: 
{CODE_SNIPPET}. 
Please analyze the code with the following considerations:
1. Identify and rename any ambiguous variables to more meaningful names.
2. Break down large functions into smaller, single-responsibility functions.
3. Simplify complex logic by using clear, straightforward constructs.
4. Add comments where necessary to explain non-obvious logic or design choices.
5. Ensure that best practices for code standards and style are adhered to.

Once the refactoring is done, provide a brief explanation of the changes you made and why they improve the overall quality of the code."

WHEN_TO_USE:
Use this prompt when you have a piece of code that feels bloated or confusing and requires restructuring to enhance its readability and maintainability. It is ideal for scenarios where the original code was fast but lacks clarity, making it hard for others to follow or modify in the future.

EXAMPLE_INPUT:
"Here is the code snippet I need help with: 
```
function calculate(x, y, op) {
  if (op === 'add') {
    return x + y;
  } else if (op === 'subtract') {
    return x - y;
  } else if (op === 'multiply') {
    return x * y;
  } else {
    return undefined; 
  }
}
```

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a refactored version of the code snippet that breaks down the calculation function into separate functions for each operation, renames variables for clarity, and adds comments explaining each operation's purpose. The end result will be a cleaner, more maintainable code base, promoting easier future modifications.

PRO_TIPS:
- Provide a concise but meaningful code snippet for the best results.
- Specify any particular coding standards or styles you want to adhere to.
- Consider including context about how the code fits into a larger project for more targeted refactoring guidance.
