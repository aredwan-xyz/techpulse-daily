PROMPT_TITLE: Diagnose Performance Bottlenecks

THE_PROMPT:
You are an expert performance analyst tasked with identifying and solving performance bottlenecks in a software application. Please analyze the provided performance metrics and user feedback to identify potential bottlenecks. Use the following structure:

1. **Overview**: Briefly summarize the application type and the context of the performance issues.
2. **Metrics Analysis**: Identify key performance metrics such as response time, load time, CPU usage, memory usage, and any other relevant data. Highlight any abnormal figures.
3. **User Feedback**: Summarize relevant feedback from users that points to performance issues. Note patterns or frequently mentioned problems.
4. **Bottlenecks Identified**: List the top three performance bottlenecks, providing a brief explanation for each.
5. **Recommendations**: Offer actionable solutions for each bottleneck identified. Consider strategies such as code optimization, better resource management, or architectural changes.

Fill in the details using the placeholders: 
- Application Type: {APPLICATION_TYPE}
- Performance Metrics: {PERFORMANCE_METRICS}
- User Feedback: {USER_FEEDBACK}

WHEN_TO_USE:
Use this prompt when you need to systematically identify and resolve performance issues in a software application. It's ideal for developers, product managers, or performance engineers looking to enhance user experience by addressing slowdowns effectively.

EXAMPLE_INPUT:
Application Type: E-commerce web app
Performance Metrics: Response time: 5s, CPU usage: 85%, Memory usage: 70%
User Feedback: Users report slow checkout process and difficulty loading product images

EXAMPLE_OUTPUT_SUMMARY:
The output includes a clearly structured analysis that identifies three key performance bottlenecks in the application and offers practical recommendations for improvement. This guidance can help streamline application performance and enhance user satisfaction.

PRO_TIPS:
- Ensure your performance metrics are as detailed as possible to gain a comprehensive analysis.
- Prioritize addressing user feedback, as it often reveals the most critical bottlenecks.
- Re-evaluate the application after implementing recommendations to measure improvements.
