PROMPT_TITLE: Clarity in Technical Documentation

THE_PROMPT:
You are a technical writing expert tasked with creating clear, concise, and structured documentation for a technical product. Please fill in the following template with the necessary details:

**Title:** {Title of the Document}
**Introduction:** {Brief overview of what the documentation covers and its purpose}
**Audience:** {Describe the target audience who will use this documentation}
**Prerequisites:** {List any knowledge or tools the reader should have before using the documentation}
**Sections:**
1. **Overview:** {Provide a detailed summary of the product or feature}
2. **Setup Instructions:** {Step-by-step instructions for getting started with the product}
3. **Usage:** {Detailed descriptions on how to use the product, including examples}
4. **Troubleshooting:** {Common issues and their fixes}
5. **FAQs:** {Frequently Asked Questions related to the product}
6. **References:** {Links to additional resources or documentation}
**Conclusion:** {Summarize the key takeaways and next steps for users}

WHEN_TO_USE:
Use this prompt template when you need to develop technical documentation for software, tools, or processes that require clear instructions and structured information. Ideal for product launches or updates where clarity is crucial for user understanding.

EXAMPLE_INPUT:
Title: "Installation Guide for SuperApp"
Introduction: "This document provides detailed instructions on how to install SuperApp on various operating systems."
Audience: "Software developers and system admins."
Prerequisites: "Basic understanding of command line interfaces."
Overview: "SuperApp is a cross-platform tool designed for..."
Setup Instructions: "1. Download the installer from the official website. 2. Run the installation wizard."
Usage: "To start using SuperApp, open your terminal and type..."
Troubleshooting: "If you encounter error code 101, ensure..."
FAQs: "Q: Can SuperApp run on Windows? A: Yes, it is compatible with Windows 10 and later."
References: "For more information, visit our official documentation page."
Conclusion: "Now you are ready to start using SuperApp efficiently."

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a structured and comprehensive technical document that includes all necessary sections, making it easy for the reader to understand and utilize the product effectively. Each section is clearly defined, ensuring no important details are overlooked.

PRO_TIPS:
- Use bullet points for clarity in longer sections.
- Consider including screenshots or diagrams to visually enhance understanding.
- Regularly update the documentation with user feedback to maintain accuracy and usability.
