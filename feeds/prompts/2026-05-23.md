PROMPT_TITLE: Clear Technical Documentation Creator

THE_PROMPT:
You are a technical writer tasked with creating clear and concise documentation. Write a section titled "{DOCUMENT_SECTION_TITLE}" that explains the following topic: "{TECHNICAL_TOPIC}". Include the purpose of the topic, key features, step-by-step instructions for usage, potential pitfalls, and any troubleshooting tips. Structure the documentation using clear headings and bullet points where appropriate. Aim for clarity and simplicity while using technical language that is easily understandable to your target audience.

WHEN_TO_USE:
Use this prompt whenever you need to generate comprehensive technical documentation that is user-friendly. Whether you are documenting software features, hardware installations, or procedural guidelines, it’s ideal for ensuring clarity and usability for readers.

EXAMPLE_INPUT:
DOCUMENT_SECTION_TITLE: "How to Configure the XYZ Router"
TECHNICAL_TOPIC: "Configuring the XYZ Router for Home Use"

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces structured technical documentation that clearly outlines the process of configuring the XYZ router. It includes a clear purpose, detailed features, step-by-step instructions, common issues, and troubleshooting advice, making it accessible for users of all skill levels.

PRO_TIPS:
- Always consider your target audience's technical proficiency to tailor the language and depth of information.
- Use visuals such as diagrams or screenshots if applicable to enhance understanding.
- Keep paragraphs short and use bullet points for lists to improve readability.
