PROMPT_TITLE: Refactor for Clarity and Maintainability

THE_PROMPT:
You are a software engineer tasked with refactoring code to enhance its clarity and maintainability. Given the following code snippet: 
```
{CODE_SNIPPET}
```
Please analyze the code and identify areas for improvement, including but not limited to variable naming, structure, and commenting. Provide a refactored version that adheres to best coding practices, ensuring the code is easier to understand and maintain. Also, include a brief explanation of the changes you've made and why they improve the code's clarity and maintainability.

WHEN_TO_USE:
Use this prompt when you have existing code that feels convoluted or challenging to maintain. It’s particularly useful in collaborative environments where improving code readability can benefit the entire team.

EXAMPLE_INPUT:
Here is a JavaScript function that calculates the factorial of a number, but it's hard to read:
```javascript
function f(n) {
    if (n === 0) return 1;
    return n * f(n - 1);
}
```

EXAMPLE_OUTPUT_SUMMARY:
The output will provide a more readable version of the original code, complete with improved variable names, clearer structure, and additional comments. It will also explain how these changes enhance the code's readability and maintainability.

PRO_TIPS:
- Provide a code snippet that is representative of the complexity you wish to refactor.
- Be specific about aspects you find unclear or maintainable, as this can guide the refactoring process.
- Consider including context about the project's purpose for tailored advice on maintainability.
