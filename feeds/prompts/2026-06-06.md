PROMPT_TITLE: Craft Comprehensive Technical Documentation

THE_PROMPT:
Create clear technical documentation for {PRODUCT_NAME}. This documentation should include the following sections: 
1. Overview: Provide a brief description of {PRODUCT_NAME} and its purpose.
2. Features: List and explain key features of {PRODUCT_NAME}, using bullet points for clarity.
3. Installation Instructions: Detail step-by-step instructions on how to install {PRODUCT_NAME} with prerequisites listed.
4. Usage Guide: Offer guidelines on how to effectively use {PRODUCT_NAME}, including examples of common tasks.
5. Troubleshooting: Identify common issues users may encounter and offer practical solutions or tips.
6. Frequently Asked Questions: List responses to common queries related to {PRODUCT_NAME}.
Ensure the language is straightforward and jargon is minimized to enhance user understanding.

WHEN_TO_USE:
Use this prompt when you need to create technical documentation for software, tools, or products. It’s particularly useful for helping users understand complexities in an organized and approachable manner.

EXAMPLE_INPUT:
PRODUCT_NAME: MyApp

EXAMPLE_OUTPUT_SUMMARY:
This prompt reliably produces a well-structured technical document that clearly outlines the function and usage of MyApp. It includes a comprehensive overview, detailed installation and usage instructions, troubleshooting tips, and answers to frequently asked questions — making it easy for users to understand and utilize the application effectively.

PRO_TIPS:
- Ensure that the product name and other placeholders are filled in with accurate details for relevancy.
- Use simple language and clear examples to cater to users with varied technical expertise.
- Review and revise the output to adapt it to specific audiences, ensuring clarity and precision in the documentation.
