TITLE: Balanced Array
DIFFICULTY: Medium
TOPIC: Two Pointers
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given an array of integers where each integer represents the score of a player in a series of games. The challenge is to determine if it is possible to partition the array into two subarrays such that the sum of the scores in each subarray is equal. The scores can be split at any position, and you must provide the length of the left subarray and the right subarray if a valid partition exists. If no partition is possible, return [-1, -1].

EXAMPLES:
Input: [1, 2, 3, 3]
Output: [3, 1]
Explanation: The array can be split into [1, 2] and [3, 3] both of which sum to 3.

Input: [5, 2, 1, 4]
Output: [-1, -1]
Explanation: There is no way to split the array into two subarrays with equal sum.

CONSTRAINTS:
- The array length will be between 2 and 1000.
- The integers in the array will range from -10^4 to 10^4.

HINT:
Consider using a prefix sum approach to simplify the problem of checking for equal sums in the partitions.

SOLUTION_PYTHON:
```python
def balanced_array(arr):
    total_sum = sum(arr)
    left_sum = 0
    
    for i in range(len(arr) - 1): # All possible split points except the last
        left_sum += arr[i]
        right_sum = total_sum - left_sum
        
        if left_sum == right_sum:
            return [i + 1, len(arr) - (i + 1)]  # Return lengths of left and right subarrays
    
    return [-1, -1]  # No valid partition found

# Time complexity: O(n)
# Space complexity: O(1)
```

SOLUTION_JS:
```javascript
function balancedArray(arr) {
    const totalSum = arr.reduce((a, b) => a + b, 0);
    let leftSum = 0;

    for (let i = 0; i < arr.length - 1; i++) { // All possible split points except the last
        leftSum += arr[i];
        const rightSum = totalSum - leftSum;

        if (leftSum === rightSum) {
            return [i + 1, arr.length - (i + 1)];  // Return lengths of left and right subarrays
        }
    }

    return [-1, -1];  // No valid partition found
}

// Time complexity: O(n)
// Space complexity: O(1)
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The solution uses a prefix sum approach to keep track of the sum of the elements in the left subarray while iterating through the array. By calculating the right subarray sum as the total sum minus the current left sum, we can efficiently check for equal partition sums at each split point. This method is optimal because it avoids the need for nested loops, resulting in a linear-time solution.

FOLLOW_UP:
Modify the function to return the actual subarrays instead of their lengths if a valid partition exists.
