TITLE: Path to Itineraries
DIFFICULTY: Medium
TOPIC: Recursion & Backtracking
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given a list of flight connections, where each connection is represented as a pair of strings, with the first string being the departure airport and the second string being the arrival airport. Your task is to find all possible itineraries that can be constructed using these flights, visiting each flight exactly once. An itinerary is defined as an ordered list of airport codes, and your results should be sorted lexicographically. If there are no possible itineraries, return an empty list.

EXAMPLES:
Input: 
```
flights = [["JFK", "SFO"], ["JFK", "ATL"], ["SFO", "ATL"], ["ATL", "JFK"], ["ATL", "SFO"]]
```
Output: 
```
["JFK", "ATL", "JFK", "SFO", "ATL", "SFO", "JFK", "SFO", "ATL"]
```
Explanation: You can take the route where you start from JFK, followed by flying to ATL, returning to JFK, then flying to SFO, stopping at ATL before finally landing at SFO.

Input: 
```
flights = [["LAX", "JFK"], ["JFK", "LAX"]]
```
Output: 
```
["JFK", "LAX", "JFK"]
```
Explanation: You can start from JFK, fly to LAX, then return back to JFK.

CONSTRAINTS:
- The flight connections list will have at least 1 and at most 20 connections.
- Each airport code will be a string of at most 3 uppercase letters.

HINT:
Consider using depth-first search (DFS) to explore each possible path while keeping track of visited flights.

SOLUTION_PYTHON:
```python
from collections import defaultdict

def find_itineraries(flights):
    def backtrack(path):
        if len(path) == len(flights) + 1:
            result.append(path[:])
            return
        for i in range(len(flights)):
            if not visited[i]:
                visited[i] = True
                dep, arr = flights[i]
                backtrack(path + [arr])
                visited[i] = False

    # Create a list of flight connections
    result = []
    visited = [False] * len(flights)
    for dep, arr in flights:
        visited.append(False)
    
    # Start from the JFK airport
    backtrack(["JFK"])
    return sorted(["->".join(it) for it in result])

# Time complexity: O(N!), where N is the number of flights, given the backtracking nature.
# Space complexity: O(N) for the recursion stack and path storage.
```

SOLUTION_JS:
```javascript
function findItineraries(flights) {
    let result = [];
    let visited = new Array(flights.length).fill(false);

    function backtrack(path) {
        if (path.length === flights.length + 1) {
            result.push(path.slice());
            return;
        }
        for (let i = 0; i < flights.length; i++) {
            if (!visited[i]) {
                visited[i] = true;
                path.push(flights[i][1]);
                backtrack(path);
                path.pop();
                visited[i] = false;
            }
        }
    }

    // Start the itinerary from JFK
    backtrack(['JFK']);
    return result.sort();
}

// Time complexity: O(N!), where N is the number of flights.
// Space complexity: O(N) for the recursion stack and path storage.
```

TIME_COMPLEXITY: O(N!)
SPACE_COMPLEXITY: O(N)

EXPLANATION:
The algorithm employs a backtracking technique where it explores every possible flight path starting from "JFK". By maintaining a visited list to track used flights, it prevents revisiting any flight. The DFS nature of the approach ensures all potential itineraries are generated, and the results are sorted for output. Given the nature of permutations, the time complexity is factorial in relation to the number of flights.
