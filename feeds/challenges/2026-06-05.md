TITLE: Merging Sorted Events
DIFFICULTY: Medium
TOPIC: Two Pointers
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given two lists of events, where each event is represented by a timestamp in the form of a string. Both lists are sorted in non-decreasing order. Your task is to merge these two lists into one sorted list while maintaining the order of events. After merging, return a list of unique events, which means if two events have the same timestamp, only one should remain in the final list.

For example, you might be tracking the start and end times of meetings. Each list of events may represent different events, and you need to combine them efficiently.

EXAMPLES:
Input: 
`events1 = ["2026-06-01T10:00:00", "2026-06-02T14:00:00", "2026-06-04T09:00:00"]`
`events2 = ["2026-06-01T10:00:00", "2026-06-03T11:00:00", "2026-06-05T15:30:00"]`

Output: 
`["2026-06-01T10:00:00", "2026-06-02T14:00:00", "2026-06-03T11:00:00", "2026-06-04T09:00:00", "2026-06-05T15:30:00"]`
Explanation: The final output contains unique timestamps from both lists in sorted order.

Input: 
`events1 = ["2026-06-01T10:00:00", "2026-06-01T11:00:00"]`
`events2 = ["2026-06-02T10:00:00"]`

Output: 
`["2026-06-01T10:00:00", "2026-06-01T11:00:00", "2026-06-02T10:00:00"]`
Explanation: Merging the events, we get a List of unique events in sorted order.

CONSTRAINTS:
- Both event lists can have up to 100,000 timestamps.
- Each timestamp is a valid ISO 8601 format date-time string.

HINT:
Consider using the two pointers technique to iterate through both lists simultaneously while taking care of duplicates.

SOLUTION_PYTHON:
```python
def merge_sorted_events(events1, events2):
    # Initialize pointers for both lists
    i, j = 0, 0
    merged_events = []
    
    # Loop until we run out of elements in either list
    while i < len(events1) and j < len(events2):
        if events1[i] < events2[j]:
            if not merged_events or merged_events[-1] != events1[i]:  # Check for uniqueness
                merged_events.append(events1[i])
            i += 1
        elif events1[i] > events2[j]:
            if not merged_events or merged_events[-1] != events2[j]:
                merged_events.append(events2[j])
            j += 1
        else:
            if not merged_events or merged_events[-1] != events1[i]:  # Events are equal
                merged_events.append(events1[i])
            i += 1
            j += 1
    
    # Add remaining events from events1
    while i < len(events1):
        if not merged_events or merged_events[-1] != events1[i]:
            merged_events.append(events1[i])
        i += 1
    
    # Add remaining events from events2
    while j < len(events2):
        if not merged_events or merged_events[-1] != events2[j]:
            merged_events.append(events2[j])
        j += 1
    
    return merged_events

# Time Complexity: O(n + m)
# Space Complexity: O(n + m) for the merged list
```

```javascript
function mergeSortedEvents(events1, events2) {
    let i = 0, j = 0;
    const mergedEvents = [];

    while (i < events1.length && j < events2.length) {
        if (events1[i] < events2[j]) {
            if (mergedEvents[mergedEvents.length - 1] !== events1[i]) {
                mergedEvents.push(events1[i]);
            }
            i++;
        } else if (events1[i] > events2[j]) {
            if (mergedEvents[mergedEvents.length - 1] !== events2[j]) {
                mergedEvents.push(events2[j]);
            }
            j++;
        } else {
            if (mergedEvents[mergedEvents.length - 1] !== events1[i]) {
                mergedEvents.push(events1[i]);
            }
            i++;
            j++;
        }
    }

    while (i < events1.length) {
        if (mergedEvents[mergedEvents.length - 1] !== events1[i]) {
            mergedEvents.push(events1[i]);
        }
        i++;
    }

    while (j < events2.length) {
        if (mergedEvents[mergedEvents.length - 1] !== events2[j]) {
            mergedEvents.push(events2[j]);
        }
        j++;
    }

    return mergedEvents;
}

// Time Complexity: O(n + m)
// Space Complexity: O(n + m) for the merged list
```

TIME_COMPLEXITY: O(n + m)
SPACE_COMPLEXITY: O(n + m)

EXPLANATION:
The two pointers technique allows us to efficiently merge the two sorted lists in a single pass. We compare the timestamps at the current index of both lists, adding the smaller one to the result while ensuring we avoid duplicates. This results in a time complexity of O(n + m), which is optimal for merging two sorted lists.
