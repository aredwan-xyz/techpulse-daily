TITLE: Tree Connectivity Checker  
DIFFICULTY: Hard  
TOPIC: Trees & Graphs  
ESTIMATED_TIME: 60 minutes  

PROBLEM:  
You are a software engineer building a networking application where each device on the network can be represented as a node in a tree structure. The devices on this tree can communicate with each other directly or indirectly through their parent-child relationships. Your task is to determine if all devices can communicate with each other given an initial set of connections.

Given a list of parent-child relationships that define the tree and an initial set of active devices, return a boolean indicating if all devices in the tree can reach each other through the established connections.

EXAMPLES:  
Input:  
```   
connections = [(1, 2), (1, 3), (2, 4), (2, 5)]  
active_devices = [1, 3, 4]
```  
Output:  
```
False
```  
Explanation:  
Device 1 can communicate with devices 2 and 3, but device 4 cannot communicate with device 3 since there’s no direct path.

Input:  
```  
connections = [(1, 2), (1, 3), (3, 4), (3, 5)]  
active_devices = [1, 2, 4, 5]
```  
Output:  
```
True
```  
Explanation:  
Device 1 can reach devices 2, 3 (which connects to 4 and 5) directly and indirectly, thus all devices can communicate.

CONSTRAINTS:  
- The number of connections (n) will be in the range [1, 10^5].  
- Each device's ID will be a unique integer in the range [1, 10^5].  

HINT:  
Consider using Depth-First Search (DFS) or Breadth-First Search (BFS) to traverse the tree starting from the initial active devices.

SOLUTION_PYTHON:
```python
def can_communicate(connections, active_devices):
    from collections import defaultdict, deque

    # Build adjacency list for the tree
    graph = defaultdict(list)
    
    for parent, child in connections:
        graph[parent].append(child)
        graph[child].append(parent)  # since it’s undirected
    
    # Start BFS/DFS from each active device, if they are interconnected
    visited = set()
    
    def bfs(start):
        queue = deque([start])
        while queue:
            node = queue.popleft()
            visited.add(node)
            for neighbor in graph[node]:
                if neighbor not in visited:
                    queue.append(neighbor)
    
    # Start BFS from the first active device
    if active_devices:
        bfs(active_devices[0])
    
    # Check if all active devices are visited
    return all(device in visited for device in active_devices)

# Example usage
connections = [(1, 2), (1, 3), (2, 4), (2, 5)]
active_devices = [1, 3, 4]
print(can_communicate(connections, active_devices))
```

SOLUTION_JS:
```javascript
function canCommunicate(connections, activeDevices) {
    const graph = new Map();

    // Build the adjacency list for the tree
    for (let [parent, child] of connections) {
        if (!graph.has(parent)) graph.set(parent, []);
        if (!graph.has(child)) graph.set(child, []);
        graph.get(parent).push(child);
        graph.get(child).push(parent); // as undirected
    }

    const visited = new Set();

    // BFS function to traverse the tree
    function bfs(start) {
        const queue = [start];
        while (queue.length > 0) {
            const node = queue.shift();
            visited.add(node);
            for (const neighbor of graph.get(node) || []) {
                if (!visited.has(neighbor)) {
                    queue.push(neighbor);
                }
            }
        }
    }

    // Start BFS from the first active device
    if (activeDevices.length > 0) {
        bfs(activeDevices[0]);
    }

    // Check if all active devices are visited
    return activeDevices.every(device => visited.has(device));
}

// Example usage
const connections = [[1, 2], [1, 3], [2, 4], [2, 5]];
const activeDevices = [1, 3, 4];
console.log(canCommunicate(connections, activeDevices));
```

TIME_COMPLEXITY: O(n)  
SPACE_COMPLEXITY: O(n)  

EXPLANATION:  
The approach uses a graph traversal technique, either BFS or DFS, to explore all reachable nodes in the tree starting from any active device. Given that we use a set to track visited nodes, the time and space complexity are linear relative to the number of connections. This approach ensures that we efficiently check whether all active devices can communicate with each other through the established paths.
