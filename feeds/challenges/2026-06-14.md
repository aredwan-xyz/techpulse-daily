TITLE: Maximizing Event Attendance  
DIFFICULTY: Medium  
TOPIC: Greedy Algorithms  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are responsible for organizing a series of events throughout a week. Each event has a start time and an end time, as well as a maximum capacity for attendees. Given a list of events with their respective start time, end time, and capacity, your goal is to maximize the total number of attendees that can be accommodated throughout the week. You can only attend one event at a time. Determine the maximum number of attendees that can be accommodated.

You are provided with a list of events, where each event is represented as a tuple (start_time, end_time, capacity). Start and end times are represented in a 24-hour format.

EXAMPLES:  
Input: [(9, 12, 50), (12, 14, 30), (14, 16, 20), (10, 13, 40), (15, 17, 60)]  
Output: 130  
Explanation: You can attend the events from (9, 12) with 50 capacities, and then (12, 14) with 30 capacities, and finally (15, 17) with 60 capacities to maximize attendance to 50 + 30 + 60 = 140.

Input: [(8, 10, 25), (10, 12, 15), (12, 14, 20), (14, 16, 10)]  
Output: 70  
Explanation: Attend events in sequence: (8, 10), (10, 12), (12, 14), resulting in a total attendance of 25 + 15 + 20 = 70.

CONSTRAINTS:  
- Events list will have at most 100 events.  
- Start time and end time will be in the range of 0 to 24 hours.  
- Capacity will be a positive integer not exceeding 100.  

HINT:  
Consider sorting the events based on their end time to help choose the optimal events that can be attended sequentially.

SOLUTION_PYTHON:
```python
def max_attendance(events):
    # Sort events by their end time
    events.sort(key=lambda x: x[1])
    
    # Initialize variables
    total_attendees = 0
    last_end_time = -1
    
    # Iterate through sorted events
    for start, end, capacity in events:
        if start >= last_end_time:  # if the event can be attended
            total_attendees += capacity  # add capacity of this event
            last_end_time = end  # update last end time to current event's end
    
    return total_attendees

# Example usage
events = [(9, 12, 50), (12, 14, 30), (14, 16, 20), (10, 13, 40), (15, 17, 60)]
print(max_attendance(events))  # Output: 130
```

SOLUTION_JS:
```javascript
function maxAttendance(events) {
    // Sort events by their end time
    events.sort((a, b) => a[1] - b[1]);

    let totalAttendees = 0;
    let lastEndTime = -1;

    // Iterate through sorted events
    for (const [start, end, capacity] of events) {
        if (start >= lastEndTime) { // if the event can be attended
            totalAttendees += capacity; // add capacity of this event
            lastEndTime = end; // update last end time to current event's end
        }
    }

    return totalAttendees;
}

// Example usage
const events = [[9, 12, 50], [12, 14, 30], [14, 16, 20], [10, 13, 40], [15, 17, 60]];
console.log(maxAttendance(events));  // Output: 130
```

TIME_COMPLEXITY: O(n log n)  
SPACE_COMPLEXITY: O(1)  

EXPLANATION:  
The approach is based on sorting the events by their end time and then using a greedy method to iterate through the sorted events while keeping track of the last attended event's end time. By only adding the capacity of events that can be attended sequentially without overlap, we ensure the maximum number of attendees is achieved. This method leverages the property that attending events that finish earlier allows for more events to be attended afterward.
