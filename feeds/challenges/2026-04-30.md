TITLE: Word Search Path Finder
DIFFICULTY: Medium
TOPIC: Recursion & Backtracking
ESTIMATED_TIME: 30 minutes

PROBLEM:
Given a 2D grid of characters and a word, your task is to find if the word can be formed by a sequence of adjacent cells in the grid, where adjacent cells are horizontally or vertically neighboring. Once you find the word in the grid, return the path (as a list of coordinates) that traces the word. If the word cannot be found, return an empty list. 

Adjacent cells can only be used once during the search.

EXAMPLES:
Input: 
grid = [['A', 'B', 'C', 'E'],
        ['S', 'F', 'C', 'S'],
        ['A', 'D', 'E', 'E']]
word = "ABCCED"
Output: [(0, 0), (0, 1), (0, 2), (1, 2), (2, 2)]

Explanation: The word "ABCCED" can be found in the grid with the path traced as above.

Input: 
grid = [['A', 'B', 'C', 'E'],
        ['S', 'F', 'C', 'S'],
        ['A', 'D', 'E', 'E']]
word = "SEE"
Output: [(2, 1), (1, 1), (1, 0)]

Input:
grid = [['A', 'B', 'C', 'E'],
        ['S', 'F', 'C', 'S'],
        ['A', 'D', 'E', 'E']]
word = "ABCB"
Output: []

CONSTRAINTS:
- The grid will have dimensions of at most 100 x 100.
- The word length will be at most 15 characters.

HINT:
Try using a depth-first search (DFS) approach with backtracking to explore potential paths for forming the word.

SOLUTION_PYTHON:
```python
def find_word_path(grid, word):
    if not grid or not grid[0] or not word:
        return []
    
    rows, cols = len(grid), len(grid[0])
    path = []

    # Helper function to perform DFS
    def dfs(r, c, index):
        if index == len(word):
            return True  # We've found the whole word
        
        if r < 0 or c < 0 or r >= rows or c >= cols or grid[r][c] != word[index]:
            return False  # Out of bounds or character doesn't match
        
        # Path can be traced at (r, c)
        path.append((r, c))
        temp = grid[r][c]  # Store the current character
        grid[r][c] = '#'    # Mark this cell as visited

        # Explore adjacent cells
        found = (dfs(r + 1, c, index + 1) or 
                 dfs(r - 1, c, index + 1) or 
                 dfs(r, c + 1, index + 1) or 
                 dfs(r, c - 1, index + 1))

        grid[r][c] = temp  # Restore the character
        if not found:  # If not found, backtrack
            path.pop()
        return found

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == word[0]:  # Start searching from potential starting cells
                if dfs(r, c, 0):
                    return path

    return []  # Word not found
```

SOLUTION_JS:
```javascript
function findWordPath(grid, word) {
    if (!grid || !grid.length || !word) return [];

    const rows = grid.length;
    const cols = grid[0].length;
    const path = [];

    function dfs(r, c, index) {
        if (index === word.length) return true;

        if (r < 0 || c < 0 || r >= rows || c >= cols || grid[r][c] !== word[index]) {
            return false;
        }

        path.push([r, c]);
        const temp = grid[r][c];
        grid[r][c] = '#'; // Mark as visited

        const found = (
            dfs(r + 1, c, index + 1) ||
            dfs(r - 1, c, index + 1) ||
            dfs(r, c + 1, index + 1) ||
            dfs(r, c - 1, index + 1)
        );

        grid[r][c] = temp; // Restore character
        if (!found) path.pop(); // Backtrack if not found

        return found;
    }

    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            if (grid[r][c] === word[0]) {
                if (dfs(r, c, 0)) return path;
            }
        }
    }

    return []; // Word not found
}
```

TIME_COMPLEXITY: O(N * M * 4^L) where N is the number of rows, M is the number of columns, and L is the length of the word.
SPACE_COMPLEXITY: O(L) for the recursion stack.

EXPLANATION:
The approach employs depth-first search (DFS) with backtracking to explore every possible path in the grid for forming the goal word. Starting from every grid cell that matches the first character of the word, it recursively checks adjacent cells for matching subsequent characters while marking cells as visited, preventing the use of already considered cells within the current search path. This backtracking method ensures that once a character doesn't lead to completing the word, the path can be retraced and other possibilities explored.

FOLLOW_UP:
Consider how you could modify the algorithm to return all possible paths found for the word in the grid instead of just one.
