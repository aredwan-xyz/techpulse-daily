TITLE: Treasure Map Exploration  
DIFFICULTY: Medium  
TOPIC: Recursion & Backtracking  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are a treasure hunter navigating a grid representing a map. The grid consists of cells marked with different characters:  
- 'S' for Start (your initial position)  
- 'T' for Treasure (where you want to go)  
- 'X' for obstacles (you cannot move here)  
- '.' for open paths (you can move here)  

Your goal is to find if there exists a valid path from the Start ('S') to the Treasure ('T') following the open paths. You can move in four directions: up, down, left, and right.  

Write a recursive backtracking function that returns `True` if a valid path to the treasure exists, and `False` otherwise.  

EXAMPLES:  
Input:  
```
[
 ['S', '.', 'X', 'X'],
 ['.', 'X', '.', 'T'],
 ['.', '.', 'X', 'X'],
 ['X', '.', '.', '.']
]
```  
Output: `True`  
Explanation: You can move from 'S' right to (0,1), then down to (1,1), then down to (2,1), and then right to the treasure at (1,3).  

Input:  
```
[
 ['S', 'X', 'X', 'T'],
 ['.', 'X', '.', '.'],
 ['X', '.', 'X', 'X']
]
```  
Output: `False`  
Explanation: There are no valid paths leading from 'S' to 'T' due to blocks in between them.  

CONSTRAINTS:  
- The grid size will be at most 20x20.  
- There will be exactly one Start ('S') and one Treasure ('T') in the grid.  

HINT:  
Consider marking the cells you've visited in order to avoid cycles when searching for the treasure.  

SOLUTION_PYTHON:  
```python
def canReachTreasure(grid):
    rows, cols = len(grid), len(grid[0])
    start = None
    
    # Find the starting point
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 'S':
                start = (r, c)
                break
        if start:
            break
    
    def backtrack(x, y):
        # Check bounds and obstacles
        if x < 0 or x >= rows or y < 0 or y >= cols or grid[x][y] == 'X':
            return False
        # Check if we've found the treasure
        if grid[x][y] == 'T':
            return True
        # Mark as visited
        temp = grid[x][y]
        grid[x][y] = 'X'  # Mark as obstacle to avoid revisiting
        # Explore all four directions
        found = (backtrack(x + 1, y) or  # down
                 backtrack(x - 1, y) or  # up
                 backtrack(x, y + 1) or  # right
                 backtrack(x, y - 1))    # left
        # Unmark this cell to restore the grid for other paths
        grid[x][y] = temp
        return found
    
    return backtrack(*start)

# Time complexity: O(N*M) where N is the number of rows and M is the number of columns
# Space complexity: O(N*M) for the recursion stack (in the worst case)
```

SOLUTION_JS:  
```javascript
function canReachTreasure(grid) {
    const rows = grid.length;
    const cols = grid[0].length;
    let start = null;
    
    // Find the starting point
    for (let r = 0; r < rows; r++) {
        for (let c = 0; c < cols; c++) {
            if (grid[r][c] === 'S') {
                start = [r, c];
                break;
            }
        }
        if (start) break;
    }
    
    function backtrack(x, y) {
        // Check bounds and obstacles
        if (x < 0 || x >= rows || y < 0 || y >= cols || grid[x][y] === 'X') return false;
        // Check if we've found the treasure
        if (grid[x][y] === 'T') return true;
        
        // Mark as visited
        const temp = grid[x][y];
        grid[x][y] = 'X';  // Mark as obstacle to avoid revisiting
        
        // Explore all four directions
        const found = (backtrack(x + 1, y) ||  // down
                       backtrack(x - 1, y) ||  // up
                       backtrack(x, y + 1) ||  // right
                       backtrack(x, y - 1));   // left
        
        // Unmark this cell to restore the grid for other paths
        grid[x][y] = temp;
        return found;
    }
    
    return backtrack(start[0], start[1]);
}

// Time complexity: O(N*M) where N is the number of rows and M is the number of columns
// Space complexity: O(N*M) for the recursion stack (in the worst case)
```

TIME_COMPLEXITY: O(N*M)  
SPACE_COMPLEXITY: O(N*M)  

EXPLANATION:  
The approach utilizes a recursive backtracking technique to explore all possible paths from the Start cell to the Treasure cell. The function marks cells as visited to prevent cycles and infinite loops. Upon reaching the Treasure or any obstacle, the recursion will appropriately return `True` or `False` respectively. This method ensures that each cell is only explored once, leading to a time complexity proportional to the total number of cells in the grid.
