TITLE: Optimal Task Scheduling  
DIFFICULTY: Medium  
TOPIC: Greedy Algorithms  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are a project manager with a list of tasks that need to be completed within different time intervals. Each task has a start time, an end time, and a priority level indicated by a number. Your objective is to select a subset of tasks that can be completed without overlapping, while maximizing the total priority of the chosen tasks. You can only work on one task at a time. Each selected task should have a start time less than or equal to its end time.

Write a function that receives a list of tasks (each represented as a tuple containing the start time, end time, and priority) and returns the maximum sum of priorities of non-overlapping tasks you can schedule.

EXAMPLES:  
Input:  
```python
tasks = [(1, 3, 5), (2, 5, 6), (4, 7, 5), (6, 9, 4)]
```
Output:  
```python
11
```  
Explanation:  
You can select tasks (1, 3, 5) and (4, 7, 5). Their combined priority value is 5 + 6 = 11.

Input:  
```python
tasks = [(0, 1, 1), (1, 2, 2), (2, 3, 3), (3, 4, 4)]
```
Output:  
```python
10
```  
Explanation:  
You can choose all tasks since none overlap. Their combined priority is 1 + 2 + 3 + 4 = 10.

CONSTRAINTS:  
- Each task’s start and end time will be non-negative integers.  
- The number of tasks, n, will be at most 1000.  

HINT:  
Try sorting the tasks by their end times and then iteratively selecting the tasks based on the maximum priority.

SOLUTION_PYTHON:
```python
def max_priority(tasks):
    # Sort the tasks based on their end time
    tasks.sort(key=lambda x: x[1])
    
    n = len(tasks)
    dp = [0] * n
    
    for i in range(n):
        # Current task's priority
        current_priority = tasks[i][2]
        
        # Include the current task
        dp[i] = current_priority
        
        # Find the last non-overlapping task
        for j in range(i - 1, -1, -1):
            if tasks[j][1] <= tasks[i][0]:
                dp[i] += dp[j]
                break
        
        # Maintain the maximum priority sum
        if i > 0:
            dp[i] = max(dp[i], dp[i-1])
    
    return dp[-1]

# Example usage:
tasks = [(1, 3, 5), (2, 5, 6), (4, 7, 5), (6, 9, 4)]
print(max_priority(tasks))  # Output: 11
```

SOLUTION_JS:
```javascript
function maxPriority(tasks) {
    // Sort tasks based on end time
    tasks.sort((a, b) => a[1] - b[1]);
    
    const n = tasks.length;
    const dp = new Array(n).fill(0);
    
    for (let i = 0; i < n; i++) {
        let currentPriority = tasks[i][2];
        dp[i] = currentPriority;
        
        for (let j = i - 1; j >= 0; j--) {
            if (tasks[j][1] <= tasks[i][0]) {
                dp[i] += dp[j];
                break;
            }
        }
        
        if (i > 0) {
            dp[i] = Math.max(dp[i], dp[i-1]);
        }
    }
    
    return dp[n - 1];
}

// Example usage:
const tasks = [[1, 3, 5], [2, 5, 6], [4, 7, 5], [6, 9, 4]];
console.log(maxPriority(tasks)); // Output: 11
```

TIME_COMPLEXITY: O(n log n)  
SPACE_COMPLEXITY: O(n)  

EXPLANATION:  
The optimal approach is to use dynamic programming by first sorting the task list based on their ending times. For each task, we maintain the maximum priority achievable either by selecting or skipping that task by filling a `dp` array. By doing this, you ensure that you always stick to non-overlapping tasks while maximizing priority.
