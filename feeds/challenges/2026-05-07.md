TITLE: Inventory Tracker
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are building an inventory tracking system for a small warehouse. Each item in the warehouse has a unique ID and a quantity indicating how many units of that item are available. Your task is to create a class `Inventory` that supports the following operations:

1. `add_item(item_id: str, quantity: int)`: Add a specified quantity of the item with the given ID to the inventory. If the item already exists in the inventory, update its quantity.
2. `remove_item(item_id: str, quantity: int)`: Remove a specified quantity of the item with the given ID from the inventory. If the quantity to remove exceeds the available quantity, set the quantity to zero and remove the item from the inventory.
3. `get_inventory() -> List[Tuple[str, int]]`: Return the current inventory as a list of tuples, each containing the item ID and its quantity, sorted in ascending order by item ID.

EXAMPLES:
Input:
```python
inventory = Inventory()
inventory.add_item("item1", 10)
inventory.add_item("item2", 5)
inventory.remove_item("item1", 3)
output = inventory.get_inventory()
```
Output:
[("item1", 7), ("item2", 5)]
Explanation: "item1" has 7 remaining and "item2" has 5.

Input:
```python
inventory.add_item("item3", 20)
inventory.remove_item("item2", 10)
output = inventory.get_inventory()
```
Output:
[("item1", 7), ("item3", 20)]
Explanation: "item2" was removed since all units were taken out.

CONSTRAINTS:
- Item IDs are unique strings.
- The quantity added or removed will always be a non-negative integer.

HINT:
Consider using a dictionary to track the quantities of items easily and efficiently.

SOLUTION_PYTHON:
```python
class Inventory:
    def __init__(self):
        self.items = {}  # A dictionary to store item_id and its quantity.

    def add_item(self, item_id: str, quantity: int):
        # Add item quantity, if item exists update the quantity.
        if item_id in self.items:
            self.items[item_id] += quantity
        else:
            self.items[item_id] = quantity

    def remove_item(self, item_id: str, quantity: int):
        if item_id in self.items:
            if self.items[item_id] <= quantity:
                del self.items[item_id]  # Remove item if quantity is depleted.
            else:
                self.items[item_id] -= quantity

    def get_inventory(self):
        # Return sorted list of items by item_id.
        return sorted(self.items.items())
```

SOLUTION_JS:
```javascript
class Inventory {
    constructor() {
        this.items = {};  // Object to store item_id with its quantity.
    }

    add_item(item_id, quantity) {
        if (this.items[item_id]) {
            this.items[item_id] += quantity; // Update existing item's quantity.
        } else {
            this.items[item_id] = quantity; // Add new item to the inventory.
        }
    }

    remove_item(item_id, quantity) {
        if (this.items[item_id]) {
            if (this.items[item_id] <= quantity) {
                delete this.items[item_id]; // Remove item if depleted.
            } else {
                this.items[item_id] -= quantity; // Reduce item quantity.
            }
        }
    }

    get_inventory() {
        // Sort items by item_id and return as an array of arrays.
        return Object.entries(this.items).sort((a, b) => a[0].localeCompare(b[0]));
    }
}
```

TIME_COMPLEXITY: O(n log n) for sorting the inventory in `get_inventory()`, O(1) for add and remove operations.
SPACE_COMPLEXITY: O(n) where n is the number of unique items in the inventory.

EXPLANATION:
The `Inventory` class uses a hash map (dictionary in Python, object in JavaScript) to efficiently track the quantity of each item based on its unique ID. Adding or removing items is done in constant time on average, while generating the inventory list involves sorting based on item IDs. This approach is optimal for both the operations we need to support and the space-tracking of items.
