TITLE: Grocery Store Checkout
DIFFICULTY: Medium
TOPIC: Sorting & Searching
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are managing a grocery store and need to assist in checkout line optimization. Each customer has a list of items they intend to buy, represented as an array of integers, where each integer represents the price of an item. Your task is to determine the minimum number of checkout lines required so that no line exceeds a specified maximum total price (the maximum amount that can be checked out in one line) when all customers go through. Every customer can only join one line.

Implement the function `min_checkout_lines(customers: List[List[int]], max_price: int) -> int`, which returns the minimum number of checkout lines needed.

EXAMPLES:
Input: 
```python
min_checkout_lines([[20, 30, 50], [10, 20], [40], [15, 5, 10]], 60)
```
Output: 
```python
3
```
Explanation: The optimal distribution would be:
- Line 1: Customer 1 (20 + 30 = 50)
- Line 2: Customer 2 (10 + 20 = 30)
- Line 3: Customer 3 (40) and Customer 4 (15 + 5 + 10 = 30)

Input: 
```python
min_checkout_lines([[70], [90], [30, 40]], 100)
```
Output: 
```python
3
```
Explanation: Each customer's total exceeds the max price alone, so each customer requires a separate checkout line.

CONSTRAINTS:
- The number of customers will be between 1 and 1000.
- Each item price is a positive integer not exceeding 1000.

HINT:
Consider utilizing a greedy approach to minimize the number of lines.

SOLUTION_PYTHON:
```python
def min_checkout_lines(customers, max_price):
    lines = 0
    current_line_total = 0

    # Sort customers by their total prices in descending order
    customer_totals = sorted([sum(customer) for customer in customers], reverse=True)
    
    while customer_totals:
        current_line_total = 0
        i = 0
        
        # Fill the current line as much as possible without exceeding max_price
        while i < len(customer_totals):
            if current_line_total + customer_totals[i] <= max_price:
                current_line_total += customer_totals[i]
                customer_totals.pop(i)
            else:
                i += 1
        
        lines += 1  # Increment the number of lines used

    return lines

# Time Complexity: O(n log n) due to sorting and O(n) for line filling.
# Space Complexity: O(n) for storing total prices.
```

SOLUTION_JS:
```javascript
function minCheckoutLines(customers, maxPrice) {
    let lines = 0;

    // Calculate the total for each customer
    const customerTotals = customers.map(customer => customer.reduce((sum, price) => sum + price, 0));
    
    // Sort in descending order
    customerTotals.sort((a, b) => b - a);

    while (customerTotals.length) {
        let currentLineTotal = 0;

        // Fill the current checkout line
        for (let i = 0; i < customerTotals.length; i++) {
            if (currentLineTotal + customerTotals[i] <= maxPrice) {
                currentLineTotal += customerTotals[i];
                customerTotals.splice(i, 1);
                i--; // Adjust index due to removal
            }
        }

        lines++; // Increment the number of lines used
    }

    return lines;
}

// Time Complexity: O(n log n) due to sorting and O(n) for line filling.
// Space Complexity: O(n) for storing total prices.
```

TIME_COMPLEXITY: O(n log n)
SPACE_COMPLEXITY: O(n)

EXPLANATION:
The problem is approached by first calculating the total price for each customer and then sorting these totals in descending order. A greedy method is used to fill the checkout lines by trying to maximize the usage of the maximum price without exceeding it. By removing customers from the list as they are assigned to lines, we ensure efficient line allocation. This keeps the time complexity manageable, primarily dominated by the sorting step.
