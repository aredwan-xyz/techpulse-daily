TITLE: Social Media Group Suggestion
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are building a social media platform where users can create and join various groups. Each user has a unique identifier, and each group contains a list of user identifiers. Your task is to implement a function that suggests a group to a user based on the groups that their friends belong to. A friend is defined as a user that has the same identifier prefix with a given length. Your function should return a list of unique group identifiers that the user is not already a member of. 

Write a function `suggest_groups(user_id: str, friends: List[str], groups: Dict[str, List[str]], prefix_length: int) -> List[str]` where:
- `user_id` is the unique identifier of the user for whom suggestions are generated.
- `friends` is a list of user identifiers that represent the user's friends.
- `groups` is a dictionary where keys are group identifiers and values are lists of user identifiers representing the members of each group.
- `prefix_length` indicates the number of characters at the start of the user identifier to consider while finding friends.

Return a sorted list of group identifiers (as strings) that the user can be suggested, based on the groups their friends are a part of.

EXAMPLES:
Input: 
```python
suggest_groups("alice123", ["alice1", "alice2", "bob1"], {"group1": ["alice1", "charlie"], "group2": ["bob1", "dave"], "group3": ["alice123", "eve"]}, 5)
```
Output: 
```python
["group1"]
```
Explanation: User `alice123` has friends `alice1`, `alice2`, and `bob1`. They can be suggested `group1` as it's the only group that their friends are a part of that they do not belong to.

Input:
```python
suggest_groups("johnDoe", ["john1", "john2", "jane"], {"groupA": ["john1", "lucy"], "groupB": ["johnDoe", "mark"], "groupC": ["jane", "sarah"]}, 4)
```
Output: 
```python
["groupA", "groupC"]
```
Explanation: `johnDoe` can be suggested `groupA` and `groupC`, as they belong to friends' groups but are not members themselves.

CONSTRAINTS:
- The length of `user_id` is between 3 and 100 characters.
- The number of friends will not exceed 100.
- Each group can contain up to 100 users.

HINT:
Consider using a set to keep track of groups that have already been joined by the user.

SOLUTION_PYTHON:
```python
def suggest_groups(user_id: str, friends: List[str], groups: Dict[str, List[str]], prefix_length: int) -> List[str]:
    # Create a set for groups the user is already a member of
    user_groups = {group for group, members in groups.items() if user_id in members}
    
    # Create a set for friend user IDs based on the prefix
    friend_prefix = user_id[:prefix_length]
    friend_ids = {friend for friend in friends if friend.startswith(friend_prefix)}

    suggestions = set()

    # Iterate through each group to find suggestions
    for group, members in groups.items():
        # Check if any friend is a member of the current group and user is not
        if any(friend in members for friend in friend_ids) and group not in user_groups:
            suggestions.add(group)

    # Return sorted list of unique group IDs
    return sorted(suggestions)
```

SOLUTION_JS:
```javascript
function suggest_groups(user_id, friends, groups, prefix_length) {
    const userGroups = new Set();
    for (const [group, members] of Object.entries(groups)) {
        if (members.includes(user_id)) {
            userGroups.add(group);
        }
    }

    const friendPrefix = user_id.slice(0, prefix_length);
    const friendIds = friends.filter(friend => friend.startsWith(friendPrefix));

    const suggestions = new Set();

    for (const [group, members] of Object.entries(groups)) {
        if (friendIds.some(friend => members.includes(friend)) && !userGroups.has(group)) {
            suggestions.add(group);
        }
    }

    return Array.from(suggestions).sort();
}
```

TIME_COMPLEXITY: O(F + G*M)
SPACE_COMPLEXITY: O(G)

EXPLANATION:
In this solution, we first identify all groups that the user is already a member of by iterating through the groups dictionary. Then, we gather the user identifiers of friends who share the same prefix. Finally, we iterate over the groups to check if any friends are members while ensuring the user is not already in the group. The use of sets helps to maintain uniqueness and improve lookup times, making this approach efficient.
