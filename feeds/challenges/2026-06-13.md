TITLE: Order of Processing
DIFFICULTY: Hard
TOPIC: Stack & Queue
ESTIMATED_TIME: 45 minutes

PROBLEM:
You are responsible for processing long-running tasks in a system. Each task has a unique identifier (an integer), and tasks can depend on the completion of other tasks. You will receive a list of tasks in the form of pairs, where each pair [x, y] means task x must be completed before task y can begin. Your goal is to determine the order in which the tasks can be processed given these dependencies. If a circular dependency exists, return an empty list. 

Write a function `determine_processing_order(dependencies: List[Tuple[int, int]]) -> List[int]` to solve the problem.

EXAMPLES:
Input: `dependencies = [(1, 2), (2, 3), (3, 4)]`
Output: `[1, 2, 3, 4]`
Explanation: Task 1 must be completed before task 2, task 2 before task 3, and task 3 before task 4. Thus, a valid order is [1, 2, 3, 4].

Input: `dependencies = [(1, 2), (2, 3), (3, 1)]`
Output: `[]`
Explanation: There is a circular dependency among tasks 1, 2, and 3. Thus, it is impossible to determine a processing order.

CONSTRAINTS:
- Each task identifier is a unique integer between 1 and 10^5.
- The length of the dependencies list will not exceed 10^4 pairs.

HINT:
Use topological sorting to determine the linear order of tasks based on their dependencies.

SOLUTION_PYTHON:
```python
from collections import defaultdict, deque
from typing import List, Tuple

def determine_processing_order(dependencies: List[Tuple[int, int]]) -> List[int]:
    # Graph construction
    graph = defaultdict(list)
    in_degree = defaultdict(int)

    for x, y in dependencies:
        graph[x].append(y)
        in_degree[y] += 1
        if x not in in_degree:
            in_degree[x] = 0

    # Queue for tasks with no dependencies
    zero_in_degree_queue = deque([task for task in in_degree if in_degree[task] == 0])
    order = []

    while zero_in_degree_queue:
        current_task = zero_in_degree_queue.popleft()
        order.append(current_task)

        for neighbor in graph[current_task]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                zero_in_degree_queue.append(neighbor)

    return order if len(order) == len(in_degree) else []

# Example usage
print(determine_processing_order([(1, 2), (2, 3), (3, 4)]))  # Output: [1, 2, 3, 4]
print(determine_processing_order([(1, 2), (2, 3), (3, 1)]))  # Output: []
```

SOLUTION_JS:
```javascript
function determineProcessingOrder(dependencies) {
    const graph = new Map();
    const inDegree = new Map();

    for (const [x, y] of dependencies) {
        if (!graph.has(x)) graph.set(x, []);
        if (!graph.has(y)) graph.set(y, []);
        graph.get(x).push(y);
        inDegree.set(y, (inDegree.get(y) || 0) + 1);
        inDegree.set(x, inDegree.get(x) || 0);
    }

    const zeroInDegreeQueue = [];
    for (const [task, degree] of inDegree.entries()) {
        if (degree === 0) zeroInDegreeQueue.push(task);
    }

    const order = [];
    while (zeroInDegreeQueue.length > 0) {
        const currentTask = zeroInDegreeQueue.shift();
        order.push(currentTask);

        for (const neighbor of graph.get(currentTask)) {
            inDegree.set(neighbor, inDegree.get(neighbor) - 1);
            if (inDegree.get(neighbor) === 0) {
                zeroInDegreeQueue.push(neighbor);
            }
        }
    }

    return order.length === inDegree.size ? order : [];
}

// Example usage
console.log(determineProcessingOrder([[1, 2], [2, 3], [3, 4]]));  // Output: [1, 2, 3, 4]
console.log(determineProcessingOrder([[1, 2], [2, 3], [3, 1]]));  // Output: []
```

TIME_COMPLEXITY: O(V + E)
SPACE_COMPLEXITY: O(V)

EXPLANATION:
The approach utilizes topological sorting on a directed graph where each task represents a node and dependencies represent edges. We first build the graph and calculate the in-degrees for each node. Then, using a queue, we process nodes with zero in-degrees while updating in-degrees of neighboring nodes. If at the end the order contains all tasks, it means there's no circular dependency.
