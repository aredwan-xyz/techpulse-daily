TITLE: Inventory Management System
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are building an inventory management system for a warehouse that needs to track the quantity of various products. Each product is identified by a unique alphanumeric SKU (Stock Keeping Unit). Your goal is to implement a function that takes a list of transactions representing additions and removals of inventory items and returns the current stock levels of each product. Each transaction is represented as a tuple `(SKU, quantity)`, where `quantity` can be positive (addition) or negative (removal). If a removal would cause the quantity to fall below zero, it should not be applied.

EXAMPLES:
Input: 
```python
transactions = [("A123", 10), ("B456", 5), ("A123", -3), ("B456", -10), ("C789", 1), ("A123", -8)]
```
Output: 
```python
{"A123": 0, "B456": 0, "C789": 1}
```
Explanation: The final stock levels after processing the transactions are: A123 has 0 left after a total of 10 added and 11 removed, B456 has 0 (5 added and 10 removed), and C789 has 1.

Input: 
```python
transactions = [("A123", 2), ("B456", 3), ("C789", 5), ("A123", -1), ("C789", -2), ("B456", -5)]
```
Output: 
```python
{"A123": 1, "B456": 0, "C789": 3}
```
Explanation: The final counts: A123 has 1 left (2 added, 1 removed), B456 has 0, and C789 has 3 remaining (5 added, 2 removed).

CONSTRAINTS:
- The SKU is a non-empty string containing alphanumeric characters only.
- The quantity is an integer (positive or negative) and can go up to ±1000 for each transaction.

HINT:
Consider using a hash map to keep track of the stock levels efficiently.

SOLUTION_PYTHON:
```python
def current_stock(transactions):
    stock = {}  # Create a hash map to store stock levels
    
    for sku, quantity in transactions:
        if sku not in stock:
            stock[sku] = 0  # Initialize SKU if it doesn't exist
            
        # Update stock level while ensuring it doesn't go below zero
        if quantity > 0 or (quantity < 0 and stock[sku] + quantity >= 0):
            stock[sku] += quantity
            
    return stock
```

SOLUTION_JS:
```javascript
function currentStock(transactions) {
    const stock = {}; // Create an object to store stock levels
    
    for (const [sku, quantity] of transactions) {
        if (!(sku in stock)) {
            stock[sku] = 0; // Initialize SKU if it doesn't exist
        }
        
        // Update stock level while ensuring it doesn't go below zero
        if (quantity > 0 || (quantity < 0 && stock[sku] + quantity >= 0)) {
            stock[sku] += quantity;
        }
    }
    return stock;
}
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(m) where n is the number of transactions and m is the number of unique SKUs.

EXPLANATION:
The algorithm iterates through each transaction and updates the stock quantities in a hash map based on the SKU. This allows for efficient access and updates with an average time complexity of O(1) for both operations, resulting in an overall time complexity of O(n). The space complexity is determined by the number of unique SKUs stored in the hash map.

FOLLOW_UP:
What would change in your implementation if you also needed to enforce a maximum stock limit per SKU? Implement the necessary modifications.
