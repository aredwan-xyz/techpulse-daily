TITLE: Distributing Tasks to Workers  
DIFFICULTY: Medium  
TOPIC: Greedy Algorithms  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are managing a team of workers, and you need to distribute a set of tasks to them. Each task has an associated deadline and a reward value. Your goal is to maximize the total reward while ensuring that no two tasks are assigned to the same worker at the same time. You can assume that every worker can only handle one task at a time and they can work on only one task per day. Tasks that are not completed by their deadlines will have zero reward.

Function signature:
```python
def max_total_reward(tasks: List[Tuple[int, int]]) -> int:
```

Where `tasks` is a list of tuples where each tuple contains two integers:
- The first integer represents the `deadline` of the task (1-indexed).
- The second integer represents the `reward` for completing the task.

EXAMPLES:  
Input: `[(2, 100), (1, 50), (2, 30), (1, 20)]`  
Output: `150`  
Explanation: Assign task (2, 100) to day 2 and (1, 50) to day 1; tasks (2, 30) and (1, 20) will be ignored as they overlap.

Input: `[(1, 10), (1, 20), (3, 30), (2, 25), (5, 15)]`  
Output: `75`  
Explanation: Assign (1, 20) to day 1, (2, 25) to day 2, (3, 30) to day 3, and (5, 15) to day 5.

CONSTRAINTS:  
- The number of tasks will be in the range of `1 ≤ len(tasks) ≤ 1000`.  
- The `deadline` of each task will range from `1` to `1000`.  
- The `reward` will be in the range of `1` to `10^5`.  

HINT:  
Consider sorting the tasks based on their rewards and then greedily assigning them to the latest available day before their deadline.

SOLUTION_PYTHON:
```python
from typing import List, Tuple

def max_total_reward(tasks: List[Tuple[int, int]]) -> int:
    # Sort tasks in descending order by reward
    tasks.sort(key=lambda x: -x[1])
    
    max_deadline = max(task[0] for task in tasks)
    # Create an array to track days when a task can be scheduled
    days = [False] * (max_deadline + 1)
    
    total_reward = 0
    
    for deadline, reward in tasks:
        # Find the latest available day before or on the deadline
        for day in range(deadline, 0, -1):
            if not days[day]:  # If the day is free
                days[day] = True  # Mark this day as taken
                total_reward += reward  # Add reward to total
                break
    
    return total_reward
```

SOLUTION_JS:
```javascript
function maxTotalReward(tasks) {
    // Sort tasks in descending order by reward
    tasks.sort((a, b) => b[1] - a[1]);
    
    let maxDeadline = Math.max(...tasks.map(task => task[0]));
    let days = new Array(maxDeadline + 1).fill(false);
    
    let totalReward = 0;
    
    for (const [deadline, reward] of tasks) {
        // Find the latest available day before or on the deadline
        for (let day = deadline; day > 0; day--) {
            if (!days[day]) { // If the day is free
                days[day] = true; // Mark this day as taken
                totalReward += reward; // Add reward to total
                break;
            }
        }
    }
    
    return totalReward;
}
```

TIME_COMPLEXITY: O(n log n)  
SPACE_COMPLEXITY: O(n)  

EXPLANATION:  
The approach utilizes a greedy strategy where tasks are sorted by their rewards. For each task, we attempt to assign it to the latest possible day available that is before its deadline. This maximizes our chances of scheduling higher-reward tasks while ensuring no two tasks overlap. This solution effectively balances both the need to meet deadlines and maximize the overall reward.
