TITLE: Build Positive Integer Combinations
DIFFICULTY: Medium
TOPIC: Recursion & Backtracking
ESTIMATED_TIME: 30 minutes

PROBLEM:
Given a target positive integer `N`, create a function that returns all unique combinations of positive integers that sum up to `N`. Each number can be used multiple times in the combination. The output should consist of combinations sorted in non-decreasing order, and the combinations themselves should be sorted lexicographically.

EXAMPLES:
Input: `N = 5`
Output: `[[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]`
Explanation: Here, combinations like [1, 1, 1, 1, 1] indicate that we use five 1s. Other combinations utilize different integers while summing up to 5.

Input: `N = 4`
Output: `[[1, 1, 1, 1], [1, 1, 2], [1, 3], [2, 2], [4]]`

CONSTRAINTS:
- `1 <= N <= 50`

HINT:
Consider using a backtracking approach to explore all potential combinations, where you maintain a running sum and a current combination as you delve deeper.

SOLUTION_PYTHON:
```python
def combination_sum(N):
    result = []
    
    def backtrack(current_combination, remaining_sum, start):
        if remaining_sum == 0:
            result.append(current_combination.copy())
            return
        for num in range(start, N + 1):
            if remaining_sum - num >= 0:
                current_combination.append(num)
                backtrack(current_combination, remaining_sum - num, num)
                current_combination.pop()

    backtrack([], N, 1)
    return result

# Example usage
print(combination_sum(5))  # Output: [[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]
```

SOLUTION_JS:
```javascript
function combinationSum(N) {
    const result = [];
    
    function backtrack(currentCombination, remainingSum, start) {
        if (remainingSum === 0) {
            result.push([...currentCombination]);
            return;
        }
        for (let num = start; num <= N; num++) {
            if (remainingSum - num >= 0) {
                currentCombination.push(num);
                backtrack(currentCombination, remainingSum - num, num);
                currentCombination.pop();
            }
        }
    }

    backtrack([], N, 1);
    return result;
}

// Example usage
console.log(combinationSum(5));  // Output: [[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]
```

TIME_COMPLEXITY: O(2^N)
SPACE_COMPLEXITY: O(N)

EXPLANATION:
This problem utilizes recursion and backtracking to explore all valid combinations. By starting from the number `1`, each path is incrementally built based on remaining sums. Each time `remaining_sum` hits zero, a valid combination is found, which is added to the results. Thus, the exploration grows exponentially in terms of potential combinations, leading to a time complexity of O(2^N).
