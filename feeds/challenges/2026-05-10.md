TITLE: Finding Fixed Points in a Rotated Array  
DIFFICULTY: Medium  
TOPIC: Binary Search  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are given a rotated sorted array of distinct integers and need to find if there's any index where the element at that index is equal to the index itself. In other words, find all indices `i` such that `nums[i] == i`. If such indices exist, return them; otherwise, return an empty list.

A rotated sorted array is created by taking a sorted array and rotating it at some pivot unknown to you beforehand. For example, an original sorted array `[0, 1, 2, 3, 4]` could be rotated to `[3, 4, 0, 1, 2]` or any other combination.

EXAMPLES:  
Input: `[0, 2, 5, 8, 17]`  
Output: `[0]`  
Explanation: At index `0`, the element is `0`, which matches its index.

Input: `[1, 2, 3, 4, 5]`  
Output: `[]`  
Explanation: There are no indices where `nums[i] == i`.

EXAMPLES:  
Input: `[-1, 0, 1, 3, 5]`  
Output: `[3]`  
Explanation: At index `3`, the element is `3`, which matches its index.

Input: `[2, 3, 4, 5, 6, 7]`  
Output: `[]`  
Explanation: No indices match their respective values.

CONSTRAINTS:  
- The array length must be between `1` and `10^5`.  
- The elements of the array will be between `-10^9` and `10^9`.

HINT:  
Consider leveraging the properties of the binary search to minimize your checks when determining if an index is a fixed point.

SOLUTION_PYTHON:
```python
def find_fixed_points(nums):
    fixed_points = []
    left, right = 0, len(nums) - 1
    
    while left <= right:
        mid = left + (right - left) // 2
        
        # Check if mid is a fixed point
        if nums[mid] == mid:
            fixed_points.append(mid)
            # Continue searching left and right for other fixed points
            left_sub, right_sub = mid + 1, mid - 1
            while left_sub <= right:
                if nums[left_sub] == left_sub:
                    fixed_points.append(left_sub)
                left_sub += 1
            while right_sub >= left:
                if nums[right_sub] == right_sub:
                    fixed_points.append(right_sub)
                right_sub -= 1
            break
        elif nums[mid] < mid:
            left = mid + 1
        else:
            right = mid - 1

    return sorted(fixed_points)

# Time Complexity: O(n) in the worst case when looking for fixed points
# Space Complexity: O(k), where k is the number of fixed points found.
```

SOLUTION_JS:
```javascript
function findFixedPoints(nums) {
    const fixedPoints = [];
    let left = 0, right = nums.length - 1;

    while (left <= right) {
        const mid = Math.floor((left + right) / 2);
        
        if (nums[mid] === mid) {
            fixedPoints.push(mid);
            // Check for more fixed points in both directions
            for (let i = mid + 1; i < nums.length; i++) {
                if (nums[i] === i) fixedPoints.push(i);
            }
            for (let i = mid - 1; i >= 0; i--) {
                if (nums[i] === i) fixedPoints.push(i);
            }
            break;
        } else if (nums[mid] < mid) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return fixedPoints.sort((a, b) => a - b);
}

// Time Complexity: O(n) in the worst case when looking for fixed points
// Space Complexity: O(k), where k is the number of fixed points found.
```

TIME_COMPLEXITY: O(n)  
SPACE_COMPLEXITY: O(k)  

EXPLANATION:  
By combining binary search with a linear scan, the approach efficiently identifies fixed points. The binary search helps narrow the range of indices where we need to check for fixed points. When an index matches its value, we then check both sides linearly for additional fixed points. This is optimal given the constraints of the problem.
