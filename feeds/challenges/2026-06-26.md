TITLE: Sort the Exceptions
DIFFICULTY: Medium
TOPIC: Sorting & Searching
ESTIMATED_TIME: 30 minutes

PROBLEM:
In a software system, you are logging various types of exceptions that occur, each represented by a string. A requirement has arisen to sort these exceptions based on how often they occur. If two exceptions have the same frequency, they should be sorted lexicographically (in dictionary order) within that frequency group. Your task is to write a function that takes a list of exception strings and returns a list of unique exceptions sorted as per the given rule.

EXAMPLES:
Input: `["NullReferenceException", "ArgumentException", "NullReferenceException", "IndexOutOfRangeException", "ArgumentException", "ArgumentException"]`
Output: `["ArgumentException", "NullReferenceException", "IndexOutOfRangeException"]`
Explanation: "ArgumentException" occurs 3 times, "NullReferenceException" occurs 2 times, and "IndexOutOfRangeException" occurs 1 time, thus sorted in descending frequency and lexicographically.

Input: `["FileNotFoundException", "IOException", "IOException", "FileNotFoundException", "IOException", "ArgumentNullException"]`
Output: `["IOException", "FileNotFoundException", "ArgumentNullException"]`
Explanation: "IOException" occurs 3 times, "FileNotFoundException" occurs 2 times, and "ArgumentNullException" occurs 1 time, thus sorted accordingly.

CONSTRAINTS:
- The input list can have a length of up to 10^4.
- Each exception string can have a length of up to 100 characters.

HINT:
Consider using a dictionary to count the occurrences of each exception before sorting them based on the defined criteria.

SOLUTION_PYTHON:
```python
def sort_exceptions(exceptions):
    from collections import Counter
    
    # Count the occurrences of each exception
    counts = Counter(exceptions)
    
    # Sort by frequency (in descending order) and lexicographically (ascending)
    sorted_exceptions = sorted(counts.keys(), key=lambda x: (-counts[x], x))
    
    return sorted_exceptions
```

SOLUTION_JS:
```javascript
function sortExceptions(exceptions) {
    const counts = {};
    
    // Count the occurrences of each exception
    exceptions.forEach(exception => {
        counts[exception] = (counts[exception] || 0) + 1;
    });

    // Convert the counts object to an array and sort it
    return Object.keys(counts)
        .sort((a, b) => {
            // Sort by negative count to get descending order
            // In case of tie, sort alphabetically
            return counts[b] - counts[a] || a.localeCompare(b); 
        });
}
```

TIME_COMPLEXITY: O(n log n)
SPACE_COMPLEXITY: O(n)

EXPLANATION:
The approach leverages a counting dictionary to tally occurrences of each exception, which takes O(n) time. Sorting the unique exceptions based on their frequency and lexicographical order takes O(m log m), where m is the number of unique exceptions. The final time complexity is O(n + m log m), making it efficient for the given constraints, especially using built-in sorting methods.
