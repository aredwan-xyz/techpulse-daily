TITLE: Find the Peak Element
DIFFICULTY: Medium
TOPIC: Binary Search
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given a list of integers that is guaranteed to be unimodal, meaning that it ascends first, reaches a peak element, and then descends. Your task is to implement a function that finds and returns the peak element. A peak element is an element that is greater than its neighbors. If the element is at the boundary (first or last), it should be compared only with its one neighbor.

A unimodal array can be one such as `[1, 3, 20, 4, 2]` where `20` is the peak. Your solution should work in O(log n) time.

EXAMPLES:
Input: `[1, 3, 20, 4, 2]`
Output: `20`
Explanation: The peak element in the array is `20`, which is greater than its neighbors `3` and `4`.

Input: `[5, 10, 20, 15]`
Output: `20`
Explanation: The peak element in the array is `20`, which is greater than its neighbor `10`.

CONSTRAINTS:
- The array will have at least one element.
- All elements in the array are distinct.

HINT:
Consider using the properties of binary search by comparing the middle element with its neighbors.

SOLUTION_PYTHON:
```python
def find_peak(nums):
    left, right = 0, len(nums) - 1
    
    while left < right:
        mid = left + (right - left) // 2
        
        # Compare mid with its next element
        if nums[mid] < nums[mid + 1]:
            left = mid + 1  # peak is on the right side
        else:
            right = mid  # peak is on the left side or mid is the peak
            
    return nums[left]  # left == right is the peak

# Time Complexity: O(log n)
# Space Complexity: O(1)
```

SOLUTION_JS:
```javascript
function findPeak(nums) {
    let left = 0, right = nums.length - 1;

    while (left < right) {
        let mid = Math.floor((left + right) / 2);

        // Compare mid with its next element
        if (nums[mid] < nums[mid + 1]) {
            left = mid + 1;  // peak is on the right side
        } else {
            right = mid;  // peak is on the left side or mid is the peak
        }
    }
    
    return nums[left]; // left == right is the peak
}

// Time Complexity: O(log n)
// Space Complexity: O(1)
```

TIME_COMPLEXITY: O(log n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach utilizes binary search to efficiently locate the peak element. By comparing the middle element with its right neighbor, we can determine in which direction to search for the potential peak. If the middle element is less than its neighbor, it implies that a peak exists in the right half, hence we adjust our search range accordingly. This guarantees that we will find the peak in logarithmic time.
