TITLE: Find the Maximum Element in a Rotated Sorted Array
DIFFICULTY: Medium
TOPIC: Binary Search
ESTIMATED_TIME: 20 minutes

PROBLEM:
Imagine you are given an array that was sorted in ascending order but then rotated at some pivot unknown to you beforehand. Your task is to find the maximum element in the rotated sorted array. For example, if you are given the array [4, 5, 6, 7, 0, 1, 2], the maximum element is 7.

You need to accomplish this using a binary search approach to maintain efficiency.

EXAMPLES:
Input: [3, 4, 5, 1, 2]
Output: 5
Explanation: The maximum element 5 is located before the rotation point.

Input: [1, 2, 3, 4, 5]
Output: 5
Explanation: The maximum element 5 is at the end of the array and there is no rotation.

CONSTRAINTS:
- The input array must have at least 1 element and at most 10^6 elements.
- All elements in the array are unique integers.

HINT:
Consider how the properties of a sorted array can help you identify the maximum element by focusing on the rotation points.

SOLUTION_PYTHON:
```python
def find_max_in_rotated_array(nums):
    if not nums:
        return None  # Handle edge case for an empty array

    left, right = 0, len(nums) - 1
    
    while left < right:
        mid = (left + right) // 2
        
        # If middle element is greater than the rightmost element
        # then the maximum is in the right half
        if nums[mid] > nums[right]:
            left = mid + 1
        else:
            # Otherwise, the maximum is in the left half
            right = mid
    
    # When the loop ends, left is pointing at the smallest element,
    # the maximum is the previous element.
    return nums[left - 1]

# Example usage
print(find_max_in_rotated_array([4, 5, 6, 7, 0, 1, 2]))  # Output: 7
```

SOLUTION_JS:
```javascript
function findMaxInRotatedArray(nums) {
    if (!nums.length) return null; // Handle edge case for an empty array

    let left = 0, right = nums.length - 1;
    
    while (left < right) {
        const mid = Math.floor((left + right) / 2);
        
        // If middle element is greater than the rightmost element
        // then the maximum is in the right half
        if (nums[mid] > nums[right]) {
            left = mid + 1;
        } else {
            // Otherwise, the maximum is in the left half
            right = mid;
        }
    }

    // When the loop ends, left is pointing at the smallest element,
    // the maximum is the previous element.
    return nums[left - 1];
}

// Example usage
console.log(findMaxInRotatedArray([4, 5, 6, 7, 0, 1, 2])); // Output: 7
```

TIME_COMPLEXITY: O(log n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
In this problem, we employ a modified binary search to identify the rotation point and, consequently, the maximum element in a rotated sorted array. By comparing the mid element to the rightmost element, we can efficiently eliminate half of the search space based on whether the left or right segment is sorted. This approach works in logarithmic time, making it optimal for large datasets.
