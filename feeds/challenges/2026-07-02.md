TITLE: Social Media Friend Recommendations
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are developing a social media platform that includes a feature for suggesting friends to users based on their mutual connections. Given a list of user friendships, represented as pairs of user IDs, you need to write a function that returns a list of friend recommendations for a specific user. A friend recommendation is a user that shares at least one mutual friend with the specified user but is not already a friend or the user themselves.

For example, if user A is friends with users B and C, and user B is friends with user D, then user D would be a friend recommendation for user A.

Write a function `get_friend_recommendations(user_id: int, friendships: List[Tuple[int, int]]) -> List[int]` that takes in a user ID and the list of friendships.

EXAMPLES:
Input: 
```python
user_id = 1
friendships = [(1, 2), (1, 3), (2, 4), (2, 5), (3, 6)]
```
Output: 
```python
[4, 5, 6]
```
Explanation: User 1 is friends with users 2 and 3. User 2 has friends 4 and 5 (mutual friends with user 1) and user 3 has a friend 6 (mutual friend with user 1). 

Input: 
```python
user_id = 2
friendships = [(1, 2), (1, 3), (2, 4), (2, 5), (3, 6)]
```
Output:
```python
[3, 4, 5]
```
Explanation: User 2 is friends with user 1, and user 1 is friends with users 3. Additionally, user 2 is friends with 4 and 5 (they wouldn't be included in recommendations as they are already friends with 2).

CONSTRAINTS:
- The `friendships` list will contain at most 1000 friendships.
- User IDs are integers between 1 and 10^6.

HINT:
Consider using a set to manage and eliminate already-existing friendships effectively.

SOLUTION_PYTHON:
```python
def get_friend_recommendations(user_id, friendships):
    from collections import defaultdict
    
    # Create a dictionary to hold friendships
    friend_map = defaultdict(set)
    
    # Populate the friend map
    for user1, user2 in friendships:
        friend_map[user1].add(user2)
        friend_map[user2].add(user1)
    
    # Set to collect recommendations
    recommendations = set()
    
    # Get the friends of the selected user
    if user_id not in friend_map:
        return []
    
    direct_friends = friend_map[user_id]
    
    # Gather mutual friends' friends
    for friend in direct_friends:
        for friend_of_friend in friend_map[friend]:
            # Ensure it's not the user or already a friend
            if friend_of_friend != user_id and friend_of_friend not in direct_friends:
                recommendations.add(friend_of_friend)
    
    return list(recommendations)

# Example usage
print(get_friend_recommendations(1, [(1, 2), (1, 3), (2, 4), (2, 5), (3, 6)]))  # Output: [4, 5, 6]
```

SOLUTION_JS:
```javascript
function getFriendRecommendations(userId, friendships) {
    const friendMap = new Map();

    // Populate the friend map
    for (const [user1, user2] of friendships) {
        if (!friendMap.has(user1)) friendMap.set(user1, new Set());
        if (!friendMap.has(user2)) friendMap.set(user2, new Set());
        friendMap.get(user1).add(user2);
        friendMap.get(user2).add(user1);
    }

    // Set to collect recommendations
    const recommendations = new Set();
    
    if (!friendMap.has(userId)) return [];

    const directFriends = friendMap.get(userId);
    
    // Gather mutual friends' friends
    for (const friend of directFriends) {
        for (const friendOfFriend of friendMap.get(friend)) {
            // Ensure it's not the user or already a friend
            if (friendOfFriend !== userId && !directFriends.has(friendOfFriend)) {
                recommendations.add(friendOfFriend);
            }
        }
    }
    
    return Array.from(recommendations);
}

// Example usage
console.log(getFriendRecommendations(1, [[1, 2], [1, 3], [2, 4], [2, 5], [3, 6]])); // Output: [4, 5, 6]
```

TIME_COMPLEXITY: O(N)
SPACE_COMPLEXITY: O(N)

EXPLANATION:
The algorithm efficiently uses a hash map to store friendships, allowing for O(1) average-time complexity when checking direct friends and their friends. For each user, it aggregates recommendations based on mutual friendships, ensuring that we don't recommend anyone who is already a direct friend or the user themselves. This approach is optimal as it processes the friendship pairs only once and leverages the quick access capabilities of sets.
