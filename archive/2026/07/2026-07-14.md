## 💡 Coding Challenge


TITLE: Time Series Anomaly Detection with Rolling Mean
DIFFICULTY: Hard
TOPIC: AI/ML Coding (NumPy/Pandas)
ESTIMATED_TIME: 60 minutes

PROBLEM:
You are given a time series dataset as a Pandas DataFrame with two columns: 'timestamp' and 'value'. Your task is to implement an anomaly detection algorithm that identifies points in the dataset that deviate significantly (outer threshold) from the rolling mean of a specified window size. A point is detected as an anomaly if its absolute difference from the rolling mean exceeds a specified multiplier of the rolling standard deviation. Return a DataFrame containing only the anomalies with their corresponding timestamps and values.

EXAMPLES:
Input:
```python
data = {
    'timestamp': pd.date_range(start='2026-07-01', periods=10, freq='D'),
    'value': [10, 12, 11, 13, 200, 12, 12, 11, 10, 15]
}
df = pd.DataFrame(data)
window_size = 3
multiplier = 2
```

Output:
```python
            timestamp  value
4 2026-07-05       200
```
Explanation: The value 200 significantly deviates from the rolling mean calculated over the last 3 days, making it an anomaly.

Input:
```python
data = {
    'timestamp': pd.date_range(start='2026-07-01', periods=8, freq='D'),
    'value': [1, 1, 1, 10, 1, 1, 2, 1]
}
df = pd.DataFrame(data)
window_size = 2
multiplier = 3
```

Output:
```python
            timestamp  value
3 2026-07-04        10
```
Explanation: The value of 10 is an anomaly as it deviates significantly from the rolling mean of the previous 2 days.

CONSTRAINTS:
- The DataFrame will contain at least 5 entries and no more than 500,000 entries.
- The window size will be at least 2 and at most half of the DataFrame length.

HINT:
Use Pandas' rolling() function to calculate the rolling mean and standard deviation efficiently.

SOLUTION_PYTHON:
```python
import pandas as pd

def detect_anomalies(df, window_size, multiplier):
    # Calculate the rolling mean and standard deviation
    df['rolling_mean'] = df['value'].rolling(window=window_size).mean()
    df['rolling_std'] = df['value'].rolling(window=window_size).std()

    # Detect anomalies
    anomalies = df[
        (df['value'] - df['rolling_mean']).abs() > (multiplier * df['rolling_std'])
    ]

    # Return only timestamp and value for detected anomalies
    return anomalies[['timestamp', 'value']]

# Example usage
data = {
    'timestamp': pd.date_range(start='2026-07-01', periods=10, freq='D'),
    'value': [10, 12, 11, 13, 200, 12, 12, 11, 10, 15]
}
df = pd.DataFrame(data)
window_size = 3
multiplier = 2
print(detect_anomalies(df, window_size, multiplier))
```

SOLUTION_JS:
```javascript
function detectAnomalies(data, windowSize, multiplier) {
    const anomalies = [];
    const rollingMean = [];
    const rollingStd = [];
    
    for (let i = 0; i < data.length; i++) {
        if (i >= windowSize - 1) {
            const window = data.slice(i - windowSize + 1, i + 1).map(item => item.value);
            const mean = window.reduce((acc, val) => acc + val, 0) / windowSize;
            rollingMean.push(mean);
            const variance = window.reduce((acc, val) => acc + Math.pow(val - mean, 2), 0) / windowSize;
            rollingStd.push(Math.sqrt(variance));
            
            const diff = Math.abs(data[i].value - mean);
            if (diff > multiplier * rollingStd[rollingStd.length - 1]) {
                anomalies.push({ timestamp: data[i].timestamp, value: data[i].value });
            }
        } else {
            rollingMean.push(null);
            rollingStd.push(null);
        }
    }
    
    return anomalies;
}

// Example usage
const data = [
    { timestamp: '2026-07-01', value: 10 },
    { timestamp: '2026-07-02', value: 12 },
    { timestamp: '2026-07-03', value: 11 },
    { timestamp: '2026-07-04', value: 13 },
    { timestamp: '2026-07-05', value: 200 },
    { timestamp: '2026-07-06', value: 12 },
    { timestamp: '2026-07-07', value: 12 },
    { timestamp: '2026-07-08', value: 11 },
    { timestamp: '2026-07-09', value: 10 },
    { timestamp: '2026-07-10', value: 15 }
];
const windowSize = 3;
const multiplier = 2;
console.log(detectAnomalies(data, windowSize, multiplier));
```

TIME_COMPLEXITY: O(N)
SPACE_COMPLEXITY: O(N)

EXPLANATION:
The approach involves calculating the rolling mean and standard deviation for the specified window size. Each of the values is examined to determine if they are anomalies based on the defined multipliers. The time complexity of O(N) is optimal for this problem as we need to inspect every element in the dataset exactly once for the calculations.

# Litestream

**Category:** Databases
**Link:** [https://github.com/benbjohnson/litestream](https://github.com/benbjohnson/litestream)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/benbjohnson/litestream)

---

## What Is It?
Litestream is an open-source tool that provides real-time replication for SQLite databases, allowing you to persist data to a remote storage service and ensure high availability and durability even in cases of failure.

## The Problem It Solves
Before Litestream, developers using SQLite faced challenges with data persistence and reliability, especially in production environments where a single point of failure could be catastrophic. Traditionally, SQLite's reliance on local file storage meant that if the host crashed, all data could be lost. Litestream addresses this by enabling continuous recovery and replication, making SQLite a more robust solution for modern applications.

## Why It's Worth 10 Minutes of Your Time
1. **Seamless Replication**: With Litestream, you can configure continuous replication of your SQLite database to cloud storage (like AWS S3) with minimal setup, reducing the odds of data loss without overhauling your database strategy.
2. **Cost-Effective**: It is a lightweight solution for applications that do not require the overhead of heavier database systems, making it ideal for serverless and edge deployments where low-latency is crucial.
3. **Incredible Ease of Use**: Litestream can be up and running in minutes and requires little more than setting environment variables, allowing developers to integrate it without diving deep into its internals.

## Quick Start
```bash
# Install Litestream
go get github.com/benbjohnson/litestream

# Start Litestream with a configuration file
litestream replicate -config litestream.yml
```

## Real-World Example
```yaml
# litestream.yml
version: "1"
databases:
  - path: /path/to/database.db
    replicas:
      - url: s3://your-bucket-name/path/to/database.db
```
This simple configuration pushes your SQLite database to an S3 bucket for backup, ensuring data integrity and availability.

## Who Should Use It
✅ Perfect for: Developers using SQLite in web and mobile applications who need reliable data backup solutions without switching to a more complex database system.  
⏩ Skip if: Your application needs advanced database features that SQLite and Litestream cannot provide, such as complex transactions or extensive querying capabilities.

## The Honest Take
Litestream offers a practical and compelling solution for SQLite users looking for improved data durability and recovery. It's straightforward to set up, but if your project demands high-performance SQL features or complex setups, you might want to look for a more full-fledged database system. Overall, it’s an excellent tool for lightweight applications needing reliable backups.
