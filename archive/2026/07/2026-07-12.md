## 📄 Wat3R: Underwater 3D Geometry Learning without Annotations

**Authors:** Jiangwei Ren, Xingyu Jiang, Zijie Song, Wei Xu et al.
**Published:** 2026-07-09
**arXiv ID:** [2607.08772v1](https://arxiv.org/abs/2607.08772v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](

---

## 🎯 The Core Idea
The researchers developed a method called Wat3R that enables the estimation of 3D shapes in underwater environments without needing any annotated training data. Instead of relying on labeled images, their framework uses existing underwater video footage to learn how to reconstruct 3D geometry.

## 🔍 The Problem They Solved
Before this paper, creating accurate 3D models of underwater scenes was difficult because high-quality annotated datasets were nearly impossible to obtain, and existing methods relied heavily on these annotations. Moreover, the underwater environment poses unique challenges like light distortion and visibility issues that conventional methods could not address effectively.

## 💡 The Key Innovation
Wat3R introduces a semi-supervised learning approach that transfers knowledge from air-based to underwater 3D reconstruction models without requiring any annotation for underwater data. By employing a teacher-student architecture, the framework learns from unlabeled underwater videos, utilizing a new loss function that maintains consistency across different views to improve geometry accuracy despite prevalent distortions.

## 🛠️ Why Developers Should Care
This research simplifies the development of AI systems for underwater applications, significantly reducing the reliance on costly and time-consuming data annotation. By leveraging unlabeled data, developers can now create advanced 3D reconstruction models that can operate effectively in challenging underwater conditions, opening new possibilities for fields like marine biology and underwater robotics.

## ⚡ TL;DR
Wat3R enables 3D shape estimation in underwater environments using only unlabeled video footage, eliminating the need for extensive data annotation.

## 💡 Coding Challenge


TITLE: Maximize Event Participation
DIFFICULTY: Medium
TOPIC: Greedy Algorithms
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are organizing a series of events, and each event has a specific start and end time. Your goal is to determine the maximum number of events that you can attend without overlapping any sessions. You can only attend one event at a time. Each event is represented by a pair of integers (start, end), where 'start' is the start time of the event and 'end' is the end time. 

Your task is to write a function `max_events(events: List[Tuple[int, int]]) -> int` that takes a list of events and returns the maximum number of non-overlapping events you can attend.

EXAMPLES:
Input: `[(1, 3), (2, 5), (4, 6), (7, 8)]`
Output: `3`
Explanation: You can attend events (1, 3), (4, 6), and (7, 8).

Input: `[(1, 4), (2, 3), (3, 5), (6, 8)]`
Output: `3`
Explanation: You can attend events (2, 3), (3, 5), and (6, 8).

CONSTRAINTS:
- 1 <= len(events) <= 1000
- 1 <= start < end <= 10^4

HINT:
Consider sorting the events based on their end times to help with selecting non-overlapping events.

SOLUTION_PYTHON:
```python
from typing import List, Tuple

def max_events(events: List[Tuple[int, int]]) -> int:
    # Sort events based on their end times
    events.sort(key=lambda x: x[1])
    
    count = 0  # To count the number of events we can attend
    last_end_time = 0  # The end time of the last attended event
    
    for start, end in events:
        if start >= last_end_time:  # If the start time of the current event is >= last end time
            count += 1  # Attend this event
            last_end_time = end  # Update last end time
            
    return count

# Time Complexity: O(n log n) due to sorting
# Space Complexity: O(1) as we are using a constant amount of space
```

SOLUTION_JS:
```javascript
function maxEvents(events) {
    // Sort events based on their end times
    events.sort((a, b) => a[1] - b[1]);
    
    let count = 0; // To count the number of events we can attend
    let lastEndTime = 0; // The end time of the last attended event
    
    for (let [start, end] of events) {
        if (start >= lastEndTime) { // If the start time is >= last end time
            count++; // Attend this event
            lastEndTime = end; // Update last end time
        }
    }
    
    return count;
}

// Time Complexity: O(n log n) due to sorting
// Space Complexity: O(1) as we are using a constant amount of space
```

TIME_COMPLEXITY: O(n log n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach leverages sorting to arrange the events based on their end times. By doing this, we prioritize attending events that finish earlier, allowing us to maximize the number of non-overlapping events attended. The greedy choice is made by selecting the next event that starts after or when the last attended event ends, ensuring the optimal selection of events with minimal overlapping.

# Just

**Category:** Dev Tools
**Link:** [https://github.com/casey/just](https://github.com/casey/just)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](

---

## What Is It?
Just is a command runner that simplifies script execution in a way that makes it accessible and easy to use for developers. It allows you to define and run tasks directly from the command line while providing a more organized syntax compared to traditional shell scripts.

## The Problem It Solves
Before Just, many developers relied on Makefiles or complex shell scripts to manage their project tasks, which could lead to confusion and a steep learning curve for newcomers. Just addresses this by offering a more straightforward alternative where tasks can be defined with a simpler syntax, improving project documentation and usability.

## Why It's Worth 10 Minutes of Your Time
1. **Simplicity**: Just uses a user-friendly syntax that is easier to learn and remember compared to Makefiles or complex shell scripts, making task definition accessible for all team members.
2. **Organized Task Management**: You can group and document your tasks in a single `Justfile`, keeping all necessary commands in one place and eliminating the need to switch contexts or search for scripts.
3. **Instant Task Execution**: Just allows you to run tasks simply by typing `just <task_name>`, streamlining both your workflow and team communication regarding project setup and maintenance.

## Quick Start
```bash
# Install Just via package manager
brew install casey/just/just # macOS
# Create a Justfile
echo 'build: echo "Building project..."' > Justfile
# Run a task
just build
```

## Real-World Example
```bash
# Justfile
test: 
    echo "Running tests..."
    pytest tests/

# Command to run tests
just test
```

## Who Should Use It
✅ Perfect for: Teams managing complex projects, solo developers looking for quick task management, or anyone seeking to streamline command execution.  
⏩ Skip if: You require extensive cross-platform scripting features or prefer traditional shell scripting.

## The Honest Take
Just is a powerful tool that brings simplicity and clarity to task management in development workflows. While it excels at defining and running tasks, it may lack some advanced features found in more established scripting solutions. Bottom line: If you want to make task execution easier and more organized, Just is definitely worth a try!
