## 💡 Coding Challenge


TITLE: Social Network Connection
DIFFICULTY: Medium
TOPIC: Two Pointers
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are tasked with analyzing friendships in a social network represented as an array of strings, where each string is a unique username. Each username can have varying lengths. Your goal is to find whether there exists a pair of usernames such that one username is a prefix of the other. To efficiently solve this problem, utilize the two-pointer technique to check for prefixes after sorting the usernames. 

Your function should return `True` if such a pair exists, and `False` otherwise.

EXAMPLES:
Input: `["alice", "al", "bob", "alex", "alexander"]`
Output: `True`
Explanation: The username "al" is a prefix of "alice".

Input: `["charlie", "david", "eve", "frank"]`
Output: `False`
Explanation: There are no usernames that share a prefix.

CONSTRAINTS:
- All usernames consist of lowercase letters and have a length of at least 1 and at most 100.
- The total number of usernames will not exceed 10^5.

HINT:
Consider sorting the usernames first; this will help in spotting prefixes easily as they will be adjacent.

SOLUTION_PYTHON:
```python
def has_prefix_pair(usernames):
    # Sort the usernames to bring potential prefix pairs next to each other
    usernames.sort()
    
    # Iterate through the sorted list and check for prefixes
    for i in range(len(usernames) - 1):
        prefix = usernames[i]
        next_username = usernames[i + 1]
        # Check if the next username starts with the current username
        if next_username.startswith(prefix):
            return True
            
    return False

# Time Complexity: O(n log n) for sorting, O(n) for the linear scan.
# Space Complexity: O(1) for the pointers, O(n) if you consider space used by sort.
```

SOLUTION_JS:
```javascript
function hasPrefixPair(usernames) {
    // Sort the usernames to bring potential prefix pairs next to each other
    usernames.sort();
    
    // Iterate through the sorted list and check for prefixes
    for (let i = 0; i < usernames.length - 1; i++) {
        let prefix = usernames[i];
        let nextUsername = usernames[i + 1];
        // Check if the next username starts with the current username
        if (nextUsername.startsWith(prefix)) {
            return true;
        }
    }
    return false;
}

// Time Complexity: O(n log n) for sorting, O(n) for the linear scan.
// Space Complexity: O(1) for the pointers, O(n) if you consider space used by sort.
```

TIME_COMPLEXITY: O(n log n)
SPACE_COMPLEXITY: O(1) (if ignoring sort space)

EXPLANATION:
The approach leverages the sorting of usernames to group potential prefixes together, allowing for a single linear scan to check adjacent pairs for prefix matches. By utilizing the built-in string comparison methods, we ensure efficient checks for prefixes while maintaining clear code readability.

FOLLOW_UP:
What modifications would you make to your solution to handle cases where usernames can also contain numbers or special characters?

## 🛠️ Tool Spotlight


# LiteLLM

**Category:** AI/LLM
**Link:** [https://github.com/BerriAI/litellm](https://github.com/BerriAI/litellm)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/BerriAI/litellm)

---

## What Is It?
LiteLLM is a lightweight and efficient library designed for deploying large language models (LLMs) on resource-constrained environments, providing developers with the tools to build and run LLM applications without the overhead of traditional frameworks.

## The Problem It Solves
Before LiteLLM, developers struggled to find a balance between the powerful capabilities of large language models and the limitations of their environments, often needing heavy resources that were expensive and complicated to manage. LiteLLM addresses this pain by simplifying the deployment process and drastically reducing the resource requirements, enabling applications that can run anywhere.

## Why It's Worth 10 Minutes of Your Time
1. **Resource Efficiency**: LiteLLM optimizes memory usage, allowing you to run LLMs on devices with limited computational power, such as edge devices or older hardware.
2. **Easy Integration**: With a straightforward API, integrating LiteLLM into existing applications or workflows takes minimal effort, reducing the time to go from development to deployment.
3. **Versatile Use Cases**: Whether it’s for chatbots, content generation, or language translation, LiteLLM provides flexible options that can adapt to various applications, improving your efficiency and creativity.

## Quick Start
```bash
pip install litellm
litellm run --model <model_name>
```

## Real-World Example
```python
from litellm import LiteLLM

model = LiteLLM("gpt-small", device="cpu")
response = model.generate("What is the future of AI?")
print(response)
```

## Who Should Use It
✅ Perfect for: Developers looking to implement LLMs in low-resource environments, startups needing to prototype LLM-based applications affordably, and data scientists wanting quick insights without heavy infrastructure.  
⏩ Skip if: Your application requires the latest LLM advancements that demand extensive resources, or if you're working in high-performance production environments.

## The Honest Take
LiteLLM wonderfully bridges the gap between accessibility and functionality in the LLM landscape, making it a strong contender for developers with limited resources. However, while it excels in efficiency, it may not provide the most cutting-edge model performance compared to larger, heavier frameworks—so weigh your needs carefully!
