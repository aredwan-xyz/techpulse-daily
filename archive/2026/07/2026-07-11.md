## 🎯 The Core Idea
This paper presents UniClawBench, a new testing framework specifically designed to evaluate proactive AI agents in real-world tasks. It focuses on assessing agents based on their specific skills rather than just their overall performance.

## 🔍 The Problem They Solved
Previously, benchmarks for AI agents often operated in controlled environments and didn’t capture the complexities of real-world scenarios. Additionally, they combined different model capabilities into single categories, making it hard to pinpoint exactly where an agent might be failing.

## 💡 The Key Innovation
UniClawBench introduces a capability-driven approach to benchmarking, focusing on five key skills that proactive agents need to perform in real-world settings. This allows for a more nuanced evaluation of how well agents use their skills to complete varied tasks and helps isolate specific weaknesses that could lead to improvement.

## 🛠️ Why Developers Should Care
With UniClawBench, developers can better assess the individual capabilities of their AI agents, leading to targeted enhancements and ultimately more effective systems. This could streamline development processes and improve the reliability of AI applications in everyday use.

## ⚡ TL;DR
A new benchmark called UniClawBench helps evaluate AI agents more effectively in real-world tasks by focusing on their specific skills rather than just overall performance.

## 💡 Coding Challenge


TITLE: Stack of Plates
DIFFICULTY: Hard
TOPIC: Stack & Queue
ESTIMATED_TIME: 45 minutes

PROBLEM:
You are designing a service that needs to manage a stack of plates in a restaurant. The restaurant has a fixed number of plates, and they can be stacked or unstacked. When plates are added, they must be stacked on top of each other, and when they are removed, they must be unstacked from the top. However, this service also requires that we keep track of the maximum capacity allowed for each stack. 

Implement a `PlateStack` class with the following operations:
- `stack(plates: List[int])`: Accepts a list of integers (the plate identifiers) to stack on top.
- `unstack(n: int)`: Removes n plates from the top of the stack. Returns the list of unstacked plates.
- `setCapacity(capacity: int)`: Sets a new capacity for the stack. If the current number of plates exceeds the new capacity, the stack must be reduced to fit the new limit by unstacking plates from the bottom.

Your class should handle edge cases such as:
- Trying to unstack more plates than are present.
- Setting a capacity lower than the current number of stacked plates.

EXAMPLES:
Input: 
```python
plate_stack = PlateStack()
plate_stack.setCapacity(5)
plate_stack.stack([1, 2, 3, 4])
```
Output: 
```python
None
```
 
Input: 
```python
plate_stack.stack([5, 6, 7])
```
Output: 
```python
None
```
 
Input: 
```python
plate_stack.unstack(4)
```
Output: 
```python
[7, 6, 5, 4]
```
Explanation: After unstacking 4 plates, plate identifiers 7, 6, 5, and 4 are removed.

Input:
```python
plate_stack.setCapacity(3)
```
Output:
```python
None
```
 
Input:
```python
plate_stack.unstack(2)
```
Output:
```python
[3, 2]
```
Explanation: After setting the capacity to 3, the stack must unstack plates 3 and 2 to fit the new limit.

CONSTRAINTS:
- The plate identifiers are unique positive integers.
- The maximum number of plates that can be stacked at any time is 10^5.
- The operations `stack`, `unstack`, and `setCapacity` can be called in any order.

HINT:
Consider using an array to represent the stack and a pointer to manage the top of the stack efficiently.

SOLUTION_PYTHON:
```python
class PlateStack:
    def __init__(self):
        self.stack = []
        self.capacity = float('inf')  # No limit initially

    def stack(self, plates):
        for plate in plates:
            if len(self.stack) < self.capacity:
                self.stack.append(plate)

    def unstack(self, n):
        if n > len(self.stack):
            n = len(self.stack)  # Can't unstack more than available
        unstacked = []
        for _ in range(n):
            unstacked.append(self.stack.pop())
        return unstacked

    def setCapacity(self, capacity):
        self.capacity = capacity
        while len(self.stack) > capacity:
            self.unstack(1)  # Unstack one plate until within capacity

# Example Usage
plate_stack = PlateStack()
plate_stack.setCapacity(3)
plate_stack.stack([1, 2, 3, 4, 5])
print(plate_stack.unstack(2))  # Output: [5, 4]
```

SOLUTION_JS:
```javascript
class PlateStack {
    constructor() {
        this.stack = [];
        this.capacity = Infinity; // No limit initially
    }

    stack(plates) {
        for (let plate of plates) {
            if (this.stack.length < this.capacity) {
                this.stack.push(plate);
            }
        }
    }

    unstack(n) {
        n = Math.min(n, this.stack.length); // Can't unstack more than available
        let unstacked = [];
        for (let i = 0; i < n; i++) {
            unstacked.push(this.stack.pop());
        }
        return unstacked;
    }

    setCapacity(capacity) {
        this.capacity = capacity;
        while (this.stack.length > capacity) {
            this.unstack(1); // Unstack one plate until within capacity
        }
    }
}

// Example Usage
let plateStack = new PlateStack();
plateStack.setCapacity(3);
plateStack.stack([1, 2, 3, 4, 5]);
console.log(plateStack.unstack(2)); // Output: [5, 4]
```

TIME_COMPLEXITY: O(n) for stacking, O(n) for unstacking, O(n) for setting capacity, where n is the number of plates.
SPACE_COMPLEXITY: O(m) where m is the current number of plates in the stack. 

EXPLANATION:
The `PlateStack` class uses a simple array to represent the stack of plates. The `stack` method checks against the capacity before adding plates, while the `unstack` method retrieves and removes plates from the end of the array. The `setCapacity` method ensures that the current stack conforms to the new capacity, effectively unstacking as necessary to meet the limit while also maintaining efficiency with array operations.

FOLLOW_UP:
What modifications would you make to implement a minimum capacity that must be maintained, and how would that affect the logic of `setCapacity`?

# Mise

**Category:** Dev Tools
**Link:** [https://github.com/jdx/mise](https://github.com/jdx/mise)

## What Is It?
Mise is an open-source command-line tool designed to help developers manage and deploy their microservices with ease. It simplifies the process of configuring and orchestrating services, allowing teams to focus on building rather than managing infrastructure.

## The Problem It Solves
Before Mise, developers often struggled with complex setups and deployments of microservices. They typically relied on a mix of scripts, manual processes, and various orchestration tools that lead to inconsistencies, errors, and increased maintenance overhead. Mise addresses these pain points by providing a streamlined approach to managing microservices, reducing the friction involved in deployment.

## Why It's Worth 10 Minutes of Your Time
1. **Simple Configuration**: Mise allows you to define and manage your services using a clear YAML configuration file, making it easy for teams to understand and modify deployments.
2. **Integrated Support for Multiple Environments**: It offers built-in functionality to seamlessly deploy across different environments (development, staging, production), reducing the setup burden on developers.
3. **Rich CLI Features**: The command-line interface is intuitive, with commands for service health checks, logs, and scaling, giving developers the tools to manage services without the need for a web dashboard.

## Quick Start
```bash
# Install Mise globally
brew install jdx/tap/mise

# Initialize a new Mise project
mise init my-microservice

# Deploy your microservice
cd my-microservice
mise deploy
```

## Real-World Example
Here's a simple configuration for a Node.js microservice:
```yaml
services:
  api:
    image: myapi:latest
    ports:
      - 8080:80
    environment:
      NODE_ENV: production

  worker:
    image: myworker:latest
    depends_on:
      - api
```
With this setup, you can easily launch both services with a single command.

## Who Should Use It
✅ Perfect for: Developers and teams working on microservices who want to simplify deployment and management without getting bogged down in complex infrastructure tools.  
⏩ Skip if: You're building single-page applications or monolithic applications where microservice orchestration isn't a priority.

## The Honest Take
Mise is a powerful tool that fills a core gap in microservices management with its simplicity and efficiency. While it may not cover every edge case or advanced configuration, it’s a solid choice for teams looking to streamline their deployment processes. Overall, it's definitely worth a try if you're venturing into microservices.
