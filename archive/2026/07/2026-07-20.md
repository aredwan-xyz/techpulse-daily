## 📄 PagedWeight: Efficient MoE LLM Serving with Dynamic Quality-Aware Weight Quantization

**Authors:** Yuchen Yang, Yifan Zhao, Anisha Dasgupta, Sasa Misailovic et al.
**Published:** 2026-07-17
**arXiv ID:** [2607.16184v1](https://arxiv.org/abs/2607.16184v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2607.16184v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2607.16184v1)

---

## 🎯 The Core Idea
This paper introduces a technique called PagedWeight that optimizes how weights in a mixture-of-experts language model are stored and served in real-time. It dynamically adjusts the precision of these weights based on current demands, helping manage memory usage while maintaining performance.

## 🔍 The Problem They Solved
Previously, large language models that use a mixture-of-experts approach struggled with balancing the memory needs of their model weights against the requirements of their key-value (KV) caches, especially in scenarios where both were heavily used. This led to inefficiencies and potential drops in model accuracy during serving, limiting the effective use of resources.

## 💡 The Key Innovation
PagedWeight introduces a dynamic method of weight quantization, which effectively manages how model weights are represented in memory. By intelligently navigating the tradeoff between the accuracy of the model's predictions and its resource consumption, it allows for better handling of high-demand scenarios and improves overall performance compared to existing methods.

## 🛠️ Why Developers Should Care
With the rise of AI applications requiring real-time processing, the efficiencies offered by PagedWeight can lead to significant improvements in the responsiveness and resource management of AI systems. This means developers can create models that not only run faster but also make better use of available hardware without sacrificing accuracy.

## ⚡ TL;DR
PagedWeight makes language models faster and more efficient by smartly adjusting their memory needs while maintaining performance.

## 💡 Coding Challenge


TITLE: Merge Two Sorted Linked Lists with Duplicates
DIFFICULTY: Easy
TOPIC: Linked Lists
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given two linked lists, `list1` and `list2`, representing two sorted lists. Your task is to merge them into a single sorted linked list. However, if there are duplicates in the resulting merged list, they should be removed such that each element appears only once. The new merged list should also maintain the order of the first occurrences of its elements.

Write a function `merge_and_deduplicate` that takes the heads of the two linked lists and returns the head of the new sorted linked list.

EXAMPLES:
Input: 
```python
list1 = 1 -> 2 -> 2 -> 3
list2 = 2 -> 3 -> 4 -> 4
```
Output: 
```python
1 -> 2 -> 3 -> 4
```
Explanation: The merged list contains all unique elements from both lists sorted in ascending order.

Input: 
```python
list1 = 1 -> 1 -> 2
list2 = 1 -> 3 -> 4
```
Output: 
```python
1 -> 2 -> 3 -> 4
```
Explanation: The merged list again includes no duplicates and is sorted.

CONSTRAINTS:
- The elements in the linked lists are integers.
- Each linked list has at least one node.

HINT:
Consider using a set to keep track of already seen elements while you merge the lists.

SOLUTION_PYTHON:
```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def merge_and_deduplicate(list1: ListNode, list2: ListNode) -> ListNode:
    # Create a dummy node to simplify the merge process
    dummy = ListNode()
    current = dummy
    seen = set()

    while list1 or list2:
        # Get the next value from each list if available
        val1 = list1.val if list1 else float('inf')
        val2 = list2.val if list2 else float('inf')

        # Choose the smallest value
        if val1 < val2:
            next_node = list1
            list1 = list1.next
        elif val1 > val2:
            next_node = list2
            list2 = list2.next
        else:  # If they are equal, take one and move both
            next_node = list1
            list1 = list1.next
            list2 = list2.next

        # Only add unique values to the merged list
        if next_node.val not in seen:
            seen.add(next_node.val)
            current.next = next_node
            current = current.next

        current.next = None  # Mark the end of the list

    return dummy.next

```

SOLUTION_JS:
```javascript
class ListNode {
    constructor(val = 0, next = null) {
        this.val = val;
        this.next = next;
    }
}

function mergeAndDeduplicate(list1, list2) {
    let dummy = new ListNode();
    let current = dummy;
    const seen = new Set();

    while (list1 || list2) {
        let val1 = list1 ? list1.val : Infinity;
        let val2 = list2 ? list2.val : Infinity;

        let nextNode;

        if (val1 < val2) {
            nextNode = list1;
            list1 = list1.next;
        } else if (val1 > val2) {
            nextNode = list2;
            list2 = list2.next;
        } else {
            nextNode = list1;
            list1 = list1.next;
            list2 = list2.next;
        }

        if (!seen.has(nextNode.val)) {
            seen.add(nextNode.val);
            current.next = nextNode;
            current = current.next;
        }
        current.next = null;  // End the list
    }

    return dummy.next;
}
```

TIME_COMPLEXITY: O(n + m)
SPACE_COMPLEXITY: O(k)

EXPLANATION:
The approach utilizes a dummy node to simplify the process of merging and a set to keep track of the unique values. We traverse both linked lists while maintaining a pointer to the current location in the merged list, ensuring that we only add values that have not yet been seen. This allows us to achieve both the merge and deduplication efficiently in linear time with respect to the total number of nodes across both lists.

# Loco

**Category:** Backend
**Link:** [https://github.com/loco-rs/loco](https://github.com/loco-rs/loco)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/loco-rs/loco)

---

## What Is It?
Loco is a lightweight, open-source localization library for Rust, designed to help developers manage and implement internationalization (i18n) in their applications efficiently.

## The Problem It Solves
Before Loco, developers had to rely on manual string management and scattered localization solutions, often leading to complex and error-prone internationalization processes. Loco addresses this by providing a straightforward API and a scalable approach, ensuring that localization is not just easier, but also more maintainable.

## Why It's Worth 10 Minutes of Your Time
1. **Efficient String Management**: Loco allows you to define language strings in a structured way, reducing the risk of typo errors and making it easy to locate and change translations.
2. **Centralized Locale Handling**: With Loco, you can manage multiple languages conveniently, ensuring that your app is easily adaptable for different markets without chaos among resource files.
3. **Rust-Friendly Integration**: As a Rust library, Loco leverages Rust's memory safety and performance, meaning you get a robust localization solution without sacrificing the speed and reliability that Rust is known for.

## Quick Start
```bash
cargo add loco
```

## Real-World Example
```rust
use loco::{Locale, Localizer};

let localizer = Localizer::new("en");

localizer.add("greeting", "Hello, world!");
let translated = localizer.translate("greeting");
println!("{}", translated); // Outputs: Hello, world!
```

## Who Should Use It
✅ Perfect for: Rust developers looking to add internationalization support to their applications quickly and effectively.  
⏩ Skip if: Your project is not focused on international audiences or if you are using a different programming paradigm that doesn’t align with Rust's ecosystem.

## The Honest Take
Loco is a fantastic tool for Rust developers aiming to implement localization with ease and efficiency. Its simplicity and direct approach make it invaluable for i18n tasks, but it may not suit those working in environments where Rust is not applicable. Overall, if you're developing in Rust and need localization, Loco is worth a try!
