## 📄 Which Way Did It Move? Diagnosing and Overcoming Directional Motion Blindness in Video-LLMs

**Authors:** Jongseo Lee, Hyuntak Lee, Sunghun Kim, Sooa Kim et al.
**Published:** 2026-05-21
**arXiv ID:** [2605.22823v1](https://arxiv.org/abs/2605.22823v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2605.22823v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2605.22823v1)

---

## 🎯 The Core Idea
This paper tackles a problem with Video Large Language Models (Video-LLMs) that struggle to accurately interpret the direction of motion in videos. Essentially, they diagnose a common issue called "directional motion blindness," where these models fail to recognize whether an object is moving left, right, up, or down.

## 🔍 The Problem They Solved
Prior to this research, most Video-LLMs seemed to misunderstand basic motion direction, often guessing randomly. This inability to accurately assess motion direction limited their effectiveness in understanding and describing video content.

## 💡 The Key Innovation
The authors pinpoint the exact failure in how these models process motion direction information, identifying a "direction binding gap." By revealing that while motion data is available within the model, it doesn't correctly map this data to the appropriate verbal responses, they provide insightful analysis that could inform future improvements to model design.

## 🛠️ Why Developers Should Care
Understanding this directional motion blindness can guide developers in enhancing Video-LLMs, improving their accuracy and utility in real-world applications, such as video analysis or automated content creation. By addressing this issue, we can build more robust AI systems that truly understand video content and context.

## ⚡ TL;DR
Video AI models often can't tell which way an object is moving, and this paper reveals why, helping us to make them better at understanding video.

---

## 📝 Original Abstract

> Video Large Language Models (Video-LLMs) have made rapid progress on temporal video understanding, yet many fail at a basic perceptual primitive: signed image-plane motion direction. On simple videos of a single object moving left, right, up, or down, most Video-LLMs perform near chance, with above-chance cases largely attributable to prediction biases rather than genuine direction understanding. ## 💡 Coding Challenge


TITLE: Longest Substring with K Distinct Characters
DIFFICULTY: Hard
TOPIC: Sliding Window
ESTIMATED_TIME: 35 minutes

PROBLEM:
You are tasked with processing a stream of character data that comes in as a string. Your goal is to find the longest substring that contains at most K distinct characters. This task is crucial for developing features such as character frequency analysis in text processing. You need to return the longest substring's length that satisfies the condition.

EXAMPLES:
Input: "abcabcbb", K = 2  
Output: 4  
Explanation: The longest substring with at most 2 distinct characters is "abab", which has a length of 4.

Input: "aabbcc", K = 3  
Output: 6  
Explanation: The entire string is valid since it contains 3 distinct characters, thus its length is 6.

CONSTRAINTS:
- The input string will have a length between 1 and 10^5.
- K will be a positive integer less than or equal to the length of the string.

HINT:
Consider maintaining a frequency count of the characters in the current window and adjust your window size based on the number of distinct characters.

SOLUTION_PYTHON:
```python
def longest_substring_k_distinct(s: str, K: int) -> int:
    left, right = 0, 0
    char_count = {}
    max_length = 0

    while right < len(s):
        char_count[s[right]] = char_count.get(s[right], 0) + 1

        while len(char_count) > K:
            char_count[s[left]] -= 1
            if char_count[s[left]] == 0:
                del char_count[s[left]]
            left += 1
        
        max_length = max(max_length, right - left + 1)
        right += 1

    return max_length

# Example usage:
# print(longest_substring_k_distinct("abcabcbb", 2))  # Output: 4
```

SOLUTION_JS:
```javascript
function longestSubstringKDistinct(s, K) {
    let left = 0, right = 0;
    const charCount = {};
    let maxLength = 0;

    while (right < s.length) {
        charCount[s[right]] = (charCount[s[right]] || 0) + 1;

        while (Object.keys(charCount).length > K) {
            charCount[s[left]]--;
            if (charCount[s[left]] === 0) {
                delete charCount[s[left]];
            }
            left++;
        }

        maxLength = Math.max(maxLength, right - left + 1);
        right++;
    }

    return maxLength;
}

// Example usage:
// console.log(longestSubstringKDistinct("abcabcbb", 2));  // Output: 4
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(K)

EXPLANATION:
The algorithm utilizes the sliding window technique by maintaining two pointers, left and right, that define the current substring. As the right pointer expands the window by moving forward, the algorithm checks if the number of distinct characters exceeds K. If it does, the left pointer is incremented until the condition is satisfied again. This results in an optimal time complexity of O(n) as each character in the string is processed a constant number of times.

FOLLOW_UP:
Modify the challenge to find the longest substring that can be formed by removing at most one character while satisfying the original condition of K distinct characters.

# Rye

**Category:** Dev Tools
**Link:** [https://github.com/astral-sh/rye](https://github.com/astral-sh/rye)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](

---

## What Is It?
Rye is a simple yet powerful Python tool that streamlines package management and environment setup by combining the functionality of pip, venv, and pyenv into a cohesive command-line interface.

## The Problem It Solves
Before Rye, developers often juggled multiple tools like pip for package installation, venv for virtual environments, and pyenv for managing Python versions, leading to configuration headaches and inconsistencies across projects. This fragmentation often resulted in wasted time resolving environment issues and ensuring dependencies were correctly set.

## Why It's Worth 10 Minutes of Your Time
1. **Unified Workflow**: Rye integrates the management of Python versions, virtual environments, and dependencies into a single command-line interface, reducing complexity and saving you time.
2. **Automatic Environment Creation**: Rye creates virtual environments on-the-fly based on the specific dependencies of your project, ensuring that you are always working in a clean and compatible environment.
3. **Seamless Dependency Management**: With Rye, you can easily manage your project dependencies in a lock file, which helps avoid conflicts and ensures that you and your team are using the exact same versions of libraries.

## Quick Start
```bash
curl --proto '=https' --tlsv1.2 -sSf https://rye-up.com/get.sh | sh
rye init
rye add requests
```

## Real-World Example
```python
# example.py
import requests

response = requests.get('https://api.github.com')
print(response.json())
```
With Rye, you can quickly install the 'requests' library and run the above script without worrying about environment conflicts.

## Who Should Use It
✅ Perfect for: Python developers looking for a streamlined development process, quick project setup, and consistent environment management.  
⏩ Skip if: You're working on non-Python projects or prefer managing dependencies and environments manually.

## The Honest Take
Rye is a game-changer for Python development, making package and environment management straightforward and efficient. However, it might not fully replace deep-rooted workflows for those already comfortable with their current tooling. Overall, if you're seeking simplicity and speed in your Python projects, give Rye a shot!
