## 📄 Physics Is All You Need? A Case Study in Physicist-Supervised AI Development of Scientific Software

**Authors:** Nhat-Minh Nguyen
**Published:** 2026-05-28
**arXiv ID:** [2605.30353v1](https://arxiv.org/abs/2605.30353v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](

---

## 🎯 The Core Idea
Researchers supervised an AI coding agent to develop a physics software module while closely studying how well it performed under guidance. Over 12 workdays, they recorded how many issues the AI could solve on its own versus those needing human intervention.

## 🔍 The Problem They Solved
Before this study, there was little understanding of how effectively AI coding agents could work in scientific domains without expert oversight. Specifically, it was unclear how often and why these agents might misinterpret problems or fail to provide correct solutions.

## 💡 The Key Innovation
This research illustrates a nuanced interaction between a human expert and an AI system, highlighting not just the successes but also the failures of the AI to accurately address scientific challenges. The study shows that while the AI made progress in coding autonomously, it struggled with deeper, root-cause issues, indicating a limitation in its understanding of complex scientific concepts.

## 🛠️ Why Developers Should Care
For developers building AI systems, especially in specialized fields like science, this research underscores the importance of contextual human supervision. It suggests that while AI can perform many tasks independently, it still requires domain expertise to tackle more complex problems and avoid common pitfalls.

## ⚡ TL;DR
A physicist's supervision revealed that while AI can code effectively, it often misses deeper issues in complex scientific tasks without human guidance.

## 💡 Coding Challenge


TITLE: Intermediate String Buffer
DIFFICULTY: Hard
TOPIC: Stack & Queue
ESTIMATED_TIME: 45 minutes

PROBLEM:
You are tasked with implementing an intermediate string buffer that can efficiently apply multiple operations on a string. The operations include inserting characters, deleting characters, and querying the current string's length. The buffer should support these operations in a way that maintains the characters' order. Given a series of operations, your goal is to return the final string and its length after all operations have been executed.

Each operation will be represented as a tuple:
- ("insert", index, char): Inserts the character `char` into the position `index` of the current string.
- ("delete", index): Deletes the character at position `index`.
- ("query"): Returns the current length of the string.

Implement the `StringBuffer` class that supports the mentioned operations.

EXAMPLES:
Input:
operations = [
    ("insert", 0, 'H'),
    ("insert", 1, 'e'),
    ("insert", 2, 'l'),
    ("insert", 3, 'l'),
    ("insert", 4, 'o'),
    ("delete", 4),
    ("insert", 4, '!')
]
buffer = StringBuffer()
for op in operations:
    buffer.apply_operation(*op)
final_string = buffer.get_string()
length = buffer.query_length()

Output: ("Hell!", 5)
Explanation: After applying all the operations, the final string is "Hell!" with length 5.

Input:
operations = [
    ("insert", 0, 'C'),
    ("insert", 1, 'o'),
    ("insert", 2, 'd'),
    ("delete", 2),
    ("insert", 2, 'e'),
    ("insert", 3, 's'),
    ("insert", 4, 't')
]
buffer = StringBuffer()
for op in operations:
    buffer.apply_operation(*op)
final_string = buffer.get_string()
length = buffer.query_length()

Output: ("Coest", 5)

CONSTRAINTS:
- The maximum length of the buffer can be 10^6 characters.
- The index for insert/delete operations will always be within the valid range of the current string length.

HINT:
Consider using a data structure that allows both fast insertion and deletion operations to maintain efficiency.

SOLUTION_PYTHON:
```python
class StringBuffer:
    def __init__(self):
        self.buffer = []  # Using a list to accumulate characters for easy manipulation

    def apply_operation(self, operation, *args):
        if operation == "insert":
            index, char = args
            self.buffer.insert(index, char)
        elif operation == "delete":
            index = args[0]
            if 0 <= index < len(self.buffer):
                del self.buffer[index]

    def get_string(self):
        return ''.join(self.buffer)  # Joining the list to form the final string

    def query_length(self):
        return len(self.buffer)  # Returning the length of the current buffer

# Example usage:
# operations = [
#     ("insert", 0, 'H'),
#     ("insert", 1, 'e'),
#     ...
# ]
# buffer = StringBuffer()
# for op in operations:
#     buffer.apply_operation(*op)
# print(buffer.get_string())  # Final string
# print(buffer.query_length())  # Length of the final string
```

SOLUTION_JS:
```javascript
class StringBuffer {
    constructor() {
        this.buffer = []; // Using array for the buffer
    }

    applyOperation(operation, ...args) {
        if (operation === "insert") {
            const [index, char] = args;
            this.buffer.splice(index, 0, char);
        } else if (operation === "delete") {
            const [index] = args;
            if (index >= 0 && index < this.buffer.length) {
                this.buffer.splice(index, 1);
            }
        }
    }

    getString() {
        return this.buffer.join(''); // Joining array to get the string
    }

    queryLength() {
        return this.buffer.length; // Length of the buffer
    }
}

// Example usage:
// const buffer = new StringBuffer();
// operations.forEach(op => buffer.applyOperation(...op));
// console.log(buffer.getString()); // Final string
// console.log(buffer.queryLength()); // Length of the final string
```

TIME_COMPLEXITY: O(n) for each insert/delete operation in the worst case
SPACE_COMPLEXITY: O(n)

EXPLANATION:
The solution uses a dynamic array (list in Python and array in JavaScript) to store characters. Inserting and deleting characters can lead to up to O(n) complexity due to the need to shift elements. However, this structure allows for easy appending and efficient querying of the string's length. The final solution efficiently processes commands while maintaining the correct sequence of characters.

FOLLOW_UP:
What if you had to implement a feature to retrieve a substring from the buffer? How would you modify your implementation to support that?

# Drizzle ORM

**Category:** Databases
**Link:** [https://github.com/drizzle-team/drizzle-orm](https://github.com/drizzle-team/drizzle-orm)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/drizzle-team/drizzle-orm)

---

## What Is It?
Drizzle ORM is a lightweight TypeScript ORM designed to provide seamless and type-safe interactions with various databases, allowing developers to work with their data in a more intuitive and efficient way.

## The Problem It Solves
Before Drizzle ORM, developers often struggled with traditional ORMs that might be too heavy or not type-safe, leading to runtime errors or complex queries that were difficult to manage. Drizzle ORM addresses these pain points by offering a clean API and leveraging TypeScript for type safety, enabling developers to catch errors during development rather than at runtime.

## Why It's Worth 10 Minutes of Your Time
1. **Type Safety:** Drizzle ORM’s full TypeScript support means that your queries and models are checked at compile time, significantly reducing the likelihood of errors in production.
2. **Lightweight API:** With a minimalistic approach compared to heavyweight ORMs, you can easily understand and use Drizzle without a steep learning curve, speeding up your development process.
3. **Seamless Database Migrations:** Drizzle simplifies migrations with built-in support, allowing you to easily track schema changes and ensuring that your database stays in sync with your application's data model.

## Quick Start
```bash
npm install drizzle-orm
```

## Real-World Example
```typescript
import { drizzle } from 'drizzle-orm';
import { createConnection } from 'mysql2';
import { table } from 'drizzle-orm/mysql-core';

const connection = createConnection({ /* MySQL config */ });
const db = drizzle(connection);

const users = table('users');

async function fetchUsers() {
    const allUsers = await db.select().from(users).execute();
    console.log(allUsers);
}

fetchUsers();
```

## Who Should Use It
✅ Perfect for: TypeScript developers looking for a simple, type-safe ORM that enhances productivity and reduces the chance of runtime errors.  
⏩ Skip if: You’re working with a less typed language or prefer a more established, robust ORM framework with extensive community support.

## The Honest Take
Drizzle ORM shines with its straightforward API and strong TypeScript integration, which many developers will find refreshing and useful. However, as a newer tool, it might lack some advanced features of its more established counterparts, so those requiring complex ORM functionalities may want to consider this before diving in.
