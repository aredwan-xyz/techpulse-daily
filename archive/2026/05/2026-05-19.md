## 🎯 The Core Idea
The researchers explored how to teach artificial neural networks (ANNs) to recognize patterns in complex data like the human brain does, using simpler learning rules. They evaluated these rules on a specially designed dataset to see how well they help the networks understand hierarchical structures in high-dimensional data.

## 🔍 The Problem They Solved
Prior to this work, it was unclear how neural networks could efficiently learn complex hierarchical patterns without lengthy training times or complicated error computations. The existing methods tended to rely heavily on conventional error backpropagation, which has limitations in speed and flexibility.

## 💡 The Key Innovation
This study introduced two new types of local learning rules that mimic the brain's ability to learn without needing full error backpropagation. The first method uses direct feedback for learning, while the second applies self-supervised loss functions that don’t rely on output-layer error approximations, allowing for efficient learning of hierarchical data structures.

## 🛠️ Why Developers Should Care
By employing these new local learning rules, developers can create AI systems that learn faster and more efficiently without the computational overhead of traditional error propagation methods. This could lead to improved performance in tasks involving complex data representations, making AI more scalable and adaptable in real-world applications.

## ⚡ TL;DR
New learning rules for neural networks may help them grasp complex patterns in data as efficiently as our brains do, without getting bogged down by cumbersome training processes.

## 💡 Coding Challenge


TITLE: Time Series Anomaly Detection
DIFFICULTY: Hard
TOPIC: AI/ML Coding (NumPy/Pandas)
ESTIMATED_TIME: 60 minutes

PROBLEM:
You are given a time series dataset representing daily stock prices of a company over a period of time. Your task is to implement an anomaly detection algorithm that identifies significant deviations from the expected trend. You need to compute the rolling mean and rolling standard deviation over a specified window size. Any data point that falls outside 2 standard deviations from the rolling mean is considered an anomaly.

Write a function that takes in the time series data and the window size and returns a list of indices where anomalies occur.

EXAMPLES:
Input: 
```python
data = [100, 102, 101, 103, 200, 102, 101, 99, 98, 100]
window_size = 3
```
Output: 
```python
[4]
```
Explanation: The value 200 at index 4 is more than 2 standard deviations away from the rolling mean of the previous window [100, 102, 101]. 

Input: 
```python
data = [50, 51, 52, 53, 54, 50, 200, 52, 51]
window_size = 4
```
Output: 
```python
[6]
```
Explanation: The value 200 at index 6 is an anomaly as it is significantly greater than the rolling mean of the prior indices.

CONSTRAINTS:
- The length of the input data will be between 10 to 10^5.
- The window size will be a positive integer smaller than the length of the data.

HINT:
Consider using Pandas' rolling functionality to help compute the rolling mean and standard deviation efficiently.

SOLUTION_PYTHON:
```python
import pandas as pd

def detect_anomalies(data, window_size):
    # Convert the list into a Pandas DataFrame for easier manipulation
    df = pd.DataFrame(data, columns=['Price'])
    
    # Calculate the rolling mean and rolling standard deviation
    df['RollingMean'] = df['Price'].rolling(window=window_size).mean()
    df['RollingStd'] = df['Price'].rolling(window=window_size).std()
    
    # Detect anomalies by comparing the data points with the rolling mean & stddev
    anomalies = df[(df['Price'] > df['RollingMean'] + 2 * df['RollingStd']) | 
                   (df['Price'] < df['RollingMean'] - 2 * df['RollingStd'])]
    
    # Return the indices of the anomalies
    return anomalies.index.tolist()

# Time Complexity: O(n) where n is the number of data points
# Space Complexity: O(n) for storing the DataFrame
```

SOLUTION_JS:
```javascript
function detectAnomalies(data, windowSize) {
    let anomalies = [];
    
    for (let i = windowSize; i < data.length; i++) {
        // Get the current window
        const window = data.slice(i - windowSize, i);
        
        // Calculate mean and standard deviation for the window
        const mean = window.reduce((a, b) => a + b, 0) / windowSize;
        const stdDev = Math.sqrt(window.reduce((sq, n) => sq + (n - mean) ** 2, 0) / windowSize);
        
        // Check if the current point is an anomaly
        if (data[i] > (mean + 2 * stdDev) || data[i] < (mean - 2 * stdDev)) {
            anomalies.push(i);
        }
    }
    
    return anomalies;
}

// Time Complexity: O(n * m) where n is the total number of data points and m is the window size
// Space Complexity: O(1) - we only return indices
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(n)

EXPLANATION:
The proposed approach efficiently calculates the rolling mean and rolling standard deviation using a sliding window technique, allowing for quick assessment of anomalies in the dataset. By traversing the array once, while maintaining the necessary statistics, it achieves linear time complexity which is optimal for large datasets.

FOLLOW_UP:
How would you modify your implementation to consider anomalies as those deviations that occur in a sequence (e.g., 3 consecutive values) instead of just a single point?

# Instructor

**Category:** AI/LLM
**Link:** [https://github.com/jxnl/instructor](https://github.com/jxnl/instructor)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/jxnl/instructor)

---

## What Is It?
Instructor is an open-source framework designed to help developers easily fine-tune large language models (LLMs) for specific tasks using a simplified interface and streamlined pipeline.

## The Problem It Solves
Before Instructor, developers were often bogged down by the complexity of configuring and fine-tuning LLMs for their unique needs. Many faced challenges with boilerplate code, lack of clear methodologies, and the sheer resource demand required to train models effectively. Instructor addresses these headaches by providing a coherent and user-friendly environment for customization and experimentation.

## Why It's Worth 10 Minutes of Your Time
1. **User-Friendly Interface**: Instructor boasts a clean and concise interface that simplifies the fine-tuning process, allowing you to focus more on your specific use case rather than the nuances of the underlying code.
2. **Custom Task Templates**: It offers out-of-the-box templates for common LLM tasks like text classification or summarization, which speeds up implementation and reduces the learning curve significantly.
3. **Efficient Resource Management**: The tool optimizes resource allocation during training, helping developers to effectively leverage their computational resources without unnecessary overhead.

## Quick Start
```bash
git clone https://github.com/jxnl/instructor
cd instructor
pip install -r requirements.txt
python train.py --task classification --data your_dataset.json
```

## Real-World Example
```python
from instructor import Instructor

# Initialize the instructor for text classification
instructor = Instructor(task='classification')
# Load the training dataset
instructor.load_data('data/train.json')
# Start fine-tuning the model
instructor.train(model='gpt-3')
```

## Who Should Use It
✅ Perfect for: Developers working on custom use cases for LLMs, small to medium-sized teams wanting to get up and running quickly with AI solutions, and researchers needing a straightforward way to prototype experiments.  
⏩ Skip if: You require an enterprise-grade, highly scalable framework, or if you're already comfortable with extensive and complex LLM training setups.

## The Honest Take
Instructor excels in making LLM fine-tuning accessible, providing a robust framework for those looking to customize models without getting lost in the details. However, while it's fantastic for prototyping and smaller projects, it may not cater to large-scale enterprise needs where deeper customization is necessary. Overall, if you're looking to dip your toes into LLM fine-tuning, it's definitely worth a try.
