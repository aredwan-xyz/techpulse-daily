## 📄 CoupleEvo: Evolving Heuristics for Coupled Optimization Problems Using Large Language Models

**Authors:** Thomas Bömer, Bastian Amberg, Max Disselnmeyer, Anne Meyer et al.
**Published:** 2026-05-07
**arXiv ID:** [2605.06341v1](https://arxiv.org/abs/2605.06341v1)

---

## 🎯 The Core Idea
This paper presents a new method called CoupleEvo that helps solve complex problems with multiple interconnected parts by using large language models. It does this by evolving tailored strategies for optimizing each part of the problem, ensuring they work well together.

## 🔍 The Problem They Solved
Previously, approaches that utilized large language models for optimization were mostly limited to single problems and couldn't effectively handle situations where multiple subproblems were interdependent. This meant many real-world scenarios remained challenging and time-consuming to optimize, as solutions for one part could negatively impact another.

## 💡 The Key Innovation
CoupleEvo introduces three distinct strategies for evolving heuristics: a sequential method that focuses on one subproblem at a time, an iterative method that cycles through subproblems, and an integrated method that optimizes all subproblems together. This flexibility allows it to adapt and find more effective solutions across various interconnected aspects of complex optimization tasks.

## 🛠️ Why Developers Should Care
By implementing CoupleEvo, developers can streamline their approach to solving multifaceted optimization problems, leading to more efficient and effective solutions. This could drastically cut down on the time and effort needed to devise strategies for scenarios where different components influence each other.

## ⚡ TL;DR
A new tool called CoupleEvo helps solve complex, interconnected problems by tailoring optimization strategies for each part, ensuring they work together effectively.

## 💡 Coding Challenge


TITLE: Longest Distinct Subsection
DIFFICULTY: Hard
TOPIC: Sliding Window
ESTIMATED_TIME: 40 minutes

PROBLEM:
You are given a string that represents a sequence of customer orders in a restaurant. Each character in the string represents a different type of dish. The restaurant wants to find the longest contiguous subsection of the string where all the dish types are unique. Your task is to return the length of this longest subsection. Keep in mind that the restaurant must optimize for a wild rush of customers, so efficiency is key!

EXAMPLES:
Input: "PABCDEFAB"
Output: 6
Explanation: The longest distinct subsection is "ABCDEF" which has a length of 6.

Input: "AAABBC"
Output: 3
Explanation: The longest distinct subsection is "ABC" which has a length of 3.

CONSTRAINTS:
- The input string will have a length of at least 1 and at most 10^5.
- The string will only contain uppercase English letters (A-Z).

HINT:
Consider maintaining a sliding window that extends until you hit a duplicate dish type.

SOLUTION_PYTHON:
```python
def longest_distinct_subsection(s: str) -> int:
    char_index = {}
    max_length = 0
    left = 0
    
    for right in range(len(s)):
        if s[right] in char_index:
            left = max(left, char_index[s[right]] + 1)  # Move left to avoid duplicates
        
        char_index[s[right]] = right  # Update last seen index of the character
        max_length = max(max_length, right - left + 1)  # Update max length
        
    return max_length

# Time complexity: O(n), where n is the length of the string.
# Space complexity: O(min(m, n)), where m is the size of the character set.
```

SOLUTION_JS:
```javascript
function longestDistinctSubsection(s) {
    let charIndex = {};
    let maxLength = 0;
    let left = 0;

    for (let right = 0; right < s.length; right++) {
        if (charIndex[s[right]] !== undefined) {
            left = Math.max(left, charIndex[s[right]] + 1); // Move left to avoid duplicates
        }

        charIndex[s[right]] = right; // Update last seen index of the character
        maxLength = Math.max(maxLength, right - left + 1); // Update max length
    }

    return maxLength;
}

// Time complexity: O(n), where n is the length of the string.
// Space complexity: O(min(m, n)), where m is the size of the character set.
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(min(m, n))

EXPLANATION:
The solution employs a sliding window approach with two pointers: `left` and `right`. By maintaining a hashmap that records the last seen index of each character, when a duplicate is encountered, we can adjust the `left` pointer to effectively "slide" the window forward. This way, we ensure that every character within the window is unique, ensuring optimal length calculation as we iterate through the string.
