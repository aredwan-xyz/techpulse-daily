**Authors:** Yingshan Susan Wang, Linlu Qiu, Zhaofeng Wu, Roger P. Levy et al.

---

## 🎯 The Core Idea
This paper investigates whether language models can distinguish between grammatically correct and incorrect sentences beyond simply predicting the likelihood of sequences of words. The researchers trained a tool to analyze internal representations of these models to see if they implicitly learn grammaticality.

## 🔍 The Problem They Solved
Previously, it was unclear if language models were simply generating sentences based on what is probable or if they truly understood linguistic rules that define grammaticality. The study reveals that while language models can produce correct sentences, their method for evaluating grammatical structures was not well understood.

## 💡 The Key Innovation
The authors devised a method to explore the internal workings of language models by applying specific changes to natural sentences and checking how the models reacted. This approach demonstrated that language models are capable of developing a sense of grammatical rules independent of just string probability, revealing a deeper understanding of language structure.

## 🛠️ Why Developers Should Care
Understanding that language models can discern grammaticality beyond simple likelihood may improve how we design AI that interacts with human language, enhancing tasks like translation and content generation. This insight could lead to more robust language models that handle complex linguistic structures more effectively.

## ⚡ TL;DR
Language models can learn grammar rules on their own, not just predict likely word sequences.

## 💡 Coding Challenge


TITLE: Inventory Tracker
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are building an inventory tracking system for a small warehouse. Each item in the warehouse has a unique ID and a quantity indicating how many units of that item are available. Your task is to create a class `Inventory` that supports the following operations:

1. `add_item(item_id: str, quantity: int)`: Add a specified quantity of the item with the given ID to the inventory. If the item already exists in the inventory, update its quantity.
2. `remove_item(item_id: str, quantity: int)`: Remove a specified quantity of the item with the given ID from the inventory. If the quantity to remove exceeds the available quantity, set the quantity to zero and remove the item from the inventory.
3. `get_inventory() -> List[Tuple[str, int]]`: Return the current inventory as a list of tuples, each containing the item ID and its quantity, sorted in ascending order by item ID.

EXAMPLES:
Input:
```python
inventory = Inventory()
inventory.add_item("item1", 10)
inventory.add_item("item2", 5)
inventory.remove_item("item1", 3)
output = inventory.get_inventory()
```
Output:
[("item1", 7), ("item2", 5)]
Explanation: "item1" has 7 remaining and "item2" has 5.

Input:
```python
inventory.add_item("item3", 20)
inventory.remove_item("item2", 10)
output = inventory.get_inventory()
```
Output:
[("item1", 7), ("item3", 20)]
Explanation: "item2" was removed since all units were taken out.

CONSTRAINTS:
- Item IDs are unique strings.
- The quantity added or removed will always be a non-negative integer.

HINT:
Consider using a dictionary to track the quantities of items easily and efficiently.

SOLUTION_PYTHON:
```python
class Inventory:
    def __init__(self):
        self.items = {}  # A dictionary to store item_id and its quantity.

    def add_item(self, item_id: str, quantity: int):
        # Add item quantity, if item exists update the quantity.
        if item_id in self.items:
            self.items[item_id] += quantity
        else:
            self.items[item_id] = quantity

    def remove_item(self, item_id: str, quantity: int):
        if item_id in self.items:
            if self.items[item_id] <= quantity:
                del self.items[item_id]  # Remove item if quantity is depleted.
            else:
                self.items[item_id] -= quantity

    def get_inventory(self):
        # Return sorted list of items by item_id.
        return sorted(self.items.items())
```

SOLUTION_JS:
```javascript
class Inventory {
    constructor() {
        this.items = {};  // Object to store item_id with its quantity.
    }

    add_item(item_id, quantity) {
        if (this.items[item_id]) {
            this.items[item_id] += quantity; // Update existing item's quantity.
        } else {
            this.items[item_id] = quantity; // Add new item to the inventory.
        }
    }

    remove_item(item_id, quantity) {
        if (this.items[item_id]) {
            if (this.items[item_id] <= quantity) {
                delete this.items[item_id]; // Remove item if depleted.
            } else {
                this.items[item_id] -= quantity; // Reduce item quantity.
            }
        }
    }

    get_inventory() {
        // Sort items by item_id and return as an array of arrays.
        return Object.entries(this.items).sort((a, b) => a[0].localeCompare(b[0]));
    }
}
```

TIME_COMPLEXITY: O(n log n) for sorting the inventory in `get_inventory()`, O(1) for add and remove operations.
SPACE_COMPLEXITY: O(n) where n is the number of unique items in the inventory.

EXPLANATION:
The `Inventory` class uses a hash map (dictionary in Python, object in JavaScript) to efficiently track the quantity of each item based on its unique ID. Adding or removing items is done in constant time on average, while generating the inventory list involves sorting based on item IDs. This approach is optimal for both the operations we need to support and the space-tracking of items.

FOLLOW_UP:
Modify the `Inventory` class to include a method `get_total_items()`, which returns the total number of unique items currently in stock.

# Drizzle ORM

**Category:** Databases
**Link:** [https://github.com/drizzle-team/drizzle-orm](https://github.com/drizzle-team/drizzle-orm)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/drizzle-team/drizzle-orm)

---

## What Is It?
Drizzle ORM is a TypeScript/JavaScript Object-Relational Mapping (ORM) library designed to simplify database interactions by providing a clean and type-safe way to work with SQL databases.

## The Problem It Solves
Before Drizzle ORM, developers often struggled with boilerplate code and type errors while using traditional ORMs or writing raw SQL queries. This inconsistency could lead to runtime errors and increased debugging time. Drizzle ORM fixes this pain by offering a type-safe interface that reduces the likelihood of errors and facilitates smoother data manipulation.

## Why It's Worth 10 Minutes of Your Time
1. **Type Safety**: Thanks to TypeScript support, Drizzle ORM ensures your queries are type-checked at compile time, reducing runtime errors and improving code reliability.
2. **Fluid API**: The library provides an intuitive, fluent API that makes it easier to perform complex queries without the overhead of boilerplate code, leading to cleaner and more readable codebases.
3. **Lightweight and Modular**: Drizzle ORM is designed to be lightweight and modular, allowing you to only include the parts you need, which is great for performance and minimizing bundle size.

## Quick Start
```bash
npm install drizzle-orm
```

## Real-World Example
```typescript
import { drizzle } from 'drizzle-orm';
import { sql } from 'drizzle-orm/sql';

const db = drizzle(yourDatabaseConnection);

const users = await db.select().from('users').where(sql`age > 18`).execute();
console.log(users);
```

## Who Should Use It
✅ Perfect for: Developers who leverage TypeScript for database interactions, startups looking to avoid heavy and convoluted ORM solutions, or projects that need a simple, lightweight, and elegant way to interact with databases.  
⏩ Skip if: You prefer raw SQL for maximum control, or if you're working on projects where performance overhead from ORMs is critical.

## The Honest Take
Drizzle ORM offers a fantastic middle ground between raw SQL and traditional ORMs, making it a great choice for TypeScript developers. While its modular design is a huge plus, those used to very specific features of other ORMs might find some functionalities lacking. Ultimately, it’s a solid tool to improve your database interactions with minimal hassle.
