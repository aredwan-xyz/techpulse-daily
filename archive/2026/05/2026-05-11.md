## 📄 Normalizing Trajectory Models

**Authors:** Jiatao Gu, Tianrong Chen, Ying Shen, David Berthelot et al.
**Published:** 2026-05-08
**arXiv ID:** [2605.08078v1](https://arxiv.org/abs/2605.08078v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2605.08078v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2605.08078v1)

---

## 🎯 The Core Idea
The authors developed a new type of model called Normalizing Trajectory Models (NTM) that improves how we generate data by breaking down the process into clearer steps, allowing for better accuracy and efficiency. By using normalized flows in each step, NTMs can better estimate the likelihood of generating those data sequences, enhancing their overall performance.

## 🔍 The Problem They Solved
Previous models had trouble accurately generating data when they used a simplified approach of only a few steps, leading to poorer quality and less reliable results. Existing solutions attempted to fix this by changing the training approach, but they ended up sacrificing the ability to accurately measure how likely the generated data was.

## 💡 The Key Innovation
NTMs innovatively utilize conditional normalizing flows for each sampling step, ensuring that the model can precisely measure and learn from the likelihood of each generated outcome. This allows NTMs to maintain high fidelity in data generation while retaining the advantages of being trained from scratch or fine-tuned from pre-existing models.

## 🛠️ Why Developers Should Care
NTMs provide a powerful framework that can enhance the reliability and quality of AI systems that rely on data generation, making them particularly valuable for applications like image synthesis and natural language generation. By understanding how to implement these models, developers can create more robust systems that perform better with less data.

## ⚡ TL;DR
A new model called Normalizing Trajectory Models improves data generation by accurately measuring likelihood across multiple sampling steps, leading to better quality results.

---

## 📝 Original Abstract

> Diffusion-based models decompose sampling into many small Gaussian denoising steps -- an assumption that breaks down when generation is compressed to a few coarse transitions. Existing few-step methods address this through distillation, consistency training, or adversarial objectives, but sacrifice the likelihood framework in the process. We introduce Normalizing Trajectory Models (NTM), which models each reverse step as an expressive conditional normalizing flow with exact likelihood training. Architecturally, NTM combines shallow invertible blocks within each step with a deep parallel predictor across the trajectory, forming an end-to-end network trainable from scratch or initializable from pretrained flow-matching models. ## 💡 Coding Challenge


TITLE: Merge Two Sorted Linked Lists into One Sorted List
DIFFICULTY: Easy
TOPIC: Linked Lists
ESTIMATED_TIME: 20 minutes

PROBLEM:
You are given two sorted linked lists, each representing a sequence of integers. Your task is to merge these two linked lists into one sorted linked list. The resulting linked list should contain all the elements from both lists, sorted in ascending order. Implement this in a function that takes the heads of both linked lists as input and returns the head of the newly merged linked list.

EXAMPLES:
Input: 
List 1: 1 -> 3 -> 5
List 2: 2 -> 4 -> 6
Output: 1 -> 2 -> 3 -> 4 -> 5 -> 6
Explanation: The two lists are merged into a single list with all elements in sorted order.

Input:
List 1: 1 -> 2 -> 3
List 2: 4 -> 5 -> 6
Output: 1 -> 2 -> 3 -> 4 -> 5 -> 6
Explanation: Since the first list ends before the second, all elements of the first are followed by all elements of the second.

CONSTRAINTS:
- The input linked lists may have a length of 0 to 10^4.
- The elements in the linked lists are integers, and may be duplicate values.

HINT:
Consider using a dummy node to simplify merging the two linked lists.

SOLUTION_PYTHON:
```python
class ListNode:
    def __init__(self, value=0, next=None):
        self.value = value
        self.next = next

def mergeTwoLists(l1, l2):
    # Create a dummy node to act as the starting point of the merged list
    dummy = ListNode()
    current = dummy

    # Traverse both lists
    while l1 and l2:
        if l1.value < l2.value:
            current.next = l1
            l1 = l1.next
        else:
            current.next = l2
            l2 = l2.next
        current = current.next

    # Attach the remaining elements from l1 or l2
    if l1:
        current.next = l1
    else:
        current.next = l2

    # Return the merged list, which starts from the next of dummy
    return dummy.next

# Time Complexity: O(n + m) where n and m are the lengths of the two lists
# Space Complexity: O(1) since we are merging in place
```

SOLUTION_JS:
```javascript
class ListNode {
    constructor(value = 0, next = null) {
        this.value = value;
        this.next = next;
    }
}

function mergeTwoLists(l1, l2) {
    // Create a dummy node to act as the start point of the merged list
    let dummy = new ListNode();
    let current = dummy;

    // Traverse both lists
    while (l1 && l2) {
        if (l1.value < l2.value) {
            current.next = l1;
            l1 = l1.next;
        } else {
            current.next = l2;
            l2 = l2.next;
        }
        current = current.next;
    }

    // Attach the remaining elements from l1 or l2
    current.next = l1 ? l1 : l2;

    // Return the merged list, which starts from the next of dummy
    return dummy.next;
}

// Time Complexity: O(n + m) where n and m are the lengths of the two lists
// Space Complexity: O(1) since we are merging in place
```

TIME_COMPLEXITY: O(n + m)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach utilizes a dummy node to simplify the merging process. By comparing the heads of both linked lists, we can systematically append the smaller node to the merged list. This operation is repeated until we exhaust one of the lists. Finally, we can directly attach the remaining elements of the other list, making this solution efficient with O(n + m) time complexity while only using O(1) extra space.

# Encore

**Category:** Backend
**Link:** [https://github.com/encoredev/encore](https://github.com/encoredev/encore)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/encoredev/encore)

---

## What Is It?
Encore is an open-source backend development framework that streamlines the process of building full-featured web applications by providing a set of opinionated tools and libraries that manage repetitive tasks.

## The Problem It Solves
Before Encore, developers frequently faced the cumbersome process of configuring and managing repetitive backend tasks such as database setup, API routing, and authentication. This often resulted in boilerplate code and a steep learning curve for new team members. Encore simplifies this by handling these tasks out of the box, allowing developers to focus on writing application logic.

## Why It's Worth 10 Minutes of Your Time
1. **Rapid Development:** Encore offers built-in features for common tasks, like database management and integrated authentication, allowing developers to spin up a new project in minutes without wrestling with configurations.
2. **Serverless Functions Support:** With native support for serverless functions, Encore makes it easy to deploy and scale applications without the overhead of managing infrastructure, which is ideal for projects with fluctuating traffic.
3. **Type Safety with Go:** Built with Go, Encore enables type-safe programming, helping you minimize runtime errors and enhance maintainability, especially when working in larger teams or on complex applications.

## Quick Start
```bash
curl -sSL https://get.encore.dev | sh
encore new my-app
cd my-app
encore dev
```

## Real-World Example
```go
package main

import (
    "encore.dev"
)

// User is a simple struct representing a user in the application.
type User struct {
    ID   string `encore:"id,primary"`
    Name string `encore:"name"`
}

// CreateUser creates a new user in the database
func CreateUser(name string) (*User, error) {
    user := &User{Name: name}
    return encore.Create(user)
}
```

## Who Should Use It
✅ Perfect for: Developers looking to quickly prototype or build web applications using Go without the overhead of traditional backend setup.  
⏩ Skip if: You're working on a highly specialized or complex backend system that does not align with Encore's opinionated approach.

## The Honest Take
Encore significantly reduces the friction in backend development, making it a fantastic choice for teams looking to accelerate their project timelines. However, its opinionated nature may not suit every use case, particularly when custom configurations are needed. Overall, if you're looking for a straightforward way to build applications with minimum setup, Encore is definitely worth a try!
