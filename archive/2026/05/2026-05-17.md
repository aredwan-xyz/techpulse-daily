## 📄 ATLAS: Agentic or Latent Visual Reasoning? One Word is Enough for Both

**Authors:** Ziyu Guo, Rain Liu, Xinyan Chen, Pheng-Ann Heng et al.
**Published:** 2026-05-14
**arXiv ID:** [2605.15198v1](https://arxiv.org/abs/2605.15198v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2605.15198v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2605.15198v1)

---

## 🎯 The Core Idea
The authors proposed ATLAS, a framework that uses a single word called a "functional token" to streamline visual reasoning by combining two common methods: agentic (using tools or code) and latent (using hidden representations). This approach aims to make visual reasoning more efficient and easier to train.

## 🔍 The Problem They Solved
Previously, visual reasoning systems faced challenges due to computational costs and complexity, especially when generating images or executing external tools, which slowed down performance. Additionally, existing methods struggled with generalization across different tasks and were often difficult to train in a parallel fashion.

## 💡 The Key Innovation
ATLAS introduces the idea of a functional token that encapsulates the operations necessary for both agentic and latent reasoning in a compact format. This method allows the system to maintain efficiency by reducing the need for context-switching and enabling better training dynamics, while still being capable of handling diverse reasoning tasks.

## 🛠️ Why Developers Should Care
By adopting the ATLAS framework, developers can create more efficient and scalable AI systems for visual reasoning that require less computational power and training time. This means faster development cycles and the ability to deploy AI solutions that are adaptable to various tasks.

## ⚡ TL;DR
ATLAS uses a single functional token to make visual reasoning faster and easier by combining different reasoning methods into one efficient framework.

---

## 📝 Original Abstract

> Visual reasoning, often interleaved with intermediate visual states, has emerged as a promising direction in the field. A straightforward approach is to directly generate images via unified models during reasoning, but this is computationally expensive and architecturally non-trivial. Recent alternatives include agentic reasoning through code or tool calls, and latent reasoning with learnable hidden embeddings. However, agentic methods incur context-switching latency from external execution, while latent methods lack task generalization and are difficult to train with autoregressive parallelization. ## 💡 Coding Challenge


TITLE: Product Delivery Scheduling  
DIFFICULTY: Medium  
TOPIC: Greedy Algorithms  
ESTIMATED_TIME: 30 minutes  

PROBLEM:  
You are responsible for scheduling deliveries of products to multiple customers in a city. Each customer has a specific time window during which they are available to receive the delivery. You want to maximize the number of deliveries you can make by selecting a schedule within these time windows. Each customer’s availability is represented as a start time and an end time. You can only deliver to one customer at a time.

Given a list of time windows for the customers, return the maximum number of deliveries you can schedule successfully.

EXAMPLES:  
Input: `[(1, 4), (2, 3), (5, 6), (7, 8), (3, 5)]`  
Output: `4`  
Explanation: You can deliver to customers in the order of: (1, 4), (5, 6), (7, 8), and either (2, 3) or (3, 5) (choose only one).

Input: `[(1, 2), (2, 3), (3, 4), (4, 5)]`  
Output: `4`  
Explanation: You can deliver to all customers since each delivery ends when the next can start.

CONSTRAINTS:  
- The number of customers will be in the range of [1, 1000].  
- Each time window will have a start and end time in the range of [0, 24].  

HINT:  
Try to sort the time windows based on their end times to optimize your deliveries.

SOLUTION_PYTHON:
```python
def max_deliveries(time_windows):
    # Sort the time windows by their end time
    time_windows.sort(key=lambda x: x[1])
    
    count = 0
    last_end_time = -1
    
    for start, end in time_windows:
        if start >= last_end_time:  # If we can deliver to this customer
            count += 1
            last_end_time = end  # Update last end time to current end time
    
    return count

# Time and Space Complexity
# TIME_COMPLEXITY: O(n log n) due to sorting
# SPACE_COMPLEXITY: O(1) for the count and last_end_time variables
```

SOLUTION_JS:
```javascript
function maxDeliveries(timeWindows) {
    // Sort the time windows by their end time
    timeWindows.sort((a, b) => a[1] - b[1]);
    
    let count = 0;
    let lastEndTime = -1;
    
    for (let [start, end] of timeWindows) {
        if (start >= lastEndTime) { // If we can deliver to this customer
            count++;
            lastEndTime = end; // Update last end time to current end time
        }
    }
    
    return count;
}

// Time and Space Complexity
// TIME_COMPLEXITY: O(n log n) due to sorting
// SPACE_COMPLEXITY: O(1) for the count and lastEndTime variables
```

TIME_COMPLEXITY: O(n log n)  
SPACE_COMPLEXITY: O(1)  

EXPLANATION:  
The problem effectively translates to the classic "Activity Selection" problem where we choose the maximum number of non-overlapping activities. By sorting the time windows based on their end times, we can efficiently check which deliveries can fit into our schedule, allowing us to maximize the total number. This greedy approach ensures that we always select the earliest finishing delivery first, making room for potential subsequent deliveries.

FOLLOW_UP:  
How would your approach change if each delivery takes a fixed amount of time, such as 1 hour, instead of being instantaneous?

# Starship

**Category:** CLI
**Link:** [https://github.com/starship/starship](https://github.com/starship/starship)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/starship/starship)

---

## What Is It?
Starship is a highly customizable prompt for your shell that brings a sleek and informative interface right to your terminal, no matter the programming language or framework you're using.

## The Problem It Solves
Before Starship, many developers found themselves stuck with basic command-line prompts that lacked critical context about their environment—often needing additional commands or plugins to gain insight into their current project. Customization options for traditional prompts could be cumbersome and tedious, leading to a less efficient workflow.

## Why It's Worth 10 Minutes of Your Time
1. **Cross-Platform Compatibility**: Starship works seamlessly with various shells, including Bash, Zsh, Fish, and PowerShell, which means you can maintain a consistent experience regardless of your development environment.
2. **Instant Customization**: With a simple configuration file, you can personalize your prompt with colors, symbols, and information that matter most to you—whether that's Git status, Node.js version, or even the current Python virtual environment.
3. **Speed & Performance**: Starship prioritizes performance, loading quickly and displaying information without significant delays, improving your overall terminal experience without taxing system resources.

## Quick Start
```bash
# Install Starship
sh -c "$(curl -fsSL https://starship.rs/install.sh)"

# Configure your shell to use Starship
echo 'eval "$(starship init zsh)"' >> ~/.zshrc
```

## Real-World Example
```bash
# Sample configuration in ~/.config/starship.toml
[git_branch]
format = "[$branch]($style) "
style = "bold green"

[python]
format = "via [$env]($style) "
style = "cyan"
```

## Who Should Use It
✅ Perfect for: Developers looking for a modern, informative command-line interface that boosts productivity and provides immediate context about their coding environment.  
⏩ Skip if: You prefer minimal or static prompts and do not wish to customize or configure your terminal experience.

## The Honest Take
Starship delivers a sleek and powerful solution for command-line prompts, enhancing productivity with real-time insights. However, some may find the initial configuration daunting; still, the benefits far outweigh any onboarding challenges, making it a must-try for terminal enthusiasts.
