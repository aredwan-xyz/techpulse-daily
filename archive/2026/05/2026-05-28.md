## 💡 Coding Challenge


TITLE: Build Positive Integer Combinations
DIFFICULTY: Medium
TOPIC: Recursion & Backtracking
ESTIMATED_TIME: 30 minutes

PROBLEM:
Given a target positive integer `N`, create a function that returns all unique combinations of positive integers that sum up to `N`. Each number can be used multiple times in the combination. The output should consist of combinations sorted in non-decreasing order, and the combinations themselves should be sorted lexicographically.

EXAMPLES:
Input: `N = 5`
Output: `[[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]`
Explanation: Here, combinations like [1, 1, 1, 1, 1] indicate that we use five 1s. Other combinations utilize different integers while summing up to 5.

Input: `N = 4`
Output: `[[1, 1, 1, 1], [1, 1, 2], [1, 3], [2, 2], [4]]`

CONSTRAINTS:
- `1 <= N <= 50`

HINT:
Consider using a backtracking approach to explore all potential combinations, where you maintain a running sum and a current combination as you delve deeper.

SOLUTION_PYTHON:
```python
def combination_sum(N):
    result = []
    
    def backtrack(current_combination, remaining_sum, start):
        if remaining_sum == 0:
            result.append(current_combination.copy())
            return
        for num in range(start, N + 1):
            if remaining_sum - num >= 0:
                current_combination.append(num)
                backtrack(current_combination, remaining_sum - num, num)
                current_combination.pop()

    backtrack([], N, 1)
    return result

# Example usage
print(combination_sum(5))  # Output: [[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]
```

SOLUTION_JS:
```javascript
function combinationSum(N) {
    const result = [];
    
    function backtrack(currentCombination, remainingSum, start) {
        if (remainingSum === 0) {
            result.push([...currentCombination]);
            return;
        }
        for (let num = start; num <= N; num++) {
            if (remainingSum - num >= 0) {
                currentCombination.push(num);
                backtrack(currentCombination, remainingSum - num, num);
                currentCombination.pop();
            }
        }
    }

    backtrack([], N, 1);
    return result;
}

// Example usage
console.log(combinationSum(5));  // Output: [[1, 1, 1, 1, 1], [1, 1, 1, 2], [1, 2, 2], [1, 1, 3], [2, 3], [5]]
```

TIME_COMPLEXITY: O(2^N)
SPACE_COMPLEXITY: O(N)

EXPLANATION:
This problem utilizes recursion and backtracking to explore all valid combinations. By starting from the number `1`, each path is incrementally built based on remaining sums. Each time `remaining_sum` hits zero, a valid combination is found, which is added to the results. Thus, the exploration grows exponentially in terms of potential combinations, leading to a time complexity of O(2^N).

FOLLOW_UP:
What changes would be necessary if you wanted to ensure that the integers used in combinations must be distinct?

## 🛠️ Tool Spotlight


# DuckDB

**Category:** Databases
**Link:** [https://github.com/duckdb/duckdb](https://github.com/duckdb/duckdb)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/duckdb/duckdb)

---

## What Is It?
DuckDB is an in-process SQL OLAP database management system that allows you to perform fast analytical queries directly within a single application, making it particularly useful for data science and analytical tasks.

## The Problem It Solves
Before DuckDB, developers often resorted to heavy data warehouses or complex tooling for analytical queries, leading to high costs, setup times, and awkward integrations. DuckDB simplifies this by providing a lightweight, embeddable SQL engine that runs fast queries directly from your local environment or within data science frameworks.

## Why It's Worth 10 Minutes of Your Time
1. **In-Process Execution**: DuckDB runs directly in your application’s process, enabling you to query large datasets efficiently without the overhead of network communication or a separate database server.
2. **Compatibility with Data Frames**: It integrates seamlessly with Python libraries like Pandas and R, allowing you to execute SQL queries directly on data frames, which optimizes workflows for data scientists and analysts.
3. **Adaptive Optimization**: Its query planner is designed for analytical workloads, offering robust performance for aggregated queries and a simple setup process, so you can scale your data projects without hassle.

## Quick Start
```bash
# Install DuckDB using pip
pip install duckdb

# Run a quick example in Python
import duckdb
con = duckdb.connect()
con.execute("CREATE TABLE test AS SELECT * FROM range(1, 100)")
result = con.execute("SELECT avg(column1) FROM test").fetchall()
print(result)  # Outputs: [(50.5,)]
```

## Real-World Example
```python
import duckdb
import pandas as pd

# Sample data in a DataFrame
df = pd.DataFrame({'value': [10, 20, 30, 40]})

# Querying directly using DuckDB
result = duckdb.query("SELECT avg(value) as average FROM df").to_df()
print(result)  # Outputs: average
```

## Who Should Use It
✅ Perfect for: Data scientists looking for an easy-to-use SQL engine for exploratory data analysis, developers needing a quick SQL solution without heavy lifting, or teams wanting to run analytics directly on large CSV or Parquet files.  
⏩ Skip if: You're working with transactional databases or require a full-fledged relational database management system (RDBMS) for complex transactions and high concurrency.

## The Honest Take
DuckDB is an impressive tool that offers a seamless SQL experience for analytical workloads without the overhead of traditional databases. It's lightweight and fast, but since it is designed primarily for OLAP rather than OLTP, it may not be suitable for applications requiring high transaction volumes. Bottom line: If you deal with data analysis or research, DuckDB is worth trying out.
