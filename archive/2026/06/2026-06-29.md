## 📄 DexCompose: Reusing Dexterous Policies for Multi-Task Manipulation with a Single Hand

**Authors:** Dihong Huang, Zhenyu Wei, Zhuxiu Xu, Yunchao Yao et al.
**Published:** 2026-06-26
**arXiv ID:** [2606.28323v1](https://arxiv.org/abs/2606.28323v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2606.28323v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2606.28323v1)

---

## 🎯 The Core Idea 
DexCompose allows robots to reuse existing skills for more complex tasks by figuring out which fingers to use for each specific action. This way, robots can seamlessly switch between tasks without messing up what they learned before.

## 🔍 The Problem They Solved 
Before this research, it was difficult to train robots to perform multiple tasks with a single hand without interfering with existing skills. Adding new tasks often led to conflicts in finger usage, making it hard for robots to maintain stability while adapting to new actions.

## 💡 The Key Innovation 
The novel approach in DexCompose involves a framework that assigns specific "ownership" of tasks to individual fingers, preventing interference between different skills. This role-aware system allows robots to efficiently adapt their finger movements based on the requirements of the new tasks without losing their ability to perform previously learned skills.

## 🛠️ Why Developers Should Care 
By introducing DexCompose, developers can build AI systems that handle multiple manipulation tasks more effectively without needing to retrain from scratch. This efficiency saves time and resources, allowing for faster deployment of more capable robotic systems.

## ⚡ TL;DR 
DexCompose helps robots use their hands better by letting them combine old skills with new ones without messing up, making them more versatile and efficient.

## 💡 Coding Challenge


TITLE: Bit Counter
DIFFICULTY: Easy
TOPIC: Bit Manipulation
ESTIMATED_TIME: 15 minutes

PROBLEM:
Your task is to write a function that takes an integer as input and returns the count of '1' bits (also known as the Hamming weight) in its binary representation. This is useful in digital systems where it is important to assess the number of active bits in a number.

EXAMPLES:
Input: 5
Output: 2
Explanation: The binary representation of 5 is `101`, which has two '1' bits.

Input: 13
Output: 3
Explanation: The binary representation of 13 is `1101`, which has three '1' bits.

CONSTRAINTS:
- The input integer will be a non-negative integer up to 2^31 - 1.
- Your solution must run in O(log n) time, where n is the input integer.

HINT:
You can repeatedly check if the least significant bit is a '1' and use bitwise right shift to process the next bit.

SOLUTION_PYTHON:
```python
def count_one_bits(n: int) -> int:
    count = 0
    while n > 0:
        count += n & 1  # Check if the least significant bit is a '1'
        n >>= 1        # Right shift to process the next bit
    return count

# Example usage
print(count_one_bits(5))  # Output: 2
print(count_one_bits(13)) # Output: 3
```

SOLUTION_JS:
```javascript
function countOneBits(n) {
    let count = 0;
    while (n > 0) {
        count += n & 1; // Check if the least significant bit is a '1'
        n >>= 1;       // Right shift to process the next bit
    }
    return count;
}

// Example usage
console.log(countOneBits(5));  // Output: 2
console.log(countOneBits(13)); // Output: 3
```

TIME_COMPLEXITY: O(log n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach leverages bit manipulation to efficiently count the number of '1' bits in the integer. By checking the least significant bit using the bitwise AND operator and then right-shifting the number, you can count how many bits are set to '1'. This is optimal as it only processes each bit once, resulting in a logarithmic time complexity relative to the size of the number.

FOLLOW_UP:
What if you need to count the number of '1' bits in a list of integers? How would you modify your function to handle that?

# Hypothesis

**Category:** Testing
**Link:** [https://github.com/HypothesisWorks/hypothesis](https://github.com/HypothesisWorks/hypothesis)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/HypothesisWorks/hypothesis)

---

## What Is It?
Hypothesis is a property-based testing library for Python that allows developers to write tests that generate test cases automatically, helping to ensure the robustness of code by exploring a wide range of input scenarios.

## The Problem It Solves
Before Hypothesis, developers often relied on manually crafting various inputs for their unit tests, which can be tedious and may miss edge cases. Hypothesis addresses this pain by generating diverse test cases based on specified properties, effectively uncovering issues that might go unnoticed with traditional testing methods.

## Why It's Worth 10 Minutes of Your Time
1. **Automatically Generates Input Data:** Instead of creating a fixed set of test cases, Hypothesis generates a broad range of inputs. This feature can reveal hidden bugs in your code by testing unexpected edge cases it would be difficult to think of manually.
2. **Intuitive Assertions:** Hypothesis allows you to write tests in a highly readable manner, making it simple to articulate the properties your code should satisfy, which can lead to clearer and more maintainable tests.
3. **Easy to Integrate:** It works seamlessly with existing testing frameworks like unittest and pytest, making it straightforward to add into your current test suite without a complete overhaul.

## Quick Start
```bash
pip install hypothesis
```

## Real-World Example
```python
from hypothesis import given
import hypothesis.strategies as st

@given(st.lists(st.integers()))
def test_sum_is_non_negative(lst):
    assert sum(lst) >= 0
```

## Who Should Use It
✅ Perfect for: Developers who want to write thorough, robust tests and find edge cases in their code easily. Ideal for libraries and complex systems where input variations are vast.  
⏩ Skip if: You prefer or are required to stick to example-based testing with a limited number of fixed inputs, or if you're working in environments where automated testing is not feasible.

## The Honest Take
Hypothesis is a powerful tool that can greatly enhance the testing process by introducing randomness, thereby increasing the likelihood of catching hard-to-find bugs. However, the generated test cases can sometimes produce unexpected results, which may initially confuse developers unfamiliar with the concept. Bottom line: if you're serious about your project's reliability, giving Hypothesis a try could pay off significantly in the long run.
