## 💡 Coding Challenge


TITLE: Count Bits in Range
DIFFICULTY: Easy
TOPIC: Bit Manipulation
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given two integers, `start` and `end`, representing a range of integers (inclusive). Your task is to count the total number of 1-bits in all integers from `start` to `end`. A 1-bit is a bit that is set to 1 in the binary representation of the integer.

For example, if you consider the binary representation of the integers, you need to sum up the count of bits that are 1 for every integer in that range.

EXAMPLES:
Input: `start = 3`, `end = 5`
Output: `4`
Explanation: 
The binary representations are:
- 3: `011` (2 ones)
- 4: `100` (1 one)
- 5: `101` (2 ones)
Total = 2 + 1 + 2 = 5

Input: `start = 1`, `end = 2`
Output: `3`
Explanation: 
The binary representations are:
- 1: `01` (1 one)
- 2: `10` (1 one)
Total = 1 + 1 = 2

CONSTRAINTS:
- 0 <= start <= end <= 10^6
- The difference between `end` and `start` can be at most `1000`.

HINT:
Consider how you can use bitwise operations to determine the number of 1-bits in a single integer.

SOLUTION_PYTHON:
```python
def count_bits_in_range(start, end):
    # Function to count 1-bits in a given number
    def count_one_bits(n):
        count = 0
        while n:
            count += n & 1  # Increment count if the last bit is 1
            n >>= 1  # Shift the bits to the right
        return count
    
    total_count = 0
    
    # Iterate over the range from start to end
    for num in range(start, end + 1):
        total_count += count_one_bits(num)
    
    return total_count

# Example usage:
print(count_bits_in_range(3, 5))  # Output: 4
print(count_bits_in_range(1, 2))  # Output: 3
```

SOLUTION_JS:
```javascript
function countBitsInRange(start, end) {
    // Function to count 1-bits in a given number
    const countOneBits = (n) => {
        let count = 0;
        while (n) {
            count += n & 1; // Increment count if the last bit is 1
            n >>= 1; // Shift the bits to the right
        }
        return count;
    };
    
    let totalCount = 0;

    // Iterate over the range from start to end
    for (let num = start; num <= end; num++) {
        totalCount += countOneBits(num);
    }

    return totalCount;
}

// Example usage:
console.log(countBitsInRange(3, 5));  // Output: 4
console.log(countBitsInRange(1, 2));  // Output: 3
```

TIME_COMPLEXITY: O(n * m)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The problem involves iterating through each integer in the specified range from `start` to `end` and, for each integer, counting the number of 1-bits in its binary representation. The inner function utilizes bitwise operations to efficiently count the bits. As a result, the complexity is O(n * m), where `n` is the number of integers in the range and `m` is the average number of bits in those integers. Given the constraints, this approach is efficient and practical.

FOLLOW_UP:
How would you modify the solution to count the total number of bits across all integers in the given range instead of just the 1-bits?

## What Is It?
SigNoz is an open-source observability platform that helps developers monitor their applications’ performance and troubleshoot issues in real-time using metrics, logs, and traces.

## The Problem It Solves
Before SigNoz, developers often relied on piecemeal solutions for observability, which could lead to cumbersome setups and fragmented data. This fragmentation made it challenging to gain a holistic view of application performance and often resulted in lengthy debugging sessions when issues arose. SigNoz integrates all observability aspects into a single platform, simplifying the monitoring and troubleshooting processes.

## Why It's Worth 10 Minutes of Your Time
1. **Unified Observability**: SigNoz combines metrics, traces, and logs into a single interface, making it easier to correlate data and diagnose performance bottlenecks without switching between multiple tools.
2. **Open Source Flexibility**: Being open source means you can tweak and customize the platform according to your project's needs, plus you gain access to a community for support and collaboration.
3. **Easy Installation and Setup**: SigNoz can be set up locally or in the cloud with just a few commands, making it a breeze for developers to start monitoring their applications quickly.

## Quick Start
```bash
# Install SigNoz using Docker
docker run -p 4260:4260 -p 4317:4317 -p 3000:3000 signoz/signoz
```

## Real-World Example
```javascript
// Example to send traces to SigNoz using OpenTelemetry
const { NodeTracerProvider } = require('@opentelemetry/node');

provider.register();
const tracer = provider.getTracer('example-tracer');

    // Your application logic here
} finally {
    span.end(); // Finalize the span
}
```

## Who Should Use It
✅ Perfect for: Developers looking to consolidate their observability tools, teams wanting to implement distributed tracing, or anyone interested in open-source solutions that can replace traditional monitoring tools.  
⏩ Skip if: You're already invested in a specific observability suite that doesn’t integrate well or if your project doesn’t require performance monitoring.

## The Honest Take
SigNoz provides an impressive way to improve observability with a unified approach that makes troubleshooting significantly easier. However, as with any open-source tool, you may encounter challenges in setup and maintenance that require additional time and resource investment. Overall, it's a solid choice for teams looking to improve their application's monitoring without the constraints of proprietary software.
