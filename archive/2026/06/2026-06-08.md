## 📄 Sparsely gated tiny linear experts

**Authors:** Simon Schug
**Published:** 2026-06-05
**arXiv ID:** [2606.07414v1](https://arxiv.org/abs/2606.07414v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2606.07414v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2606.07414v1)

---

## 🎯 The Core Idea
This research introduces a new way to design AI models called sparsely gated tiny linear experts (sgatlin), which uses very small, simple components—a single neuron—rather than larger, complex ones. By selecting only a few of these tiny components, they can create efficient models that require less computing power and are easier to understand.

## 🔍 The Problem They Solved
Previously, while mixtures of experts models attempted to enhance performance by using many experts, the individual components were still large and computationally demanding. This approach often led to increased complexity and costs without significant gains in efficiency or interpretability.

## 💡 The Key Innovation
The novel approach of this paper is to eliminate the typical nonlinear transformations in expert models, allowing for a structure based entirely on simple linear neurons. By doing this, the researchers not only made the models much smaller and more efficient but also showed that these sparse models can outperform larger, more complicated structures under the same compute conditions.

## 🛠️ Why Developers Should Care
This innovation allows developers to build AI systems that require less computational power while maintaining or even improving performance, particularly in language tasks. It opens up opportunities for deploying more accessible AI models, especially in resource-constrained environments or applications where efficiency is paramount.

## ⚡ TL;DR
New AI model design using tiny, linear components makes machine learning faster and simpler while boosting performance.

## 💡 Coding Challenge


TITLE: Remove Duplicates from a Sorted Linked List
DIFFICULTY: Easy
TOPIC: Linked Lists
ESTIMATED_TIME: 20 minutes

PROBLEM:
You are given the head of a sorted singly linked list where some elements may appear more than once. Your task is to remove all duplicate nodes such that each element appears only once in the list. Since the list is sorted, all duplicates will be adjacent to each other. You need to modify the list in place and return the head of the modified linked list.

EXAMPLES:
Input: 1 -> 1 -> 2 -> 2 -> 3 -> null
Output: 1 -> 2 -> 3 -> null
Explanation: The duplicates 1 and 2 are removed as all elements should only appear once.

Input: 2 -> 2 -> 2 -> 3 -> 4 -> 4 -> null
Output: 2 -> 3 -> 4 -> null
Explanation: After removing duplicates, we are left with 2, 3, and 4.

CONSTRAINTS:
- The linked list may contain 0 to 1000 nodes.
- The values of the nodes will be integers and can range from -10^4 to 10^4.

HINT:
Try iterating through the list and compare each node's value with the next node's value to identify duplicates.

SOLUTION_PYTHON:
```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def deleteDuplicates(head: ListNode) -> ListNode:
    current = head
    
    while current and current.next:
        if current.val == current.next.val:
            # Skip the next node
            current.next = current.next.next
        else:
            # Move to the next node
            current = current.next
            
    return head
```

SOLUTION_JS:
```javascript
class ListNode {
    constructor(val = 0, next = null) {
        this.val = val;
        this.next = next;
    }
}

function deleteDuplicates(head) {
    let current = head;
    
    while (current && current.next) {
        if (current.val === current.next.val) {
            // Skip the next node
            current.next = current.next.next;
        } else {
            // Move to the next node
            current = current.next;
        }
    }
    
    return head;
}
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach involves iterating through the linked list while maintaining a pointer to the current node. If two adjacent nodes have the same value, we skip the next node by changing the current node's next reference to bypass the duplicate. This process is repeated until we traverse the entire list. The algorithm runs in linear time since each node is checked only once, and it uses constant space, modifying the list in place.

# Zellij

**Category:** CLI
**Link:** [https://github.com/zellij-org/zellij](https://github.com/zellij-org/zellij)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](

---

## What Is It?
Zellij is a terminal workspace and multiplexer that enhances your command line experience by allowing you to split windows, manage multiple sessions, and collaborate with others—all within a single interface.

## The Problem It Solves
Before Zellij, developers often relied on terminal multiplexers like tmux or screen, which could be cumbersome and less user-friendly for newcomers. Zellij addresses this pain point with a more modern, intuitive interface that simplifies managing terminal sessions and providing features like layout persistence and collaboration.

## Why It's Worth 10 Minutes of Your Time
1. **Modern User Interface**: Zellij boasts a visually appealing and responsive UI that makes it easier to navigate and organize your terminal sessions compared to older tools.
2. **Collaboration Features**: It allows real-time collaborative sessions, enabling teams to work together in the same terminal, which is perfect for pair programming or training sessions.
3. **Flexible Layout Management**: With Zellij, you can create and save your custom layouts, making it easy to switch between different workflows without having to rearrange your workspace each time.

## Quick Start
```bash
# Install Zellij
cargo install zellij

# Start Zellij and create a new session
zellij
```

## Real-World Example
```bash
# Start a new Zellij session
zellij

# Split the terminal vertically
Ctrl + Shift + V

# Run a development server in one pane and monitor logs in another
cd my-app && npm start  # In one pane
tail -f app.log         # In another pane
```

## Who Should Use It
✅ Perfect for: Developers looking for a modern terminal experience, teams needing collaboration tools, or users of terminal multiplexers looking for a more intuitive alternative.  
⏩ Skip if: You prefer classic CLI tools without modern GUI enhancements or have minimal terminal work to do.

## The Honest Take
Zellij is a fresh and exciting alternative to traditional multiplexers, tailored for an enjoyable user experience and team collaboration. While it may require some adjustment for seasoned tmux users, its modern features and simplicity make it a standout choice in the realm of terminal tools.
