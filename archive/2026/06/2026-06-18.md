## 📄 Adaptive Speech-to-Spike Encoding for Spiking Neural Networks

**Authors:** Taharim Rahman Anon, Jakaria Islam Emon
**Published:** 2026-06-17
**arXiv ID:** [2606.19039v1](https://arxiv.org/abs/2606.19039v1)

[![Read Paper](https://img.shields.io/badge/Read_Paper-arXiv-B31B1B?style=flat-square&logo=arxiv)](
https://arxiv.org/abs/2606.19039v1)
[![PDF](https://img.shields.io/badge/Download_PDF-B31B1B?style=flat-square&logo=adobe)](
https://arxiv.org/pdf/2606.19039v1)

---

## 🎯 The Core Idea
This paper introduces a new way to convert continuous speech signals into spike-based signals that are used by Spiking Neural Networks (SNNs). The system learns to encode speech adaptively while being trained alongside a neural network to improve performance.

## 🔍 The Problem They Solved
Previously, there was a mismatch between the continuous nature of speech and the discrete processing of SNNs, making them less efficient for speech recognition tasks. Current encoders were fixed and did not adapt to different speech inputs, which limited their effectiveness.

## 💡 The Key Innovation
The novelty of this work lies in creating a learnable speech-to-spike encoder that dynamically adapts to the specific characteristics of speech signals. By training this encoder together with a Recurrent Leaky Integrate-and-Fire (R-LIF) network, they achieved a compact and efficient representation that outperformed existing systems in accuracy without requiring a large number of parameters.

## 🛠️ Why Developers Should Care
This approach allows developers to build more efficient and accurate speech recognition systems using spiking neural networks, which can operate with lower power consumption. By leveraging the learnable aspect of the encoder, developers can create more adaptable AI systems that perform well under varying input conditions.

## ⚡ TL;DR
Researchers developed a smart system that converts speech to spike signals for neural networks, achieving high accuracy with fewer resources.

---

## 📝 Original Abstract

> The mismatch between continuous acoustic signals and discrete event-driven processing remains a fundamental bottleneck for neuromorphic speech processing. Current systems typically rely on fixed spike encoders, forcing downstream Spiking Neural Networks (SNNs) to compensate for non-adaptive input representations. To address this, we present a learnable residual speech-to-spike encoder jointly trained end-to-end with a Recurrent Leaky Integrate-and-Fire (R-LIF) backbone. We validate this approach on the Google Speech Commands v2 (GSC-v2) benchmark, achieving up to 94.97% accuracy. Notably, the learned encoder remains highly parameter-efficient with a compact 35k-parameter variant that reaches 89.8%, matching or exceeding prior baselines that require an order of magnitude more parameters. ## 💡 Coding Challenge


TITLE: Find the Common Substrings
DIFFICULTY: Medium
TOPIC: Hash Maps & Sets
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given two strings, `s1` and `s2`. Your task is to find all the substrings that appear in both strings and return them in a list. A substring is defined as a contiguous sequence of characters within a string. If there are no common substrings, return an empty list. The order of the substrings in the output should be the same as they first appear in `s1`.

EXAMPLES:
Input: 
```python
s1 = "abcdxyz"
s2 = "xyzabcd"
```
Output: 
```python
["a", "ab", "abc", "abcd", "b", "bc", "bcd", "c", "cd", "d", "x", "xy", "xyz", "y", "z"]
```
Explanation: The substrings for both strings are generated, and the common ones are noted.

Input: 
```python
s1 = "hello"
s2 = "world"
```
Output: 
```python
[]
```
Explanation: There are no common substrings between "hello" and "world".

CONSTRAINTS:
- 1 <= len(s1), len(s2) <= 1000
- The characters in `s1` and `s2` are limited to lowercase English letters.

HINT:
Consider using a sliding window approach on one string to generate substrings while employing a hash set to store substrings from the other string for quick lookups.

SOLUTION_PYTHON:
```python
def common_substrings(s1, s2):
    # Using a set to store substrings from s2
    substrings_s2 = set()
    
    # Generate all substrings of s2 and add to the set
    for i in range(len(s2)):
        for j in range(i + 1, len(s2) + 1):
            substrings_s2.add(s2[i:j])
    
    # Now, find common substrings in s1
    common = []
    for i in range(len(s1)):
        for j in range(i + 1, len(s1) + 1):
            if s1[i:j] in substrings_s2:
                common.append(s1[i:j])
    
    return list(dict.fromkeys(common))  # Remove duplicates while preserving order

# Time Complexity: O(n^2 + m^2) where n and m are the lengths of s1 and s2.
# Space Complexity: O(m) for storing substrings of s2.
```

SOLUTION_JS:
```javascript
function commonSubstrings(s1, s2) {
    const substringsS2 = new Set();
    
    // Generate all substrings of s2 and add to the set
    for (let i = 0; i < s2.length; i++) {
        for (let j = i + 1; j <= s2.length; j++) {
            substringsS2.add(s2.slice(i, j));
        }
    }
    
    const common = [];
    // Now, find common substrings in s1
    for (let i = 0; i < s1.length; i++) {
        for (let j = i + 1; j <= s1.length; j++) {
            if (substringsS2.has(s1.slice(i, j))) {
                common.push(s1.slice(i, j));
            }
        }
    }
    
    return [...new Set(common)]; // Remove duplicates while preserving order
}

// Time Complexity: O(n^2 + m^2) where n and m are the lengths of s1 and s2.
// Space Complexity: O(m) for storing substrings of s2.
```

TIME_COMPLEXITY: O(n^2 + m^2)
SPACE_COMPLEXITY: O(m)

EXPLANATION:
The approach is to generate all possible substrings of the second string and store them in a set for efficient lookup. We then create substrings from the first string and check if they exist in the set. This method is efficient given the constraints, as the hash set allows for O(1) average-time complexity for lookups, making the overall solution manageable within the input limits.

# Mise

**Category:** Dev Tools
**Link:** [https://github.com/jdx/mise](https://github.com/jdx/mise)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/jdx/mise)

---

## What Is It?
Mise is a lightweight dev tool designed to simplify the process of managing multiple environments and configurations in your applications, making it easier to switch between them seamlessly.

## The Problem It Solves
Before Mise, developers often resorted to manually changing configuration files or using complex scripts to manage different environments like development, testing, and production. This not only led to mistakes but also created unnecessary friction and time consumption. Mise addresses these pain points by allowing developers to easily switch configurations with simple commands, streamlining workflows and reducing the risk of errors.

## Why It's Worth 10 Minutes of Your Time
1. **Simplicity**: Mise requires minimal setup, allowing you to jump straight into managing your environments without complicated configurations.
2. **Flexibility**: It supports various programming languages and frameworks, making it versatile for different projects — from Node.js to Ruby, and more.
3. **Environment-Specific Overrides**: Easily create overrides for different environments, helping ensure that your development and production configurations are tailored without the risk of overwriting critical settings.

## Quick Start
```bash
# Install Mise via npm
npm install -g mise

# Initialize Mise in your project
mise init

# Switch to a different environment
mise switch production
```

## Real-World Example
```javascript
// example.config.js
const config = require('mise').getConfig();

console.log(`API Base URL: ${config.API_BASE_URL}`);
// The output will depend on whether you're in development, testing, or production
```

## Who Should Use It
✅ Perfect for: Developers managing multiple environments who want to streamline their workflow and avoid configuration headaches.  
⏩ Skip if: You're working on small projects with no significant environment variations or if you prefer manual configurations.

## The Honest Take
Mise is a tool that truly lives up to its promise of simplifying environment management. While its flexibility is impressive, some users might find that it requires a bit of a mental shift from traditional methods. Overall, if you frequently juggle multiple environments, giving Mise a try could save you significant time and hassle.
