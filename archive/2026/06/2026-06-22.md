## 📄 JanusMesh: Fast and Zero-Shot 3D Visual Illusion Generation via Cross-Space Denoising

**Authors:** Siang-Ling Zhang, Huai-Hsun Cheng, Tsung-Ju Yang, Yu-Lun Liu et al.

## 🎯 The Core Idea
The researchers developed a new method called JanusMesh that can quickly create stunning 3D visual illusions with just text prompts, without needing any prior training. This means you can view one object from different angles and see entirely different shapes and meanings.

## 🔍 The Problem They Solved
Previous methods for generating 3D visual illusions were slow and often resulted in poor color quality or awkward seams between different geometric parts. Additionally, simple methods that just stitched 3D pieces together didn't produce realistic or coherent shapes, making the final product look unnatural.

## 💡 The Key Innovation
The novel aspect of JanusMesh is its unique two-stage approach, which uses a dual-branch denoising process to create 3D shapes. This method effectively combines various 3D representations while ensuring they fit together seamlessly, guided by text prompts, thereby enhancing both the visual quality and coherence of the generated shape.

## 🛠️ Why Developers Should Care
Developers can leverage JanusMesh to create immersive and visually appealing 3D models rapidly without extensive training datasets, streamlining workflows in fields like gaming and virtual reality. This enhances the potential for creativity and usability in applications that demand high-quality 3D content generation.

## ⚡ TL;DR
Researchers created a new way to quickly generate amazing 3D visual illusions from text descriptions, making 3D design faster and easier.

## 💡 Coding Challenge


TITLE: Reverse a Linked List in Groups of K
DIFFICULTY: Easy
TOPIC: Linked Lists
ESTIMATED_TIME: 30 minutes

PROBLEM:
You are given a singly linked list and an integer `k`. Your task is to reverse the nodes of the linked list `k` at a time and return the modified linked list. If the number of nodes is not a multiple of `k`, then the remaining nodes at the end should remain in the original order. 

For example, given a linked list `1 -> 2 -> 3 -> 4 -> 5` and `k = 2`, the result should be `2 -> 1 -> 4 -> 3 -> 5`. If `k = 3`, the result would be `3 -> 2 -> 1 -> 4 -> 5`.

EXAMPLES:
Input: head = [1, 2, 3, 4, 5], k = 2  
Output: [2, 1, 4, 3, 5]  
Explanation: The first two nodes (1 and 2) are reversed to form (2, 1), followed by the next pair (3, 4) which is reversed.

Input: head = [1, 2, 3, 4, 5], k = 3  
Output: [3, 2, 1, 4, 5]  
Explanation: The first three nodes (1, 2, and 3) are reversed to form (3, 2, 1), while the remaining nodes (4, 5) stay in the same order.

CONSTRAINTS:
- The number of nodes in the linked list will be between 0 and 1000.
- `k` is a positive integer (1 ≤ k ≤ 1000).

HINT:
Consider keeping track of the previous and the current nodes while reversing the linked list.

SOLUTION_PYTHON:
```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def reverseKGroup(head: ListNode, k: int) -> ListNode:
    count = 0
    ptr = head
    
    # Check how many nodes are in the linked list
    while ptr and count < k:
        ptr = ptr.next
        count += 1

    # If we have k nodes, reverse them
    if count == k:
        prev = None
        curr = head
        for _ in range(k):
            next_node = curr.next
            curr.next = prev
            prev = curr
            curr = next_node
        
        # Recur for the remaining nodes and link them
        head.next = reverseKGroup(curr, k)
        return prev  # prev is the new head of the reversed group

    return head  # If we don't have k nodes, return head unchanged
```

SOLUTION_JS:
```javascript
class ListNode {
    constructor(val=0, next=null) {
        this.val = val;
        this.next = next;
    }
}

function reverseKGroup(head, k) {
    let count = 0;
    let ptr = head;
    
    // Check how many nodes are in the linked list
    while (ptr && count < k) {
        ptr = ptr.next;
        count++;
    }

    // If we have k nodes, reverse them
    if (count === k) {
        let prev = null;
        let curr = head;
        for (let i = 0; i < k; i++) {
            const nextNode = curr.next;
            curr.next = prev;
            prev = curr;
            curr = nextNode;
        }
        
        // Recur for the remaining nodes and link them
        head.next = reverseKGroup(curr, k);
        return prev;  // prev is the new head of the reversed group
    }
    
    return head;  // If we don't have k nodes, return head unchanged
}
```

TIME_COMPLEXITY: O(n)
SPACE_COMPLEXITY: O(1)

EXPLANATION:
The approach involves traversing the linked list and reversing segments of it that are `k` elements long. We first check whether there are `k` nodes available to reverse. If there are, we reverse them and then recursively process the remaining part of the linked list. The space complexity is O(1) since we're only using a few variables for pointers and not storing any additional data structures.

# Drizzle ORM

**Category:** Databases
**Link:** [https://github.com/drizzle-team/drizzle-orm](https://github.com/drizzle-team/drizzle-orm)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/drizzle-team/drizzle-orm)

---

## What Is It?
Drizzle ORM is a TypeScript-first Object Relational Mapping library designed to simplify database interactions and improve developer productivity while working with SQL databases.

## The Problem It Solves
Before Drizzle ORM, developers often struggled with complex SQL statements and manual data mapping, leading to verbose and error-prone code. This tool addresses those pain points by providing a type-safe, intuitive API that makes interacting with databases more straightforward and less error-prone.

## Why It's Worth 10 Minutes of Your Time
1. **Type Safety**: Drizzle ORM provides full TypeScript support, ensuring that your database queries and entity mappings are type-checked at compile time, significantly reducing runtime errors related to data mismatches.
2. **Simplified Syntax**: It features a clean, expressive query syntax that is much more readable and easier to write than raw SQL, streamlining your development workflows.
3. **Rich Ecosystem**: Drizzle ORM offers built-in support for various SQL databases, as well as extensions for migrations and other utilities, allowing for a seamless development experience without needing to integrate multiple tools.

## Quick Start
```bash
npm install drizzle-orm
```

## Real-World Example
```typescript
import { drizzle } from 'drizzle-orm';
import { sqliteConnection } from 'sqlite-orm';

const db = drizzle(sqliteConnection('database.sqlite'));

const users = await db.select().from('users').where({ active: true }).execute();
console.log(users);
```

## Who Should Use It
✅ Perfect for: TypeScript developers looking for a powerful, type-safe ORM for SQL databases. Ideal for small to medium-sized projects or teams focused on minimizing boilerplate code.  
⏩ Skip if: You prefer SQL-based ORM tools that focus on relational databases without the constraints of TypeScript, or if your project does not utilize TypeScript.

## The Honest Take
Drizzle ORM excels in bringing type safety and simplified syntax to your data access layer, making it a solid choice for TypeScript enthusiasts. However, being relatively new, it might lack documentation or community support compared to more established ORMs, so evaluate if your project can accommodate that. Overall, it's a worthwhile tool to try if you want to enhance your database interaction while embracing TypeScript.
