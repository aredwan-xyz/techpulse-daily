## 🎯 The Core Idea
This paper introduces OmniGameArena, a versatile benchmark that evaluates vision-language model (VLM) agents across various types of game environments. It also introduces a new method called Improvement Dynamics Curve (IDC) to help these agents improve their skills over multiple attempts.

## 🔍 The Problem They Solved
Previously, benchmarks for VLM agents were limited to single scores for each game and did not consider different types of gameplay, such as cooperative or player-versus-player modes. There was no standardized way to compare different types of VLM agents, making it hard to evaluate their performance comprehensively.

## 💡 The Key Innovation
The OmniGameArena offers a cohesive set of twelve games built on Unreal Engine 5, which allows for consistent evaluation of various agent types across Solo, PvP, and Cooperative modes. The Improvement Dynamics Curve provides a framework for agents to iteratively refine their skills by using a tool that helps them reflect on their performance, enabling continuous learning rather than just measuring initial success.

## 🛠️ Why Developers Should Care
By using OmniGameArena, developers can more reliably benchmark their VLM agents across different gaming scenarios, ultimately leading to more robust AI that can handle diverse interactions. The IDC offers a structured way to improve agent skills, moving beyond basic assessments to foster real-world performance improvements.

## ⚡ TL;DR
OmniGameArena is a new benchmark system that helps AI gaming agents improve their skills through real-time evaluation across various game types.

## 💡 Coding Challenge


TITLE: Minimize Path Weight Sum in Directed Acyclic Graph
DIFFICULTY: Hard
TOPIC: Trees & Graphs
ESTIMATED_TIME: 45 minutes

PROBLEM:
You are given a Directed Acyclic Graph (DAG) consisting of `N` nodes (numbered from `0` to `N-1`) and a list of edges, where each edge has a specified weight. Your task is to find the minimum path weight sum from a start node to a goal node. The weight of a path is defined as the sum of the weights of all edges in that path. If there is no valid path from the start node to the goal node, return -1.

You are also given a list of nodes that can be treated as "blocked." If you pass through a blocked node, the path is considered invalid and cannot be counted.

EXAMPLES:
Input: 
```python
N = 5
edges = [(0, 1, 4), (0, 2, 3), (1, 3, 2), (2, 1, 1), (3, 4, 3)]
start = 0
goal = 4
blocked = [1]
```
Output: 6
Explanation: The only valid path from 0 to 4 is 0 -> 2 -> 1 -> 3 -> 4, where the weight sum is 3 + 1 + 2 + 3 = 9. However, since node 1 is blocked, no valid path exists leading to the goal, thus outputting -1.

Input: 
```python
N = 4
edges = [(0, 1, 5), (0, 2, 2), (1, 3, 1), (2, 3, 3)]
start = 0
goal = 3
blocked = []
```
Output: 5
Explanation: The optimal path from 0 to 3 is 0 -> 1 -> 3, yielding a weight sum of 5.

CONSTRAINTS:
- 1 ≤ N ≤ 1000
- 0 ≤ edges.length ≤ 10^4
- 1 ≤ weight of edges ≤ 100
- 0 ≤ len(blocked) ≤ N

HINT:
Consider using a combination of Depth-First Search (DFS) and dynamic programming approaches to keep track of the minimal weight encountered in valid paths.

SOLUTION_PYTHON:
```python
def min_path_weight_sum(N, edges, start, goal, blocked):
    from collections import defaultdict
    import heapq

    graph = defaultdict(list)
    for u, v, w in edges:
        graph[u].append((v, w))

    blocked_set = set(blocked)
    
    def dijkstra(start):
        min_weight = [float('inf')] * N
        min_weight[start] = 0
        pq = [(0, start)]  # (current_weight, node)

        while pq:
            current_weight, node = heapq.heappop(pq)

            if current_weight > min_weight[node]:
                continue
            if node in blocked_set:
                continue
            
            for neighbor, weight in graph[node]:
                next_weight = current_weight + weight
                if next_weight < min_weight[neighbor]:
                    min_weight[neighbor] = next_weight
                    heapq.heappush(pq, (next_weight, neighbor))

        return min_weight[goal] if min_weight[goal] != float('inf') else -1

    return dijkstra(start)

# Example usage:
print(min_path_weight_sum(5, [(0, 1, 4), (0, 2, 3), (1, 3, 2), (2, 1, 1), (3, 4, 3)], 0, 4, [1]))  # Output: -1
```

SOLUTION_JS:
```javascript
function minPathWeightSum(N, edges, start, goal, blocked) {
    const graph = {}; 
    edges.forEach(([u, v, w]) => {
        if (!graph[u]) graph[u] = [];
        graph[u].push([v, w]);
    });

    const blockedSet = new Set(blocked);
    const minWeight = Array(N).fill(Infinity);
    minWeight[start] = 0;
    
    const pq = [[0, start]]; // [current_weight, node]
    
    while (pq.length > 0) {
        const [currentWeight, node] = pq.shift();
        
        if (currentWeight > minWeight[node]) continue;
        if (blockedSet.has(node)) continue;
        
        if (graph[node]) {
            graph[node].forEach(([neighbor, weight]) => {
                const nextWeight = currentWeight + weight;
                if (nextWeight < minWeight[neighbor]) {
                    minWeight[neighbor] = nextWeight;
                    pq.push([nextWeight, neighbor]);
                    pq.sort((a, b) => a[0] - b[0]); // Maintain min-heap
                }
            });
        }
    }
    
    return minWeight[goal] === Infinity ? -1 : minWeight[goal];
}

// Example usage:
console.log(minPathWeightSum(5, [[0, 1, 4], [0, 2, 3], [1, 3, 2], [2, 1, 1], [3, 4, 3]], 0, 4, [1])); // Output: -1
```

TIME_COMPLEXITY: O(V + E log V)
SPACE_COMPLEXITY: O(V)

EXPLANATION:
The solution uses Dijkstra's algorithm to compute the minimum path weight sum in the directed acyclic graph. By maintaining a priority queue, the algorithm efficiently selects the next node with the minimum weight and explores its neighbors. Encountering blocked nodes is handled by skipping them during processing, ensuring paths that pass through blocked nodes are invalid. The overall approach optimizes for both time and space complexity suitable for the problem constraints.

FOLLOW_UP:
Modify the function to also return the actual path taken (as a list of nodes) along with the minimum weight sum.

# Starship

**Category:** CLI
**Link:** [https://github.com/starship/starship](https://github.com/starship/starship)

[![Repo](https://img.shields.io/badge/View_on_GitHub-181717?style=flat-square&logo=github)](
https://github.com/starship/starship)

---

## What Is It?
Starship is a fast and customizable prompt for your shell that can display information from various programming languages and frameworks right in your command line interface.

## The Problem It Solves
Before Starship, developers often had to rely on generic shell prompts that provided minimal contextual information. Customizing prompts required extensive manual configuration or third-party plugins that could lead to compatibility issues. Starship streamlines this process by offering intelligent defaults and a wide range of customizable options that keep you informed without bogging down your workflow.

## Why It's Worth 10 Minutes of Your Time
1. **Speedy Performance:** Starship is designed to be fast, with minimal startup time, ensuring your shell remains responsive even with complex configurations.
2. **Customization Made Easy:** With a simple configuration file, you can tailor the prompt to show exactly what you want, from Git status to the current Python environment, all without deep diving into shell scripting.
3. **Cross-Shell Compatibility:** Whether you use Bash, Zsh, Fish, or PowerShell, Starship works seamlessly across different shells, allowing you to maintain a consistent experience regardless of your setup.

## Quick Start
```bash
# Install Starship
sh -c "$(curl -fsSL https://starship.rs/install.sh)"

# Add the initialization script to your shell configuration
echo 'eval "$(starship init bash)"' >> ~/.bashrc  # For Bash users
```

## Real-World Example
```bash
# In your Starship configuration file (usually ~/.config/starship.toml)
[git_branch]
format = '[$branch]($style) '

# Output in your terminal after navigating to a Git repo might look like:
# username@machine ~ on main »
```

## Who Should Use It
✅ Perfect for: Developers seeking a highly informative and visually appealing command line experience, particularly those managing multiple programming environments and repository states.  
⏩ Skip if: You prefer a very minimalist shell prompt, or you don't want to spend time customizing your environment for aesthetics or efficiency.

## The Honest Take
Starship is a powerful tool that enhances your command line experience without overcomplicating it. Its main downside may be the initial setup for those who want a highly customized prompt, but for most users, the trade-off is well worth the time investment. Overall, it's a top choice for anyone looking to upgrade their shell.
